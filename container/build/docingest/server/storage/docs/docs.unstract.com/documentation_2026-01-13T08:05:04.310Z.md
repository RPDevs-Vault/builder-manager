# Networking FAQ | Unstract Documentation

[Skip to main content](https://docs.unstract.com/unstract/unstract_faq/networking/unstract_faq_networking/#__docusaurus_skipToContent_fallback)

On this page

All frequent questions related to networking.

### How to configure HTTPS scheme for accessing Unstract?[​](https://docs.unstract.com/unstract/unstract_faq/networking/unstract_faq_networking/#how-to-configure-https-scheme-for-accessing-unstract "Direct link to How to configure HTTPS scheme for accessing Unstract?")

1.  Set the following to `True` in `backend/.env`:
    
        SESSION_COOKIE_SECURE=TrueCSRF_COOKIE_SECURE=True
    
2.  Update scheme to `HTTPS` in `backend/.env`:
    
             # Django settings     DJANGO_APP_BACKEND_URL=https://<your_host>     # FE Web Application Dependencies     LOGIN_NEXT_URL="https://<your_host>/org"     LANDING_URL="https://<your_host>/landing"     ERROR_URL="https://<your_host>/error"     WEB_APP_ORIGIN_URL="https://<your_host>"
    
3.  See [here](https://docs.unstract.com/unstract/unstract_faq/networking/unstract_faq_networking/#i-already-have-a-albnginx-acting-as-reverse-proxy-how-can-i-deploy-unstract-behind-it)
     to deploy Unstract behind your existing ALB or Nginx reverse proxy OR see [here](https://docs.unstract.com/unstract/unstract_faq/networking/unstract_faq_networking/#how-to-run-unstract-on-another-port)
     to change port, if required.
    
4.  Do docker compose down and then up for `backend` service.
    

### How to access Unstract via another host?[​](https://docs.unstract.com/unstract/unstract_faq/networking/unstract_faq_networking/#how-to-access-unstract-via-another-host "Direct link to How to access Unstract via another host?")

1.  Update domain or IP address in `docker/docker-compose.yaml`:
    
          # Under backend service labels  - traefik.http.routers.backend.rule=Host(`<your_host>`) && PathPrefix(`/api/v1`, `/deployment`)  # Under frontend service labels  - traefik.http.routers.frontend.rule=Host(`<your_host>`)
    
2.  Update domain or IP address in `backend/.env`:
    
            # Django settings    DJANGO_APP_BACKEND_URL=http://<your_host>    # FE Web Application Dependencies    LOGIN_NEXT_URL="http://<your_host>/org"    LANDING_URL="http://<your_host>/landing"    ERROR_URL="http://<your_host>/error"    WEB_APP_ORIGIN_URL="http://<your_host>"
    
3.  See [here](https://docs.unstract.com/unstract/unstract_faq/networking/unstract_faq_networking/#how-to-configure-https-scheme-for-accessing-unstract)
     to configure HTTPS scheme, if required.
    
4.  Do docker compose down and then up for `backend` and `reverse-proxy` services.
    

### How to run Unstract on another port?[​](https://docs.unstract.com/unstract/unstract_faq/networking/unstract_faq_networking/#how-to-run-unstract-on-another-port "Direct link to How to run Unstract on another port?")

1.  Update port in `backend/.env`:
    
            # Django settings    DJANGO_APP_BACKEND_URL=http://<your_host>:<your_port>    # FE Web Application Dependencies    LOGIN_NEXT_URL="http://<your_host>:<your_port>/org"    LANDING_URL="http://<your_host>:<your_port>/landing"    ERROR_URL="http://<your_host>:<your_port>/error"    WEB_APP_ORIGIN_URL="http://<your_host>:<your_port>"
    
2.  Update port in `docker/docker-compose-dev-essentials.yaml`:
    
          # Under reverse-proxy service labels  - "80:80"            (OLD)  - "<your_port>:80"   (NEW)
    
3.  See [here](https://docs.unstract.com/unstract/unstract_faq/networking/unstract_faq_networking/#how-to-access-unstract-via-another-host)
     to change the host, if required.
    
4.  Do docker compose down and then up for `backend` and `reverse-proxy` services.
    

### I already have a ALB/Nginx acting as reverse proxy. How can I deploy Unstract behind it?[​](https://docs.unstract.com/unstract/unstract_faq/networking/unstract_faq_networking/#i-already-have-a-albnginx-acting-as-reverse-proxy-how-can-i-deploy-unstract-behind-it "Direct link to I already have a ALB/Nginx acting as reverse proxy. Once done, start forwarding requests from your ALB/Nginx to the IP address and port of the `unstract-proxy` container (in which the `reverse-proxy` docker compose service is running).

The actual ALB/Nginx configuration can be easily obtained online hence is outside the scope of this documentation.

# Destination Configuration | Unstract Documentation

[Skip to main content](https://docs.unstract.com/unstract/unstract_platform/features/workflows/destination_configuration/#__docusaurus_skipToContent_fallback)

On this page

This page details the configuration options available when setting up destination in your workflows. Destination configurations define where and how your processed data is stored.

File System Destination Configuration[​](https://docs.unstract.com/unstract/unstract_platform/features/workflows/destination_configuration/#file-system-destination-configuration "Direct link to File System Destination Configuration")

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

When using a File System as your destination connector (common in Task pipelines), you can configure where processed output files are stored.

### Output Folder[​](https://docs.unstract.com/unstract/unstract_platform/features/workflows/destination_configuration/#output-folder "Direct link to Output Folder")

Specifies the folder path where processed output files will be stored. This path is relative to the root of the configured filesystem connector.

**Best Practices:**

*   Use descriptive folder names that reflect the workflow purpose
*   Consider organizing outputs by date or workflow type
*   Ensure the output folder is different from input folders to avoid reprocessing

Database Destination Configuration[​](https://docs.unstract.com/unstract/unstract_platform/features/workflows/destination_configuration/#database-destination-configuration "Direct link to Database Destination Configuration")

---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

When using a Database as your destination connector (common in ETL and HITL workflows), you can configure how extracted data is stored in database tables.

![Destination Configuration](https://docs.unstract.com/unstract/assets/images/destination_config-a25a04d2f32e2dfd41b7144947e2bd16.png)

### Table Name[​](https://docs.unstract.com/unstract/unstract_platform/features/workflows/destination_configuration/#table-name "Direct link to Table Name")

The database table where processed output will be stored. If your database supports schemas, use the format `schema.table` (e.g., `public.invoices` or `analytics.documents`).

info

*   The table will be created if does not exist with the database user configured with the connector
*   If the table already exists, ensure that the database user has INSERT permissions on the table

### Include 'created\_by' column[​](https://docs.unstract.com/unstract/unstract_platform/features/workflows/destination_configuration/#include-created_by-column "Direct link to Include 'created_by' column")

When enabled, adds a `created_by` column to each output row, identifying the agent or process that created the record.

### Agent Name[​](https://docs.unstract.com/unstract/unstract_platform/features/workflows/destination_configuration/#agent-name "Direct link to Agent Name")

The value to use for the `created_by` column. Takes effect only when `created_by` is enabled. Defaults to "Unstract/DBWriter".

**Use Cases:**

*   Track which workflow or process generated each record
*   Enable multi-workflow auditing in shared tables
*   Support compliance and data lineage requirements

### Include 'created\_at' column[​](https://docs.unstract.com/unstract/unstract_platform/features/workflows/destination_configuration/#include-created_at-column "Direct link to Include 'created_at' column")

When enabled, adds a `created_at` column to each output row, recording when the data was processed and inserted.

### File Path Column Name[​](https://docs.unstract.com/unstract/unstract_platform/features/workflows/destination_configuration/#file-path-column-name "Direct link to File Path Column Name")

The name of the column that will store the absolute path to the source file that was processed. This helps link database records to source documents

### Execution ID Column Name[​](https://docs.unstract.com/unstract/unstract_platform/features/workflows/destination_configuration/#execution-id-column-name "Direct link to Execution ID Column Name")

The name of the column that will store the unique execution identifier for each workflow run. This ID helps track executions and is useful while debugging and viewing logs of the execution.

### Select how you want to write the output[​](https://docs.unstract.com/unstract/unstract_platform/features/workflows/destination_configuration/#select-how-you-want-to-write-the-output "Direct link to Select how you want to write the output")

Determines how the extracted data is structured in the database table.

**Available Options:**

#### Write JSON to a single column[​](https://docs.unstract.com/unstract/unstract_platform/features/workflows/destination_configuration/#write-json-to-a-single-column "Direct link to Write JSON to a single column")

Stores all extracted data as a JSON object in a single database column. This is the most flexible option and is currently the primary supported format.

#### Single Column Name[​](https://docs.unstract.com/unstract/unstract_platform/features/workflows/destination_configuration/#single-column-name "Direct link to Single Column Name")

The name of the column that will contain the JSON data

Best Practices[​](https://docs.unstract.com/unstract/unstract_platform/features/workflows/destination_configuration/#best-practices "Direct link to Best Practices")

---------------------------------------------------------------------------------------------------------------------------------------------------------------------

### For Database Destinations[​](https://docs.unstract.com/unstract/unstract_platform/features/workflows/destination_configuration/#for-database-destinations "Direct link to For Database Destinations")

1.  **Always include audit columns** in production workflows:
    
    *   Enable `created_at` for time tracking
    *   Enable `created_by` to identify the workflow
2.  **Use descriptive column names** that match your organization's naming conventions
    
3.  **Test table permissions** before deploying:
    
    *   Ensure INSERT permissions
    *   Verify column data types match expected output

### For Filesystem Destinations[​](https://docs.unstract.com/unstract/unstract_platform/features/workflows/destination_configuration/#for-filesystem-destinations "Direct link to For Filesystem Destinations")

1.  **Organize output folders** logically:
    
    *   Use date-based folders for time series data
    *   Separate folders by workflow type or data category
2.  **Avoid output/input folder overlap** to prevent circular processing
    
3.  **Plan storage capacity** based on expected file volumes

# Setting up LLM profiles | Unstract Documentation

[Skip to main content](https://docs.unstract.com/unstract/unstract_platform/features/prompt_studio/llm_profiles/#__docusaurus_skipToContent_fallback)

On this page

Overview[​](https://docs.unstract.com/unstract/unstract_platform/features/prompt_studio/llm_profiles/#overview "Direct link to Overview")

------------------------------------------------------------------------------------------------------------------------------------------

Unstract provides a sample profile with pre-configured LLM, embedding, vectorDB, and Text Extractor components with a quota of 1 million tokens. Users can also configure their own custom LLM profiles with preferred components.

Accessing LLM Profile Settings[​](https://docs.unstract.com/unstract/unstract_platform/features/prompt_studio/llm_profiles/#accessing-llm-profile-settings "Direct link to Accessing LLM Profile Settings")

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

1.  Click on the Settings gear icon (⚙️) in the top-right corner of the Unstract interface.
2.  In the Settings menu, select "LLM Profiles" from the left sidebar.

Default Sample Profile[​](https://docs.unstract.com/unstract/unstract_platform/features/prompt_studio/llm_profiles/#default-sample-profile "Direct link to Default Sample Profile")

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

By default, Unstract provides a sample profile with the following pre-configured components:

*   **Name**: sample profile
*   **LLM**: Azure GPT-4o LLM Free Trial
*   **Embedding Model**: Azure OpenAI Free Trial Embedding
*   **Vector Database**: Postgres Free Trial VectorDB
*   **Text Extractor**: LLMWhisperer Text Extractor Free Trial

Creating a New LLM Profile[​](https://docs.unstract.com/unstract/unstract_platform/features/prompt_studio/llm_profiles/#creating-a-new-llm-profile "Direct link to Creating a New LLM Profile")

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

1.  From the LLM Profile Settings page, click the "Add New LLM Profile" button.
    
2.  Fill in the required fields in the form:
    
    *   **Name**: Enter a name for your profile (e.g., "Profile 1")
    *   **LLM**: Select your preferred Language Model (e.g., Azure GPT-4o LLM Free Trial). Refer [LLM's section](https://docs.unstract.com/unstract/unstract_platform/adapters/llms/index/)
         to configure your preferred LLM.
    *   **Vector Database**: Choose your preferred vector database (e.g., Postgres Free Trial VectorDB).Refer [VectorDB section](https://docs.unstract.com/unstract/unstract_platform/adapters/vector_dbs/index/)
         to configure your preferred VectorDB.
    *   **Embedding Model**: Select your preferred embedding model (e.g., Azure OpenAI Free Trial Embedding).Refer [Embedding section](https://docs.unstract.com/unstract/unstract_platform/adapters/embedding/index/)
         to configure your preferred Embedding.
    *   **Text Extractor**: Choose your preferred text extraction tool (e.g., LLMWhisperer Text Extractor Free Trial).Refer [Text Extractor section](https://docs.unstract.com/unstract/unstract_platform/adapters/text_extractors/index/)
         to configure your preferred Text Extractor.
    *   **Chunk Size**: Set the chunk size (set to 0 if documents are small)
    *   **Overlap**: Configure the overlap value
    *   Before setting **Chunk Size** and **Overlap**, refer [Chunk and overlap section](https://docs.unstract.com/unstract/unstract_platform/user_guides/chunking/)
        .
3.  Advanced settings are applicable only when **Chunk Size** and **Overlap** are set.
    

*   **Retrieval Strategy**:
    *   **Simple**: Based on your prompt, the chunks will be retrieved.
    *   **Subquestion**: LLM will generate a set of specific subquestions from the prompt which can be used to retrieve relevant context from vector db. Fewer questions for simpler prompt, with a maximum subquestions of 10.
*   **Matching count Limit(Similarity top-k)**: Based on the count, the top number of chunks will be sent as an input context to LLM. For example if top-k is 3, then first 3 chunks will be sent to LLM as input context.

4.  Click the "Add" button to create your new LLM profile.

Managing LLM Profiles[​](https://docs.unstract.com/unstract/unstract_platform/features/prompt_studio/llm_profiles/#managing-llm-profiles "Direct link to Managing LLM Profiles")

# Run Platform | Unstract Documentation

[Skip to main content](https://docs.unstract.com/unstract/unstract_platform/user_guides/run_platform/#__docusaurus_skipToContent_fallback)

Use the following available options of `run-platform.sh` launch script to run Unstract Open Source Edition in Docker containers.

    # Pull and run entire Unstract platform with default env config../run-platform.sh# Pull and run docker containers with a specific version tag../run-platform.sh -v v0.1.0# Upgrade existing Unstract platform setup by pulling the latest available version../run-platform.sh -u# Upgrade existing Unstract platform setup by pulling a specific version../run-platform.sh -u -v v0.2.0# Build docker images locally as a specific version tag../run-platform.sh -b -v v0.1.0# Build docker images locally from working branch as `current` version tag../run-platform.sh -b -v current# Display the help information../run-platform.sh -h# Only do setup of environment files../run-platform.sh -e# Only do docker images pull with a specific version tag../run-platform.sh -p -v v0.1.0# Only do docker images pull by building locally with a specific version tag../run-platform.sh -p -b -v v0.1.0# Upgrade existing Unstract platform setup with docker images built locally from working branch as `current` version tag../run-platform.sh -u -b -v current# Pull and run docker containers in detached mode../run-platform.sh -d -v v0.1.0

Visit [http://frontend.unstract.localhost](http://frontend.unstract.localhost/)
 in your browser to access the platform.

**NOTE**: Modify the `.env` files present in each service folder to update its runtime behavior. Run `docker compose up -d` again for the changes to take effect.
