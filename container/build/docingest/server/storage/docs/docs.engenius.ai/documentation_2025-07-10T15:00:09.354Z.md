# Organization | Cloud User Manual

A collection of hierarchy views and networks that are part of a single organizational entity, such as a company or school district. Each organization is the owner of a single license.

[](https://docs.engenius.ai/engenius-cloud/working-with-org-trees/organizations#adding-an-organization)

Adding an organization


-----------------------------------------------------------------------------------------------------------------------------------

Click **Menu > Create Organization** button to create organization

# Working with Organization Trees | Cloud User Manual

EnGenius Cloud adopts an organization tree structure to let user define the scope of their managed networks. All device managing or monitoring functions can be applied to different scopes as laid out in the user's tree. That gives VAR or MSP users great flexibility in managing their networks.

The current organization tree structure consists of three levels, from largest to smallest:

*   Organization - A grouping of one or more hierarchies under the umbrella of a single license.
    
*   Hierarchy View - A cluster of networks, which may be geographically concentrated or spread out.
    
*   Network - A set of network devices united by a single configuration set.
    

The organization tree definition is shown on the top left corner of the web GUI as follows:

# Hierarchy View | Cloud User Manual

A hierarchy view is a group of networks and/or nested hierarchy views. It follows a tree-like structure much like folders on your computer's operating system.

[](https://docs.engenius.ai/engenius-cloud/working-with-org-trees/hyperviews#adding-a-hierarchy-view)

Adding a hierarchy view


----------------------------------------------------------------------------------------------------------------------------------

You can create hierarchy views for a new organization or an existing organization, or even within an existing hierarchy view. Click **Menu** > **Choose organization or hierarchy view > Add hierarchy view**

![](https://docs.engenius.ai/engenius-cloud/~gitbook/image?url=https%3A%2F%2F3893603398-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-legacy-files%2Fo%2Fassets%252F-LgVsoqihVR3I0VRYmyO%252F-Ll5zsMCADFU6Tfe9OFT%252F-Ll5zuAVQmb4MPjTn5Sl%252F1564553004830.jpg%3Falt%3Dmedia%26token%3D1e1941bf-7580-4eb0-a417-d34b49158c15&width=768&dpr=4&quality=100&sign=c4c469e6&sv=2)

[](https://docs.engenius.ai/engenius-cloud/working-with-org-trees/hyperviews#edit-hierarchy-views)

Edit hierarchy views


----------------------------------------------------------------------------------------------------------------------------

1.  You can edit the name of a hierarchy view name by clicking **Menu** > **Choose hierarchy view** **\> Edit**
    

![](https://docs.engenius.ai/engenius-cloud/~gitbook/image?url=https%3A%2F%2F3893603398-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-legacy-files%2Fo%2Fassets%252F-LgVsoqihVR3I0VRYmyO%252F-LnFwqP3erR54DRi6EB9%252F-LnFxs-_OcieEETJteIK%252F1566873536341.jpg%3Falt%3Dmedia%26token%3D584ebbf9-257a-4c02-818a-af462577810d&width=768&dpr=4&quality=100&sign=98707e01&sv=2)

![](https://docs.engenius.ai/engenius-cloud/~gitbook/image?url=https%3A%2F%2F3893603398-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-legacy-files%2Fo%2Fassets%252F-LgVsoqihVR3I0VRYmyO%252F-Lo4HnOJWl2_eGZa4cSv%252F-Lo4Hq2l4WHHUZckbXxi%252F1567751869542.jpg%3Falt%3Dmedia%26token%3D44e2fbaa-0ddd-4b99-92de-57bb61cdc5d3&width=768&dpr=4&quality=100&sign=edffbf5b&sv=2)

Move Network under Hierarchy View


------------------------------------------------------------------------------------------------------------------------------------------------------

After created Networks and associate devices, if the HV is changed, say, the Network originally belongs to North Region, and now become North-East Region, users have to change the HV and would like to move the Network under North-Region to North-East-Region. This allows users to "drag & drop" any Network to new HV under same Organization.

# Network | Cloud User Manual

A network contains a list of devices and relevant information, such as configuration, SSID, radio settings, and firmware upgrade history. Each network contains a single configuration set for its devices, so if you have multiple configurations for devices, you can create a separate network to handle that.

[](https://docs.engenius.ai/engenius-cloud/working-with-org-trees/networks#adding-a-network)

Adding a network


------------------------------------------------------------------------------------------------------------------

1.  Click **Menu > Choose organization or hierarchy > Create network**
    

![](https://docs.engenius.ai/engenius-cloud/~gitbook/image?url=https%3A%2F%2F3893603398-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-legacy-files%2Fo%2Fassets%252F-LgVsoqihVR3I0VRYmyO%252F-LkYL4c4NDgi2Bh3esWp%252F-LkYL6spKcNo86GDfcB7%252F1563961068936.jpg%3Falt%3Dmedia%26token%3Dd694e2fe-32db-4f59-a31d-da91604e8bf3&width=768&dpr=4&quality=100&sign=c4310dc2&sv=2)

2\. Enter a **name** for the network, select the **country**, **time zone,** and then click **Create.**

![](https://docs.engenius.ai/engenius-cloud/~gitbook/image?url=https%3A%2F%2F3893603398-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-legacy-files%2Fo%2Fassets%252F-LgVsoqihVR3I0VRYmyO%252F-Lo3fnabFG5RP-1TDWRY%252F-Lo3fp8dtNbx_RGpo7gn%252F1567741633873.jpg%3Falt%3Dmedia%26token%3D8b27992c-7ecd-4f1c-8d9c-bf0c81c3b9ab&width=768&dpr=4&quality=100&sign=d9fdf43&sv=2)
