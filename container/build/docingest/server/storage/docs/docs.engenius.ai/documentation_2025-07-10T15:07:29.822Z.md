# Managing Device Inventory and License | Cloud User Manual

The **Device** page lists all devices currently found in the inventory or added to a network within the current organization. The **Device** page contains the following information about each device:

*   **Type**: type of the device.
    
*   **Name**: Device name.
    
*   **Model**: Model name of the device.
    
*   **Serial Number**: serial number of the device.
    
*   **MAC**: MAC address of the device.
    
*   **Network**: the network that the device has been added to.
    
*   **License Status:** Active, Inactive, Merging, expired, you can see the detailed explanation in the license section.
    
*   **Expiration Date**: The date that the license is expired
    
*   **Register Time**: time of the device's addition to the inventory.
    
*   **Register by**: user responsible for adding the device to the inventory.
    

![](https://docs.engenius.ai/engenius-cloud/~gitbook/image?url=https%3A%2F%2F3893603398-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252F-LgVsoqihVR3I0VRYmyO%252Fuploads%252FQH0OfvCo7em9JpTtdosJ%252F1668394857912.jpg%3Falt%3Dmedia%26token%3D4f0e29ac-6663-45fe-88d8-a09db21a0b1b&width=768&dpr=4&quality=100&sign=a9a9e9a3&sv=2)

On the top of the right corner, you could see below information

**Earliest expired date on of devices on**: This will display the earliest expired date of the devices in the organization to remind users to add license by then.

**Expired devices:** The **n**umber of expired devices in this organization. For example, If it’s AP, and since AP license mode is “Pro”, so those AP cannot be managed by Cloud and show off-line if in Network.

**Expire within 30 days:** The number of the devices will expire within 30 days in this organization. Furthermore, cloud will send you the notification when there are devices to be expired within 30 days and within 3 days.

License status

**Inactive**: A license has not started to burn.

**Active**: The license starts to burn which means the license has started to tick and the time remaining starts to decrease.

**Merging**: There is a 7 days grace period for users to undo the license associated with a device, in case users place the license wrongly. Once “merged”, after 7 days or when the license is activated, the license is bundled with the device and cannot be removed. (the only way is “replacement”)

**Expired**: The license on the device is expired

**Merged**: When the license is associated with the device after 7 days grace period but hasn't been used (activated). For example, if a license is added to a device that has a license on it already, the newly-added license will become “merged” and the activation date will be the date after the previous license.

**Canceled**: When the license order is canceled, the license will be canceled, and the associated device's expired date will be deducted.

### 

[](https://docs.engenius.ai/engenius-cloud/managing-organizations/managing-device-inventory#id-3-month-pro-trial-license)

3-Month PRO-Trial License

Any newly registered devices will have a 3-month free PRO-trial license after May 1, 2024.

The activation date is the date the device is “registered” with the organization, and the expiration date is after 3 months. If the user de-registers the device and registers again, the cloud will keep the expired date.

Note: For purchased licenses associated with the device already, once the device is de-registered from the organization, Cloud cannot associate the licenses with the device anymore, so if the device registers with the organization again, it’s like a brand new one without the license.

Binding AI License to Organization


----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

When an AI license is bound to a specified organization, the license status is immediately activated. The quantity of AI Tokens associated with the license is automatically added to the total available EnGenius Tokens for the organization.

[](https://docs.engenius.ai/engenius-cloud/managing-organizations/managing-device-inventory/cloud-ai-license#no-grace-period-mechanism)

No Grace Period Mechanism


----------------------------------------------------------------------------------------------------------------------------------------------------------------------

The AI License does NOT support the “Grace Period" mechanism because once the Cloud-AI License is added to an organization, the AI Tokens within the license are immediately available for use by the Cloud-AI cameras.

[](https://docs.engenius.ai/engenius-cloud/managing-organizations/managing-device-inventory/cloud-ai-license#transfer-ai-tokens-between-organizations)

Transfer AI Tokens Between Organizations


----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

AI tokens can be transferred between different organizations, providing greater flexibility in EnGenius Token management. Therefore, when performing the "AI token transfer function” (action name: Transfer All AI Tokens), all remaining AI tokens will be moved from one organization to another.

![](https://docs.engenius.ai/engenius-cloud/~gitbook/image?url=https%3A%2F%2F3893603398-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252F-LgVsoqihVR3I0VRYmyO%252Fuploads%252FqcLcxdUNLYR2lvgGAjc4%252F%25E6%2588%25AA%25E5%259C%2596%25202025-06-17%2520%25E4%25B8%258B%25E5%258D%25883.43.26.png%3Falt%3Dmedia%26token%3Df4b50656-5d90-49c1-a871-794ade962abb&width=768&dpr=4&quality=100&sign=11d6502d&sv=2)

NO 90-Day Activation Mechanism


--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

Cloud AI license does NOT have a 90-day activation date mechanism.

# Configuration Template | Cloud User Manual

When you manage a large-scale network or end customers such as chain stores, which have very similar network configurations in every individual network, this function can help replicate/modify network configurations (including device configuration) and make the management more efficient,

You can access this page through **Organization > Backup & Restore > Configuration Template**

![](https://docs.engenius.ai/engenius-cloud/~gitbook/image?url=https%3A%2F%2F3893603398-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252F-LgVsoqihVR3I0VRYmyO%252Fuploads%252Fup5MgBse9WNHFO3owiSM%252F1722586682801.jpg%3Falt%3Dmedia%26token%3D2dcf811a-1af5-49fc-bc30-040bc594b7fc&width=768&dpr=4&quality=100&sign=871cd251&sv=2)

**Name:** the name of the template.

**Template Summary**: A quick summary helps the user understand what template is included in this network template.

**Last Modify**: Time of last modification.

**Modify by**: User account who made the last modification.

Note : Any note you want to add on the template.

[](https://docs.engenius.ai/engenius-cloud/managing-organizations/backup-and-restore/configuration-template#add-network-template)

Add Network template


-----------------------------------------------------------------------------------------------------------------------------------------------------------

You can select the network that you want to copy from so the template will displayed

![](https://docs.engenius.ai/engenius-cloud/~gitbook/image?url=https%3A%2F%2F3893603398-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252F-LgVsoqihVR3I0VRYmyO%252Fuploads%252FqaFHYfV1WYfL6BfRHoD3%252F%25E6%2588%25AA%25E5%259C%2596%25202024-08-02%2520%25E4%25B8%258B%25E5%258D%25884.19.54.png%3Falt%3Dmedia%26token%3D55f73ee2-5ef7-49b7-ad09-8b7743793921&width=768&dpr=4&quality=100&sign=86520617&sv=2)

**Must know**

If you couldn't find the network you want to copy from,

Please check if you match the following conditions:

1\. Do not allow multiple gateways, switches, cameras, PDUs in a network template.

1.  Do not allow any AP that override the network-wide settings.

Apply Configuration


---------------------------------------------------------------------------------------------------------------------------------------------------------

When you apply configuration for the first time, you can assign networks to the network group by giving a name to the network group, or simply select the networks to apply configuration.

![](https://docs.engenius.ai/engenius-cloud/~gitbook/image?url=https%3A%2F%2F3893603398-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252F-LgVsoqihVR3I0VRYmyO%252Fuploads%252FA4mcDKe0hd6XsrESsdK8%252F1721635302794.jpg%3Falt%3Dmedia%26token%3D21814cc8-99f2-4528-a757-87c42fa551bc&width=768&dpr=4&quality=100&sign=4fa7b30&sv=2)

there are some configurations that will not be copied from network when you add the network template

1.  Configure > Gateway > interface > WAN
    
2.  Configure > Gateway > Site-to-Site VPN
    
3.  Configure > Gateway > Client VPN

# Backup & Restore | Cloud User Manual

[](https://docs.engenius.ai/engenius-cloud/managing-organizations/backup-and-restore#generating-a-new-backup)

Generating a New Backup


------------------------------------------------------------------------------------------------------------------------------------------

Users can create a new Network-wide setting and device backup by going to **Organizations > Backup & Restore**

![](https://docs.engenius.ai/engenius-cloud/~gitbook/image?url=https%3A%2F%2F3893603398-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-legacy-files%2Fo%2Fassets%252F-LgVsoqihVR3I0VRYmyO%252F-MY41vtX6blr1ngiWSpk%252F-MY42hwLmyTH_9iWc6mT%252F1618213724250.jpg%3Falt%3Dmedia%26token%3Dcee61fab-c986-477c-9cfa-cc139d1ccbf5&width=768&dpr=4&quality=100&sign=6a16fd9f&sv=2)

*   **Restore:** This allows you to restore all settings( Network-wide settings and Device settings) to the corresponding network.
    
*   **Protect:** This allows you to protect the backup, so the backup will not be rotated when you exceed 2 backups of the network.
    
*   **Re-Backup:** This allows you to update the current settings to the backup of the corresponding
    
    network.
    

[](https://docs.engenius.ai/engenius-cloud/managing-organizations/backup-and-restore#clone-network)

Clone Network


----------------------------------------------------------------------------------------------------------------------

When creating a new network you have the option to clone the configuration from another network. This will copy all network-wide configurations from the existing network with the exception of local device configurations.

![](https://docs.engenius.ai/engenius-cloud/~gitbook/image?url=https%3A%2F%2F3893603398-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-legacy-files%2Fo%2Fassets%252F-LgVsoqihVR3I0VRYmyO%252F-MY4ODwMzAP8At7457lx%252F-MY4Uoc9ZQkRVFPNN5IE%252F1618221117683.jpg%3Falt%3Dmedia%26token%3D91cc7e88-8f5c-49cc-bc7d-24fb6d66b1ac&width=768&dpr=4&quality=100&sign=feb59a7&sv=2)
