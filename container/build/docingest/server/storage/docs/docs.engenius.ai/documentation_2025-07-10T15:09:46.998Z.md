# Notification Center | Cloud User Manual

EnGenius Cloud provides a notification mechanism for alerting you to important events that occurred. You can click the bell icon to access this screen.

# Configuring Alert Settings | Cloud User Manual

There are a number of options available for email alerts to be sent when certain network or device events occur.

Alerts can be configured under **Configure > Alerts.**

![](https://docs.engenius.ai/engenius-cloud/~gitbook/image?url=https%3A%2F%2F3893603398-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252F-LgVsoqihVR3I0VRYmyO%252Fuploads%252Fpqls21Lc2YHYb858jq5u%252F1692609023466.jpg%3Falt%3Dmedia%26token%3Df605e6f6-3dba-47f6-9397-8a3f3339801f&width=768&dpr=4&quality=100&sign=c31c775c&sv=2)

[](https://docs.engenius.ai/engenius-cloud/notification-and-alerts/configuring-alert-settings#access-point-alerts)

**Access Point Alerts**


-----------------------------------------------------------------------------------------------------------------------------------------------

Alerts can be configured for the following access point events:

![](https://docs.engenius.ai/engenius-cloud/~gitbook/image?url=https%3A%2F%2F3893603398-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252F-LgVsoqihVR3I0VRYmyO%252Fuploads%252FMvBhRQdo0slFvvwRPKxy%252F1683269484249.jpg%3Falt%3Dmedia%26token%3D084cc139-b931-4feb-ac50-2fbbc6a6c512&width=768&dpr=4&quality=100&sign=5fc9ffb6&sv=2)

*   **AP(s) go offline for XXX minutes:** sends an email if one or more access points go offline for a preset (and customizable) amount of time.
    
*   **When new firmware is released, send an email notification before the scheduled firmware upgrade:** send an email when new firmware is released before the scheduled firmware upgrade happened.
    
*   **Configuration changed within the network:** sends an email if SSID, radio settings, firmware upgrade, or individual device settings override the default settings.
    
*   **Rogue SSID is detected:** Sends an email when a non-authorized SSID is detected by AP within the wireless environment.
    
*   **Evil Twins Attack is detected: Send an email** when a malicious AP is imitating a legitimate SSID to deceive users into connecting.
    
*   **Malicious Attack is detected: Send an email** about active malicious threats such as deauthentication attacks, spoofing, or intrusion attempts.
    
*   **RF Jamming is detected:** Sends an email when there is interference that appears to intentionally disrupt RF communication.
    

[](https://docs.engenius.ai/engenius-cloud/notification-and-alerts/configuring-alert-settings#switch-alerts)

**Switch Alerts**


-----------------------------------------------------------------------------------------------------------------------------------

Alerts can be configured for the following switch events:

![](https://docs.engenius.ai/engenius-cloud/~gitbook/image?url=https%3A%2F%2F3893603398-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252F-LgVsoqihVR3I0VRYmyO%252Fuploads%252F1dZ7488pCIrqRCcDzmrQ%252F1683269506823.jpg%3Falt%3Dmedia%26token%3D29b73edc-7af4-42da-9e11-92c566d7458a&width=768&dpr=4&quality=100&sign=147aa12c&sv=2)

*   **Configuration changed within the network:** sends an email if SSID, radio settings, firmware upgrades, or individual device settings override the default settings.
    
*   **When new firmware is released, send an email notification before the scheduled firmware upgrade:** send an email when new firmware is released before the scheduled firmware upgrade happened.
    
*   **Switch port link status change:** sends an email when device port link status is changed.
    
*   **Switch STP Port status change:** sends an email when device port STP status is changed.
    
*   **Switch LBD Port status change:** sends an email when device LBD status is changed.
    
*   **Switch(s) go offline for XX minutes:** sends an email when switches go offline for a preset number of minutes.
    
*   **Any/specific switch port(s) changed link speed:** sends an email when a switch port link speed changes.
    

[](https://docs.engenius.ai/engenius-cloud/notification-and-alerts/configuring-alert-settings#gateway-alerts)

Gateway Alerts


---------------------------------------------------------------------------------------------------------------------------------

Alerts can be configured for the following EnGenius Gateway events:

![](https://docs.engenius.ai/engenius-cloud/~gitbook/image?url=https%3A%2F%2F3893603398-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252F-LgVsoqihVR3I0VRYmyO%252Fuploads%252FPydjYL1R7FMg6Uhu6wLX%252F1683270189935.jpg%3Falt%3Dmedia%26token%3D1ca8a1bd-e619-4c2b-b8e7-65182a5ce586&width=768&dpr=4&quality=100&sign=fdae9555&sv=2)

*   **Gateway goes offline for (x) minutes:** Sends an email if the gateway is unreachable from the Cloud for the configured number of minutes. Please note that the gateway may still be functioning, this only indicates that it is unable to contact the Cloud
    
*   **When new firmware is released, send an email notification before the scheduled firmware upgrade:** send an email when new firmware is released before the scheduled firmware upgrade happened.
    
*   **The DHCP lease pool is exhausted:** Which Indicates that the gateway has run out of available IPs in one or more of its configured DHCP scopes and is unable to provide an IP address to a requesting client.
    
*   **A VPN connection comes up or goes down**: Send an email if the VPN connection comes up or goes down.
    
*   **The primary uplink status changes:** Send an email if the status of the primary uplink changes, which could be a failover event or a downed link.
    
*   **Gateway clients connect or disconnect from the LAN:** Send an email if the gateway client connects or disconnects from the LAN.
    

[](https://docs.engenius.ai/engenius-cloud/notification-and-alerts/configuring-alert-settings#pdu-alerts)

PDU Alerts


-------------------------------------------------------------------------------------------------------------------------

Alerts can be configured for the following PDU events:

![](https://docs.engenius.ai/engenius-cloud/~gitbook/image?url=https%3A%2F%2F3893603398-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252F-LgVsoqihVR3I0VRYmyO%252Fuploads%252FsobTunCyqzxu0zgjlKPf%252F1692609467601.jpg%3Falt%3Dmedia%26token%3Da0f89e58-3def-44d8-bf87-509bd5f96bc3&width=768&dpr=4&quality=100&sign=93b62a7b&sv=2)

*   **PDU goes offline for (x) minutes:** Sends an email if the PDU is unreachable from the Cloud for the configured number of minutes. Please note that the PDU may still be functioning, this only indicates that it is unable to contact the Cloud
    
*   **When new firmware is released, send an email notification before the scheduled firmware upgrade:** send an email when new firmware is released before the scheduled firmware upgrade happened.
    
*   **Overall Power safe ranges:** send an email when the PDUs in the network has out of the power safe range.
    
*   **Overall Current safe ranges:** send an email when the PDUs in the network has out of the amp safe range.
    
*   **Individual outlet out of amp safe range:** send an email when the outlets in the network have out of the amp safe range.
    
*   **Individual PDU out of voltage safe range:** send an email when the outlets in the network have out of the voltage safe range.
