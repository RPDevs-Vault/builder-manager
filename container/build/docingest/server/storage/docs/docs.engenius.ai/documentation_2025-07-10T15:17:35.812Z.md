# License Overview | EnGenius Cloud Licensing

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/#start-with-essential-features-and-upgrade-with-pro-licenses)

**Start with essential features and upgrade with Pro licenses.**

EnGenius Cloud provides two types of feature plans, _**Basic and PRO**_. You need to set up the feature plan to either Basic or PRO for product categories.

The _**Basic**_ feature plan comes with essential cloud features for free, and the _**PRO**_ plan requires a Pro license to enable managed devices with professional features.

![](https://docs.engenius.ai/engenius-cloud-licensing/~gitbook/image?url=https%3A%2F%2F3299519939-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252F-MkFTP6mPDb3kJRwf2Rb%252Fuploads%252Fkv3VNRZowpCiphKjAmF3%252Fimage.png%3Falt%3Dmedia%26token%3D325525a0-998a-4752-bbff-71357e3a91ad&width=768&dpr=4&quality=100&sign=38302578&sv=2)

Diagram 1 - Feature Plan for

*   Users can choose any feature plan combination for different device categories. For example, you only need basic management features of Switches but require professional features of Access Points.
    
*   Each device category will follow its Feature Plan(_**Basic**_ _or_ _**PRO**_) to keep the logs and capacity of advanced features accordingly.
    
*   System-wide features like “admin users count, and Network limitation…etc.” will remain PRO as long as there is one _**PRO**_ feature plan is enabled for any device category.
    
*   Users can change the feature plan between _**Basic**_ _and_ _**PRO**_ anytime.
    

![](https://docs.engenius.ai/engenius-cloud-licensing/~gitbook/image?url=https%3A%2F%2F3299519939-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252F-MkFTP6mPDb3kJRwf2Rb%252Fuploads%252FFShFPXm4FcOXzOYa5N03%252Fnew%2520license%2520page.PNG%3Falt%3Dmedia%26token%3D08550d3e-9667-4dde-bb23-c439b7bbbf75&width=768&dpr=4&quality=100&sign=3d61a04f&sv=2)

When a device category is switched from _**PRO**_ _to_ _**Basic**_, all privileges that exceed the limit of the _**Basic**_ plan will be erased and all professional features related will be disabled.

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/#engenius-cloud-is-per-device-licensing)

**EnGenius Cloud is per-device licensing**

The **EnGenius Cloud License** on each device operates independently. Per-device licensing allows the IT managers to manage the validity of single or multiple licenses across sites and regions. In addition, per-device licensing makes network management more effortless.

Users with _**Org admin privilege**_ can associate the license to any registered device. Once you assign this _**license-associated device**_ to any Network in your Organization, the license is activated and will expire after the license term defined in the license model, as measured from the activation date.

# Terminology | EnGenius Cloud Licensing

EnGenius Cloud includes network hardware devices and a cloud platform, consisting of two essential components: an “Organization” and a “Network.” The following defined terms are used in this License Agreement.

Term

Description

**Associate**

Associate means to bind a License to a specific device.

**Associated Device**

A device with at least one associated License.

**License (or “Device License")**

One unit of usage rights granted to you and which can only be associated to one and only one device.

**License Activation Date**

The date on which a Device License commences, and thereby begins the licensing term.

**License Key**

A unique passcode (key) that a subscriber receives after the order is placed. A License Key may comprise multiple units of License of the same License model.

**License Issued Date**

The date that a license key is issued.

**Org Feature Plan**

The feature set available under different feature plans, including but not limited to a Basic Plan and a Pro plan, in the Organization.

# License Order and License Key | EnGenius Cloud Licensing

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/license-overview/license-order-and-license-key#license-order-and-license-key-dispatch)

License order and license key dispatch

Partners purchase EnGenius Cloud licenses just like purchasing other hardware devices from EnGenius, except "delivery email list" information is required for the License Key to be mailed to. Once license P.O. is completed, the EnGenius license server will generate License Keys based on the order content and email to the recipients listed in the "delivery email list".

*   A purchase order (P.O.) contains multiple order lines.
    
*   One order line will generate one license key.
    
*   An Email list is mandatory in the License purchase order.
    
*   License key information (PDF) will be sent to the "delivery email list".

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/license-overview/license-activation#scenarios-to-activate-cloud-licenses)

Scenarios to activate Cloud licenses

Each EnGenius Cloud Pro license comes with a validity period (depends on the license type you purchased) and will remain active after being activated until it is expired.

There are two ways to activate a license

![](https://docs.engenius.ai/engenius-cloud-licensing/~gitbook/image?url=https%3A%2F%2F3299519939-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252F-MkFTP6mPDb3kJRwf2Rb%252Fuploads%252FwC9NmhJPVsprpNKj51rM%252Fimage.png%3Falt%3Dmedia%26token%3D764c120a-edd2-41c9-bdc6-ad10dbc2c483&width=768&dpr=4&quality=100&sign=89c95caa&sv=2)

Diagram 3 - Scenarios to activate Cloud licenses

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/license-overview/license-activation#manual-activation)

**Manual Activation**

**Associate licenses to the device and assign the device to a Network**

When you assign this license-associated device to any network in your organization, the license will start to activate and operate during its validity period. For example, Device A/B are assigned to Network 123 and 456, so AP-License 1 and 2 are activated from the date they are assigned to the Networks.

The license associated with the device will not be activated during the planning period **until the device is assigned to Network** and apply the configuration.

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/license-overview/license-activation#forced-activation)

**Forced Activation**

**No license association or network assignment after 90 days of licenses issued**

Suppose you have never associated the license with any device, or the license-associated device has not been assigned to any managed network. In that case, this license will be automatically turned into an active state after it is issued for 90 days. _(Device C, diagram 3)_

After a license is forced activation, the validity period would start to burn. No matter the license is associated with a device or not.

# License Expiration | EnGenius Cloud Licensing

The validity period of the license for each device managed by EnGenius Cloud is independent. If the license of one device has expired, it will not impact other devices with valid licenses. You can still use the professional features and manage these devices whose licenses are still valid.

![](https://docs.engenius.ai/engenius-cloud-licensing/~gitbook/image?url=https%3A%2F%2F3299519939-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252F-MWghGkmPM86t8ANp9kZ%252Fuploads%252FOq8PXWS5gDFh5fFYWtVG%252FExpired%2520Device.jpg%3Falt%3Dmedia%26token%3D229fc655-c183-43b6-8113-41535467e34b&width=768&dpr=4&quality=100&sign=f6e812fc&sv=2)

Diagram 4 - License expiration for devices

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/license-overview/license-expiration#when-the-license-of-a-device-in-the-managed-network-expires)

**When the license of a device in the managed Network expires:**

*   When one of the devices exceeds the expired date of its license, the license-expired device is still functional as usual, but cannot be managed(off-lined) by EnGenius Cloud or using any Cloud-related features like Cloud authentication. _(Device D license is expired and off-lined)_
    
*   The other devices within the same Network with a valid license won’t be affected by the use of professional features. _(Device A&B will still be managed even Device D license is expired)_
    
*   If the Feature Plan is changed **from PRO to Basic**, all your devices (expired or active) can then be managed by EnGenius Cloud, all privileges exceed Basic limit will be erased and Pro features will be disabled. (see [Feature Plans Comparison](https://www.engenius.ai/cloud/licenses)
    )
    

For a license-expired device, to manage it with EnGenius Cloud again:

1.  Add a new license to the device (with professional features)
    
2.  Change the Feature Plan from PRO to Basic. (without professional features)

# Add License | EnGenius Cloud Licensing

To enable professional features for managed devices, you need to purchase **PRO License Key** from EnGenius resellers first and add it to your managed **Organization**.

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/license-overview/add-license#adding-a-license-to-the-organization)

**Adding a license to the Organization**

1.  Login to EnGenius Cloud and select the managed Organization
    
2.  Go to _**Inventory & License > Licenses**_
    
3.  Click ' **\+ Add License**' on the right
    
4.  Enter the **purchased license key(s)** and the system will unzip the license key(s) into one or multiple licenses. Click "OK", adding to Organization
    
5.  All licenses added will be displayed in the license table with related license information.

# Associate License | EnGenius Cloud Licensing

Associate a license means to bundle a license to a specific device.

After you select PRO Feature Plan and already added license to the Organization, you need to associate license(s) to the device(s) and assign the device(s) into a managed Network, thus those devices can use Professional features.

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/license-overview/associate-license#associate-license-s-with-the-device-s)

**Associate license(s) with the device(s)**

1.  Login to EnGenius Cloud and select the managed Organization
    
2.  Go to _**Inventory & License > Devices**_
    
3.  Click ' **\+ Assign License**' at the right of each device
    
4.  Select any available and valid license(s) to associate it to the device
    
5.  License-associated device(s) will display the information for license status and the expiration date.
    

![](https://docs.engenius.ai/engenius-cloud-licensing/~gitbook/image?url=https%3A%2F%2F3299519939-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252F-MkFTP6mPDb3kJRwf2Rb%252Fuploads%252FjOqS1uP7p8kjVwb5HEFb%252Fnew%2520device.PNG%3Falt%3Dmedia%26token%3Dc8e7d420-bd32-4fb3-94f5-7378aa09ba4d&width=768&dpr=4&quality=100&sign=9a8ef8f&sv=2)

# Move License | EnGenius Cloud Licensing

EnGenius Cloud licensing allows administrators to move the licenses and devices between Organizations in the following use cases.

*   Move device(s) and transfer associated licenses to another Organization
    
*   Move unassociated licenses to another Organization directly
    

When you move around licenses and devices between Organizations, you need to have privileges (**Organization administrator**) on both sides.

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/license-overview/move-license#moving-unassociated-licenses)

Moving unassociated licenses

*   Only the unassociated license, the license with the empty association, can be moved to another **Organization**
    

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/license-overview/move-license#moving-device-s-with-licenses)

Moving device(s) with licenses

*   When the device is moved to another **Organization**, the associated licenses will be moved simultaneously.
    
*   The license information will remain the same whenever you move devices (associated licenses) to another Organization.
    
    *   License Issue date
        
    *   License Activation date
        
    *   License Status

# License Status | EnGenius Cloud Licensing

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/license-guidelines/license-status#will-i-be-notified-before-my-license-expires)

Will I be notified before my license expires?

> By default, the system will automatically notify the user 60/30/3 days before any license is going to expire.

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/license-guidelines/license-status#what-happens-when-the-devices-license-expires)

What happens when the device's license expires?

> If the feature plan is "**PRO**", then when the device's license is expired, Those devices with license expired will still be functional as usual, but cannot be managed nor change any configuration via the EnGenius Cloud platform until the user switches the Feature Plan from "Pro" to "Basic".

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/license-guidelines/license-status#what-is-the-license-merging-merged-status)

What is the license merging/merged status?

> When a license is added to a device, there is a 7 days grace period to allow users to "undo" the association. Within the grace period, the license status is **Merging**. (Go to **Inventory & License > Licenses**)
> 
> After the grace period, the license will be bound with the device and cannot be unbundled. And then, the license status will change to **Merged**. (Go to **Inventory & License > Licenses**)
> 
> If more than two licenses are installed on the same device within a 7-day grace period, only one license can be disassociated, and the remaining licenses cannot be disassociated.

# License Operation | EnGenius Cloud Licensing

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/license-guidelines/license-operation#how-do-i-associate-the-license-with-the-device)

How do I associate the license with the device?

> A license can only be associated with a device; however, a device can have multiple licenses.
> 
> Two ways to add licenses:
> 
> 1.  Go to **Inventory & license > Devices page,** choose the device and click "Assign license".
>     
> 2.  Go to **Inventory & License > License page,** choose one available license, and associate to a device
>     

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/license-guidelines/license-operation#what-happens-when-i-change-feature-plans-from-pro-to-basic)

What happens when I change feature plans from PRO to Basic?

> After changing the feature plan from PRO to Basic, those professional features won't be available anymore. The logs or users exceeding the limit of "**Basic**" plan will be deleted, and it cannot be undone, even user switches back from "Basic" to "PRO".

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/license-guidelines/license-operation#what-happens-if-the-licensed-device-is-dis-registered-from-an-org)

What happens if the licensed device is dis-registered from an Org?

> When the device is dis-registered from an Organization, the licenses associated with the device will be gone, except the one-year free license that came with the newly purchased device.

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/license-guidelines/license-operation#will-the-90-day-free-trial-license-be-gone-when-the-device-is-unregistered-from-the-org)

Will the 90-day free trial license be gone when the device is unregistered from the Org?

> No. When the device is registered, the EnGenius Cloud will record the expired date (90 days after the registered date), so no matter how many times the device registers/un-register to a different Organization, the expired date of the free license will be kept the same.

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/license-guidelines/license-operation#can-i-move-licenses-between-organizations)

Can I move licenses between Organizations?

> Yes. You can move un-associated licenses to the other Organization you have the administrator privilege.

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/license-guidelines/license-operation#what-happens-if-i-move-a-device-with-multiple-licenses-to-another-organization)

What happens if I move a device with multiple licenses to another Organization?

> If the device has multiple licenses associated with it, all licenses will be moved to another Organization together with the device.

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/license-guidelines/license-operation#can-i-extract-a-license-from-an-organization)

Can I extract a license from an Organization?

> Users can move the license among the Organization the user has privilege. However, users cannot extract the license to re-enter other Organizations.

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/license-guidelines/license-operation#can-i-co-terminate-different-license-expiration-dates-for-devices)

Can I co-terminate different license expiration dates for devices?

> Yes. Users can purchase a **Co-Termination License** to add to the devices of a Network to make their expired date to be terminated on the same date. The **Co-Term Calculator** tool can help users to calculate how many units of Co-Termination license needs to be purchased.
> 
> For example, under same organization, there might be different networks with different accounting periods, so the termination date might be different. A **Co-Termination License** can ensure that the devices within a network have the same termination date.
> 
> It can be applied to multiple networks, but only to devices with 'Valid-License.

# General Licensing | EnGenius Cloud Licensing

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/license-guidelines/general-licensing#can-i-keep-using-the-basic-plan-for-free)

Can I keep using the Basic plan for free?

> YES. The Basic plan feature set includes the most-frequent-used features already and is perfectly suitable for SMB. If you don't need the advanced features, you can keep using the Basic features as before without any license required.

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/license-guidelines/general-licensing#how-do-i-change-my-feature-plan-in-the-org)

How do I change my feature plan in the Org?

> User can switch between Basic and Pro plan anytime manually under _**Org > Inventory & License.**_ However, please be reminded that if you want to use the Pro plan, make sure you have the Pro license associated with the devices you wish to manage.

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/license-guidelines/general-licensing#can-i-have-different-plans-for-different-networks)

Can I have different plans for different Networks?

> The Feature Plan is per Org basis and applies to all Networks under the Org. With the hierachical view design, users should be very easy to manage multiple Orgs with different feature plans under a single-pane-of-glass.

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/license-guidelines/general-licensing#how-do-i-purchase-a-cloud-license-and-apply-to-a-device)

How do I purchase a Cloud license and apply to a device?

> 1.  Contact your local EnGenius Resellers or EnGenius for details
>     
> 2.  Place the license order (had better a separated license order from hardware order) with a mail list which you would like the license key to be mailed to
>     
> 3.  One License Key will be generated per each line of the License Order (i.e. AP-1YR-LIC \* 10)
>     
> 4.  People in the mail list will then receive a mail with multiple PDF files. Each PDF file includes one unique License Key. For example, for a license order with 2 lines, AP-1YR-LIC \* 10 and SW-1YR-LIC \* 5, you will receive 2 PDF files in the mail, one with License Key of AP-1YR-LIC \* 10 and the other one with License Key of SW-1YR-LIC \* 5
>     
> 5.  You then can enter the License Key in the _**Inventory & License > License > Add license**_, where you can double-check the content of the License Key before applying.
>     
> 6.  Once the License Key is applied, multiple licenses will be generated by the License Key in the License page. In this case, 10 lines of AP-1YR-LIC and 5 lines of SW-1YR-LIC will be generated on the license page
>     
> 7.  You then can associate each License to a device.
>     

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/license-guidelines/general-licensing#how-many-types-of-cloud-licenses)

How many types of Cloud licenses?

> Cloud feature Plan includes Basic and Pro plan. For the "**Basic**" plan, no license is required for a device. However, for the "**PRO**" plan, users need to purchase a "**PRO**" license for each device.

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/license-guidelines/general-licensing#do-i-have-to-purchase-switch-pro-license-if-i-only-need-ap-pro-features)

Do I have to purchase Switch Pro license if I only need AP Pro features?

> If you only need AP Pro features, you just need to purchase Pro licenses for your AP device and keep the Switch Feature Plan to "Basic" without any Switch license required.

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/license-guidelines/general-licensing#how-does-licensing-vary-between-different-engenius-product-lines-e.g.-wireless-switches)

How does licensing vary between different EnGenius product lines, e.g., wireless, switches?

> EnGenius offers the flexibility to choose your plan for different product types, i.e., users can set the AP feature plan to "**PRO**" while SW feature plan to "**Basic**" in the case that users only need to use "**PRO**" features in AP and no need for PRO features in SW.

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/license-guidelines/general-licensing#do-i-have-to-buy-more-licenses-every-time-i-buy-new-hardware)

Do I have to buy more licenses every time I buy new hardware?

> When a new device is purchased, the device comes with one year free **PRO** license starting from the date the user registers the device to an Organization.

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/license-guidelines/general-licensing#do-i-need-to-have-a-license-for-every-engenius-cloud-hardware)

Do I need to have a license for every EnGenius Cloud hardware?

> For the "**Basic**" feature plan, no license is required for devices. However, if the user chooses the "**PRO**" feature plan, then all devices of the category need to have a valid license (not expired) associated with the device to be able to be managed by Cloud.

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/license-guidelines/general-licensing#will-a-device-in-inventory-require-a-license)

Will a device in inventory require a license?

> No. Only when the device is assigned to a Network under the "**PRO**" feature plan requires a valid license. Other devices in inventory do not need any license.

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/license-guidelines/general-licensing#when-will-the-license-start-to-burn-count-down)

When will the license start to burn (count-down)?

> No. Only when the device is assigned to a Network under the "**PRO**" feature plan requires a valid license. Other devices in inventory do not need any license.

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/license-guidelines/general-licensing#will-the-device-still-be-functional-when-the-license-expires)

Will the device still be functional when the license expires?

> YES. If the license is expired under the "**PRO**" feature plan, the device can still be functional as before the license expired, but it can not be configurable nor manageable from the EnGenius Cloud platform.

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/license-guidelines/general-licensing#if-i-choose-the-basic-feature-plan-will-the-license-keep-burning)

If I choose the "Basic" feature plan, will the license keep burning?

> YES. When the license starts to burn from "activation date", it will keep burning (countdown) until "Expired date" no matter if the user chooses which feature plans (Basic or RRO).

### 

[](https://docs.engenius.ai/engenius-cloud-licensing/license-guidelines/general-licensing#any-free-trial-for-the-professional-feature-plan)

Any free trial for the professional feature plan?

> Every device comes with a 90-day free "**Pro**" license for users to try on the "**Pro**" features, starting from the date the user registers the device to an Organization.
