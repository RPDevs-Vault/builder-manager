# Interactions Wiki | Interactions

[hashtag](https://ajneb97.gitbook.io/interactions#description)

Description


-------------------------------------------------------------------------------

This plugin allows you to add dialogues to your NPCs, creating complex conversation trees. You can add different options to these conversations, so the player can continue talking to the NPC writing the number of the option in the chat or clicking on it.

What makes **Interactions** better than any other _"conversation/dialogue/quests"_ plugin is that you have total control of the conversations you create. You can add requirements for an option to be selected, or even create "conditional dialogues" that will only be sent to the player if some conditions are met. You can also execute different actions to the player within a conversation, like giving him a potion effect or executing a command.

# How to Start | Interactions

[hashtag](https://ajneb97.gitbook.io/interactions/how-to-start#requirements)

Requirements


----------------------------------------------------------------------------------------------

**1) Spigot/Paper**

circle-exclamation

You need Spigot or Paper for the plugin to work. Don't use Craftbukkit.

**2) NPC Plugin**

circle-info

If you want to create conversations with NPCs you need one of the following plugins:

**Citizens:** [www.spigotmc.org/resources/citizens.13811/arrow-up-right](https://www.spigotmc.org/resources/citizens.13811/)
 **FancyNpcs:** [https://modrinth.com/plugin/fancynpcsarrow-up-right](https://modrinth.com/plugin/fancynpcs)

If you don't use one of these plugins you can still create conversations with other named entities. You can also use the `/interactions start` command to start conversations in certain locations even from other plugins (including other NPC plugins).

**3) PlaceholderAPI**

circle-info

This dependency is completely optional. If you want to create conditional dialogues or options with requirements you will need to install PlaceholderAPI on your server to get a lot of variables. [https://www.spigotmc.org/resources/placeholderapi.6245/arrow-up-right](https://www.spigotmc.org/resources/placeholderapi.6245/)
 The plugin also has support for PlaceholderAPI variables in dialogue messages!

circle-exclamation

Remember to follow the instructions for downloading variables. If you need to use variables from the **Player** expansion you need to use the following commands to get them to work:

1) /papi ecloud download Player 2) /papi reload

**4) ProtocolLib**

circle-info

This dependency is completely optional. You can use it to set the `block_movement_mode` option to `PROTOCOLLIB` in the config to completely block player movement while in conversation. [https://www.spigotmc.org/resources/protocollib.1997/arrow-up-right](https://www.spigotmc.org/resources/protocollib.1997/)

**5) Hologram Plugin**

circle-info

These dependencies are completely optional. You can use them to enable the `hologram_dialogues` option in [Conversation Entities](https://ajneb97.gitbook.io/interactions/conversation-entities-properties)
.

**Holographic Displays:** [https://dev.bukkit.org/projects/holographic-displaysarrow-up-right](https://dev.bukkit.org/projects/holographic-displays)
 **Decent Holograms:** [https://www.spigotmc.org/resources/96927/arrow-up-right](https://www.spigotmc.org/resources/96927/)
 **FancyHolograms:** [modrinth.com/plugin/fancyhologramsarrow-up-right](https://modrinth.com/plugin/fancyholograms)

[hashtag](https://ajneb97.gitbook.io/interactions/how-to-start#installation)

Installation


----------------------------------------------------------------------------------------------

To install the plugin on your server just place the Interactions file inside your plugins folder and start your server. Examples files will be generated in the **conversations** folder, you can use them as a reference or delete them.

# Creating a Conversation: Config | Interactions

There are 2 methods for creating a new conversation with a NPC or entity. You can use them both, but I suggest you to start using the first method since it is more interactive and you don't have to touch the config files.

[hashtag](https://ajneb97.gitbook.io/interactions/creating-a-conversation-config#using-the-config-file)

Using the Config file


----------------------------------------------------------------------------------------------------------------------------------

As well as the first method, you need to create a new conversation entity by using the `/interactions create <conversation>` command. A new file will be generated, but you'll see that is almost empty, with no conversations or dialogues. You can replace the config with the following text so you can start adding conversations, dialogues and options correctly.

Copy

    name: '&a&lExample NPC'
    starts_with: []
    block_movement: true
    start_conversation_radius: 0
    end_conversation_radius: 5
    conversation:
      conversation1:
        dialogue:
          dialogue1:
            text:
            - "&7Hello citizen, what can I do for you?"
            time: 4
        options:
          option1:
            text: "&eTell me about the city."
            start_conversation: conversation2
          option2:
            text: "&eBye"
            start_conversation: conversation3

With the current config we just have one dialogue `"Hello citizen, what can I do for you?"` that last 4 seconds and then, two options will be displayed to the player. These options currently do nothing since we haven't created the conversation2 and conversation3. The `start_conversation` path for each option defines what conversation should start when the player selects the option.

As you can see now, the conversations 2 and 3 were created. The conversation2 has 2 dialogues and the conversation3 just one. But, for now, when the dialogues of conversation2 are completed, the whole conversation will stop, so we can redirect the player to the options of conversation1 again by adding the`start_options: conversation1` property. If we don't set a _start\_options_ for conversation3 it means this conversation will be the last one.

Finally, we need to set a starting point for the conversation, in this case we want the player to right click a NPC with id "1" so we add this to the `starts_with` path.

Starting Points


------------------------------------------------------------------------------------------------------------------------

Conversations should start with an NPC or entity. You need to use the following format.

`1) NPC with id <id>`

`2) <entity> named <name>`

circle-info

The <name> value is the custom name of the entity. If you have a Zombie called "Bob", then this is the name. Don't use color codes.

circle-info

If you want to start a conversation without using a Citizens/FancyNpcs NPC, and instead, a Minecraft entity, replace the <entity> value with an EntityType from this link: [https://hub.spigotmc.org/javadocs/bukkit/org/bukkit/entity/EntityType.htmlarrow-up-right](https://hub.spigotmc.org/javadocs/bukkit/org/bukkit/entity/EntityType.html)

circle-info

If you are starting a conversation using the `/interactions start` command, you don't need to add a starting point.

Save Dialogue Option


----------------------------------------------------------------------------------------------------------------------

A very important option for dialogues is the "Save Dialogue" one. In some cases you will want the user to previously have reach to a certain dialogue in a conversation. For example, the first time a player greets a NPC the following dialogue will show: `"Hello citizen, what can I do for you"`.

But the next time, another different dialogue will be sent, meaning the player already talked to this NPC. If you want something like this to happen, enable this option. This will save this dialogue for the player and will allow the use of the following PlaceholderAPI variable:

`%interactions_has_dialogue_<conversation entity>.<conversation>.<dialogue>%`

This variable will return `true` if the player has reached to that dialogue, and `false` if not.

[hashtag](https://ajneb97.gitbook.io/interactions/dialogues-properties#conditional-dialogues)

Conditional Dialogues


------------------------------------------------------------------------------------------------------------------------

A conditional dialogue is a REPLACEMENT for the dialogue and will be sent to the player ONLY if some requirements are met. For example, we have this dialogue: `"Hello adventurer, I hope you are having a great day"`

We can add a conditional dialogue that will replace this dialogue ONLY if the player is lower than level 10. This can be done by adding the next requirement: `"%player_level% < 10"`

# Requirements | Interactions

This section is used for player options and conditional dialogues. Every line of the **requirements list** refers to a condition that MUST be true. You can use all kind of PlaceholderAPI variables.

circle-info

For example, in the config below, there is an option which requires the player to be level 20 or greater to appear on the chat. If the player doesn't accomplishes this condition the option will not be displayed.

Copy

    option3:
      text: "&eI am kind of a hero here."
      requires:
      - "%player_level% >= 20"
      start_conversation: conversation4

circle-info

You can also check for 'or' conditions, meaning that a condition line will be approved if one of many conditions in the same line is accomplished. For example, in this option there is just one condition. The player must be level 20 or greater, OR the player must be op.

Copy

    option3:
      text: "&eI am kind of a hero here."
      requires:
      - "%player_level% >= 20 or %player_is_op% == yes"
      start_conversation: conversation4

circle-info

You can also compare two variables in one condition line.

Copy

    conditions:
    - '%something% contains %player_name%'

# Conversation | Interactions

A conversation refers to a group of dialogues which follows an order. A [conversation entity](https://ajneb97.gitbook.io/interactions/keywords/conversation-entity)
 could have multiple conversations and all of them should end by showing the player some options, redirecting to other conversation, or ending the whole conversation.

The main parts of a conversation:

Attribute Name

Description

[**Dialogues**](https://ajneb97.gitbook.io/interactions/keywords/dialogue)

These are the dialogues present in this conversation, they follow

an order and each of them have unique properties. A conversation MUST

have dialogues.

[**Options**](https://ajneb97.gitbook.io/interactions/keywords/option)

How to start conversations from other NPC plugins


-------------------------------------------------------------------------------------------------------------------------------------------------------------------

Interactions has direct support with Citizens and FancyNpcs NPCs, but even so you can start a conversation from an NPC if you are using another plugin to manage them. You just need to add the `/int start <conversation> <player>` command to the NPC.

If you want to use the start/end radius feature, you can also start a conversation from a certain location using `/int start <conversation> <player> <x> <y> <z> <world>`.

Users can't click on options on chat or use the 'next' button


----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

If this happens its most likely you have a plugin that block all commands. Interactions manages these functionalities by commands, so you must whitelist these ones:

*   /interactions useoption
    
*   /interactions skipdialogue

# Variables Tutorial | ServerVariables

Each of the variables in the config follows a certain format. Here you can find the explanation of the options you can use. Everytime you modify something in this file you must reload the plugin using `/svar reload`

[hashtag](https://ajneb97.gitbook.io/servervariables/variables-tutorial#variable-type)

Variable Type


---------------------------------------------------------------------------------------------------------

Type of the variable. You can use PLAYER or GLOBAL.

*   GLOBAL: The variable is set for the whole server.
    
*   PLAYER: The variable is set for a player in particular.
    

Copy

    variable_type: GLOBAL

[hashtag](https://ajneb97.gitbook.io/servervariables/variables-tutorial#value-type)

Value Type


---------------------------------------------------------------------------------------------------

Value format of the variable. You can use TEXT, INTEGER or DOUBLE.

*   TEXT: The variable accepts all values.
    
*   INTEGER: The variable accepts only numbers.
    
*   DOUBLE: The variable accepts numbers with decimals.
    
*   LIST: The variable will consist of a list of values.
    

circle-exclamation

If you want to use the add/reduce commands you must set this option to INTEGER or DOUBLE.

Copy

    value_type: TEXT

[hashtag](https://ajneb97.gitbook.io/servervariables/variables-tutorial#initial-value)

Initial Value


---------------------------------------------------------------------------------------------------------

The initial or default value of the variable.

[hashtag](https://ajneb97.gitbook.io/servervariables/variables-tutorial#possible-values)

Possible Values


-------------------------------------------------------------------------------------------------------------

The possible values of this variable. You'll not be able to set the variable to a different value. Format: `<value>;(optional)<display>`

The "display" is optional and it is a visual representation of the variable which can be used as a PlaceholderAPI variable. Always use a single word for the value option, if you need multiple words, use something like this: "this\_is\_the\_first\_value".

circle-info

This parameter is completely optional.

[hashtag](https://ajneb97.gitbook.io/servervariables/variables-tutorial#limitations)

Limitations


-----------------------------------------------------------------------------------------------------

Conditions that the variable will have. You will not be able to change the variable if the new value doesn't respect these conditions.

### 

[hashtag](https://ajneb97.gitbook.io/servervariables/variables-tutorial#min-value)

Min Value

Minimum value of the variable. Only works with [Value Type](https://ajneb97.gitbook.io/servervariables/variables-tutorial#value-type)
 on INTEGER or DOUBLE.

### 

[hashtag](https://ajneb97.gitbook.io/servervariables/variables-tutorial#max-value)

Max Value

Maximum value of the variable. Only works with [Value Type](https://ajneb97.gitbook.io/servervariables/variables-tutorial#value-type)
 on INTEGER or DOUBLE.

### 

[hashtag](https://ajneb97.gitbook.io/servervariables/variables-tutorial#manage-out-of-range)

Manage Out of Range

If this option is set to true, when changing the variable's value, to something that is outside the max/min limits, the out of range error will disappear, and instead, the limit will be used as the new value.

### 

[hashtag](https://ajneb97.gitbook.io/servervariables/variables-tutorial#max-characters)

Max Characters

Maximum number of characters of the variable. Only works with [Value Type](https://ajneb97.gitbook.io/servervariables/variables-tutorial#value-type)
 on TEXT.

### 

[hashtag](https://ajneb97.gitbook.io/servervariables/variables-tutorial#max-decimals)

Max Decimals

Maximum number of decimals. Only works with [Value Type](https://ajneb97.gitbook.io/servervariables/variables-tutorial#value-type)
 on DOUBLE.

circle-info

In contrast to the previous options, this is not really a limitation, but a transformation, which means the final value of the variable will always have a maximum of X decimals, even if you used many more decimals on the command to modify its value.

Explanation


-----------------------------------------------------------------------------------------------------------

We have 6 different conversations, I will explain what each of them does.

conversation1

conversation2

conversation3

conversation4

conversation5

conversation6

The first time the player speaks with this NPC, it will ask for help and show 2 options. The first option will send the player to conversation2 and the second option to conversation3. If the player has already completed the quest `bob_the_king` , conversation6 will be started instead. If the player is currently doing the quest, conversation4 will be started. These conditions/requirements are managed by the conditional dialogues. In the case of the plugin Quests each variable will return something else (it's king of weird) `%quests_player_has_completed_quest_<quest>%` should return `true` or `false`. `%quests_player_has_current_quest_<quest>%` should return `Yes` or `No`.

The player wants to help the NPC so conversation2 starts and a console\_command action is defined, which will start the quest executing this command from the console: `questadmin give %player% <quest>`

The player doesn't want to help the NPC so nothing happens and the conversation ends.

This conversation will start when the player is currently doing the quest and speaks again with this NPC. First, the conditional dialogue will be verified. If the player has at least x1 Iron Ingot in its inventory, conversation5 will start. If the player doesn't have x1 iron ingot a dialogue will be sended telling that the player doesn't have the iron ingot and the conversation ends.

This conversation will start when the player speaks with the NPC having an iron ingot. The following command will be executed, finishing the quest: `questadmin finish %player% bob_the_king`

It is important also to remove the iron ingot from the player's inventory using the remove\_item action: `remove_item: IRON_INGOT;1`

This conversation will start when the player speaks with the NPC having already finished the quest. We don't want the player to access the quest again so we just send a new dialogue that ends the conversation.
