Baseten runs trained models as production endpoints. You package the model with [Truss](https://github.com/basetenlabs/truss)
, our open-source CLI, then push to Baseten for deployment, autoscaling, and observability.

[​](https://docs.baseten.co/development/model/overview#pick-a-starting-point)

Pick a starting point
------------------------------------------------------------------------------------------------------

Two paths cover most deployments:

*   **Config-only:** Most popular open-source LLMs deploy from a single `config.yaml`. Baseten compiles and serves them on TensorRT-LLM with an OpenAI-compatible API. Start with [Build your first model](https://docs.baseten.co/development/model/build-your-first-model)
    .
*   **Custom Python:** When you need preprocessing, postprocessing, or an unsupported architecture, write a `Model` class with `__init__`, `load`, and `predict`. Start with [Custom model code](https://docs.baseten.co/development/model/custom-model-code)
    .

For pre-built containers like vLLM, SGLang, or Triton, see [Custom Docker containers](https://docs.baseten.co/development/model/custom-server)
.

[​](https://docs.baseten.co/development/model/overview#anatomy-of-a-truss)

Anatomy of a Truss
------------------------------------------------------------------------------------------------

A Truss is the packaged unit you push to Baseten. It contains:

*   `config.yaml`: runtime environment, dependencies, GPU, and deployment settings. See [Configuration](https://docs.baseten.co/development/model/configuration)
    .
*   `model/model.py` (custom code only): the `Model` class with `load` and `predict`. See [Implementation](https://docs.baseten.co/development/model/implementation)
    .
*   `data/`, `packages/`, and weights: optional assets. See [Data and storage](https://docs.baseten.co/development/model/data-directory)
     and [BDN](https://docs.baseten.co/development/model/bdn)
     for large weights.

Development loop
--------------------------------------------------------------------------------------------

`truss push --watch` creates a [development deployment](https://docs.baseten.co/development/model/deploy-and-iterate)
 that live-patches code changes in seconds and scales to one replica. `truss push` (no flags) creates a published deployment, autoscaled and ready for production traffic. Iterate in development mode, then promote with `truss push --promote` when the model is ready. See [Deploy and iterate](https://docs.baseten.co/development/model/deploy-and-iterate)
 for the full workflow and [Environments](https://docs.baseten.co/deployment/environments)
 for promotion semantics.
