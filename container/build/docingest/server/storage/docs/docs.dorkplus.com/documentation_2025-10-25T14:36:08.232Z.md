# Dumper presets | DorkPlus Documentation

### 

[](https://docs.dorkplus.com/fundamentals/dumper-presets#how-it-works)

How it works

The default preset is made for combos, it'll target the columns with regex matching an email and column containing "**pass**".

### 

[](https://docs.dorkplus.com/fundamentals/dumper-presets#how-is-it-searching)

How is it searching

If you enter a keyword like "**password**" it'll search for columns that contain this keyword. Meaning if a column is called "**user\_password**" or "**accountpassword**" it'll automatically count it as a match.

[](https://docs.dorkplus.com/fundamentals/dumper-presets#options)

Options


------------------------------------------------------------------------------

*   must:
    
*   regex:
    

If you're using a regex string to identify emails for example it'll allow you to target any databases no matter what's the name of the column.

If you want to only dump the lines that match one column specifically then you'll use "**must:**" this way if it finds for example the must column "**pass**" it'll dump it, otherwise it'll skip.

### 

[](https://docs.dorkplus.com/fundamentals/dumper-presets#live-example)

Live example

Let's say I want to target combos, first we'll use a regex string to target emails and add "**must:regex:****string\_here**" to make sure there is an email, then we'll add "**must:****pass**" keyword to dump email & pass.

### 

[](https://docs.dorkplus.com/fundamentals/dumper-presets#how-to-use-options)

How to use options

When create a regex preset make sure to type "**regex:**" it'll add a small badge at the start with text "**regex**" you can now paste your regex string. Example below:

![](https://docs.dorkplus.com/~gitbook/image?url=https%3A%2F%2F1187884479-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252FRSbQDgRdAU6v635ZYzD7%252Fuploads%252FJSAoRd4sWbFLnV8u6HSs%252Fimage.png%3Falt%3Dmedia%26token%3Dddee694c-82a1-4ecf-9ce4-81644dc6d4fb&width=768&dpr=4&quality=100&sign=37609aad&sv=2)

As you can see on the example I also added "**must:**" cause I want to make sure there is an email when dumping. The order doesn't matter but you'll need to type "**must**:" and then "**regex:**", once you see the badge "regex & must" you can paste your regex string.

For phone numbers we could use this regex string: (won't fit for all numbers cause of country code, format...)

Copy

    /^\+?\d{1,4}[-.\s]?\(?\d{1,3}\)?[-.\s]?\d{1,4}[-.\s]?\d{1,4}[-.\s]?\d{1,9}$/

For emails we could use this one:

Copy

    ^[a-z0-9._%+\-]+@[a-z0-9.\-]+\.[a-z]{2,4}$

# Dorks to Lines | DorkPlus Documentation

[](https://docs.dorkplus.com/guides/dorks-to-lines#making-your-dorks-queries-that-are-used-on-the-scraper-parser-to-return-links)

Making your dorks (queries that are used on the Scraper/Parser to return links)


----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

1.  Identify your target (shopping, gaming, streaming...)
    
2.  Using your brain or AI make 50-100 base keywords and save to a text file (2-4 words, in the language of your target)
    
3.  Make a text file for the page types (php, asp, aspx), you can also use php only if you want to focus on recent results
    
4.  Make a text file for the parameters (id, product, article, collection...), you can find a public list of parameters below
    
5.  Upload all those files to your dashboard ([https://dash.dorkplus.com/](https://dash.dorkplus.com/)
    [upload](https://dash.dorkplus.com/uploads)
    )
    
6.  From your dashboard, top right toolbox icon: open the Dork Generator, keep Google enabled, select your files (keywords, page types and parameters), set the amount to generate (1K keywords for ~25-50K dorks)
    
7.  Hit generate, it'll be saved to your cloud storage
    

[](https://docs.dorkplus.com/guides/dorks-to-lines#scraper-parser-to-get-links)

Scraper/Parser (to get links)


------------------------------------------------------------------------------------------------------------------

#### 

[](https://docs.dorkplus.com/guides/dorks-to-lines#for-the-non-proxyless-plan-usd25-weekly-usd45-monthly)

For the non-proxyless plan ($25 weekly / $45 monthly)

*   Make a new task, select your threads (4vcpu & 8gb ram = 50-75 threads / 8vcpu 16gb ram = 100-200 threads), keep 7500 as timeout, select your machine, select your dorks, select your proxies
    
*   Don't run it yet, go to settings and select the search engines you want (yahoo will slow down the task, it's recommended to run yahoo alone, others can be combined)
    
*   Start your task, once it's done save the filtered links
    

#### 

[](https://docs.dorkplus.com/guides/dorks-to-lines#for-the-proxyless-plan-usd45-weekly-usd65-monthly)

For the proxyless plan ($45 weekly / $65 monthly)

*   Make a new task, select your threads (4vcpu & 8gb ram = 50-75 threads / 8vcpu 16gb ram = 100-200 threads), keep 7500 as timeout, select your machine, select your dorks, don't select any proxy
    
*   Don't run it yet, Google should be selected by default in the settings, make sure you're running Google only
    
*   Start your task, once it's done save the filtered links
    

[](https://docs.dorkplus.com/guides/dorks-to-lines#vulnerability-scanner-check-if-a-link-is-vulnerable-to-an-injection)

Vulnerability Scanner (check if a link is vulnerable to an injection)


--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

We recommend at least 100K links to be effective otherwise it'll be a waste of time.

*   Make a new task, select your threads (4vcpu & 8gb ram = 25-50 threads / 8vcpu 16gb ram = 75-100 threads), 7500/15000 as timeout, select your machine, select your links (filtered you saved from the scraper), select proxies (not required but recommended to avoid getting your rdp banned)
    
*   If you want to dump databases you don't need to edit the settings and can start right away, otherwise feel free to edit the settings for other type of vulnerability
    
*   Once it's done save your results, don't need to select DBMS it'll download all links
    

[](https://docs.dorkplus.com/guides/dorks-to-lines#dumper-to-inject-a-website-and-dump-their-database-that-contains-data-such-as-email-password-usernam)

Dumper (to inject a website and dump their database that contains data such as email, password, username...)


--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

We recommend at least 1K SQL vulnerable links to be effective otherwise it'll be a waste of time.

*   Load the links you saved from the vulnerability scanner, select your proxies (10-20 DC/ISP proxies recommended, resi won't work fine), select your machine
    
*   Don't start it yet, go to the settings and update as you want, for the threads 50/50 (main/secondary) if you've got 6-8vcpu and 16gb ram, 25/25 for 4vcpu and 8gb ram, better specs = you'll need to see what works the best
    
*   For the other settings it's up to you, default presets are for email:pass
    
*   Don't forget to hit save if you updated the settings
    
*   Start the dumper and let it finish
    
*   You'll see websites being added, it doesn't mean they're getting dumped, if you see rows being 0/0 that means it didn't find a match from the presets (eg. no email or no pass), you can filter the results by selecting from the dropdown next to Save results button to only show the Dumped ones
    
*   Once it's finished hit the Save results button
    

[](https://docs.dorkplus.com/guides/dorks-to-lines#dehasher-transform-the-hashed-lines-into-clean-lines)

Dehasher (transform the hashed lines into clean lines)


--------------------------------------------------------------------------------------------------------------------------------------------------------------------

It's an additional step but you'll most likely need to dehash your lines.

*   Select the list of your choice that you saved from the dumper (eg. \[2000\]\[website.com\]) select item:hash (item = email, hash = password), select your machine and hit start
    
*   If you see 0 results that means it couldn't be dehashed, next list
    

### 

[](https://docs.dorkplus.com/guides/dorks-to-lines#thats-it-just-need-to-repeat-the-process-now)

That's it, just need to repeat the process now

### 

[](https://docs.dorkplus.com/guides/dorks-to-lines#advices)

Advices

*   Don't just make random keywords, for example if you're targeting gaming, learn about what's currently trending in that niche, browse steam and see what games is trending, what dropped...
    
*   If you want to get the best results you'll need to get the proxyless plan to run the scraper on Google, otherwise you'll get way less results
    
*   When running the modules make sure to keep an eye on your CPU/RAM usages, if they're too high reduce the threads or it won't run properly, if it's low (under 50-60%) you can increase the threads and see how it goes
    
*   Running multiple tasks at the same time is possible but that means you'll need to adjust the threads accordingly
    
*   Resi proxies are good for the Scraper/Parser only, if you run the Vulnerability Scanner or Dumper you'll need ISP/DC proxies, they're faster and more stable

# Presets example | DorkPlus Documentation

[](https://docs.dorkplus.com/fundamentals/dumper-presets/presets-example#email-pass)

Email:pass

# Scraper/Parser | DorkPlus Documentation

### 

[](https://docs.dorkplus.com/pages/scraper-parser#valid-or-filtered)

Valid or Filtered?

The valid links are all links scraped from the search engine(s), meaning they're not sorted and contains a lot of duplicates, links without parameters and other useless lines.

We always recommend people to download filtered links if they want to be effective, save time and get high injectable rates.

The filtered links are cleaned links without duplicates, bad links or empty parameters.

### 

[](https://docs.dorkplus.com/pages/scraper-parser#page-count)

Page count

This feature works with all search engines except for Google since we already grab as much pages as possible.

### 

[](https://docs.dorkplus.com/pages/scraper-parser#url-check)

URL check

This setting will ensure you don't get links you already scraped before, it's a local AntiPublic system basically.

### 

[](https://docs.dorkplus.com/pages/scraper-parser#domain-check)

Domain check

The domain check is here to clean up the duplicates, if you get for example those 2 links:

Copy

    https://website.com/blog?id=123
    https://website.com/docs?id=456

It'll only keep one of these two links.

### 

[](https://docs.dorkplus.com/pages/scraper-parser#remove-duplicates)

Remove duplicates

This setting is here to back up the domain check if it's disabled. It'll remove exact same links and filter them out.

### 

[](https://docs.dorkplus.com/pages/scraper-parser#keep-unfiltered)

Keep unfiltered

This feature is only useful if you know what to do with links that are raw without parameters. Do not enable if you need links with parameters.

# Settings | DorkPlus Documentation

Setting name

Description

Concurrent dumps

The concurrent dumps are the number of websites you want to dump at one time, higher value will increase the loading on network, CPU and RAM.

Minimum rows

Target databases with a amount of rows that match your requirements

Auto-dumper

The auto-dumper setting will basically proceed each databases that match your requirements and dump all rows, no need to manually explore the databases.

Auto-dumper preset

Add column names as a preset and target the right data, let's say you enter "**email**" and "**password**", every database that has those 2 columns will be dumped, the others won't.

Maximum retries

Set the maximum amount of retries before skipping

Maximum timeout

Set how much time (in ms) you want to wait before skipping

Main threads

Set the amount of threads you want to use to process the websites (check if injection works)

Secondary threads

Set the amount of threads you want to use to process the databases (dump tables, columns, rows)

---------------------------------------------------------------------------------------------------------

Keywords

Dork Generator

**The goal**: First you need to identify your goal, do you plan to get lines around the Gaming theme, Streaming space or something else. Once you've got that in your mind let's get base keywords.

**The base keywords**: Now it's time to write a bunch of HQ keywords by yourself that are matching your goal, you can use ChatGPT to help you or just follow the good old days by typing on Amazon, Google, YouTube and grab some keywords.

Using the **Keyword Scraper:** When you've got about 50-100 keywords it's time to grab a LOT of keywords using our Built-in Keyword Scraper, once you've got them move to the next part, the **Page Types**.

Copy

    // Sample file for Gaming theme
    GTA V Stunt
    GTA V Compilation
    GTA V Money Glitch
    ...

A page type is basically the page extension, most of the time we'll go for "**php**" or "**asp**".

Copy

    // Sample file
    php
    asp
    aspx

A parameter is something after the domain and page type, eg. `https://store.com/``product` here we want to make parameters that match the "`product`".

Copy

    // Sample file
    product
    id
    q
    search
    collection
    category

We've made an Utility called "**Dork Generator**" which can be found on the top right menu next to the "**New task**" button.

# FAQ | DorkPlus Documentation

How much dorks do I need?[](https://docs.dorkplus.com/pages/faq#how-much-dorks-do-i-need)

It depends on the quality but we recommend at least 50K unique dorks (check for duplicates to avoid wasting time).

How can I make my own parameters?[](https://docs.dorkplus.com/pages/faq#how-can-i-make-my-own-parameters)

The best way to get your own parameters is to first get public ones, scrape a bunch of links using our parser and then use our URL utilities to extract the unique parameters from your links.

![](https://docs.dorkplus.com/~gitbook/image?url=https%3A%2F%2Fiili.io%2FdwFNWXa.png&width=300&dpr=4&quality=100&sign=e1efbb66&sv=2)

How can I talk to the support?[](https://docs.dorkplus.com/pages/faq#how-can-i-talk-to-the-support)

Open a ticket on our Discord server: [https://discord.com/invite/m7R8eDeRAh](https://discord.com/invite/m7R8eDeRAh)

How can I talk to the support?[](https://docs.dorkplus.com/pages/faq#how-can-i-talk-to-the-support-1)

Open a ticket on our Discord server: [https://discord.com/invite/m7R8eDeRAh](https://discord.com/invite/m7R8eDeRAh)

How can I increase the speed of the Scraper, Vulnerability scanner and other tasks?[](https://docs.dorkplus.com/pages/faq#how-can-i-increase-the-speed-of-the-scraper-vulnerability-scanner-and-other-tasks)

You can either increase task count or reduce timeout but don't go crazy. There is also limitation depending on your machine (hardware, network, VPN...)

What type of RDP do I need?[](https://docs.dorkplus.com/pages/faq#what-type-of-rdp-do-i-need)

The minimum requirement is 4vCPU and 8GB of ram, but we highly recommend getting at least 8vCPU and 16GB of ram for the best performances. Network speed also needs to be around 500-1000 mbps.

Do I need proxies or VPN?[](https://docs.dorkplus.com/pages/faq#do-i-need-proxies-or-vpn)

If you've got the proxyless plan you can run the parser without proxies or VPN. The other modules are by default working without proxies or VPN but we highly recommend using at least a VPN when running the vulnerability scanner or dumper for security reasons.

⚠️ Using a VPN or proxies can drastically impact the performances in a bad way, make sure to run some tests before using it. Most VPN such as ExpressVPN or NordVPN tend to reduce the speed and throw some timeouts.

I keep getting "Error" injection on the Dumper, is that fine?[](https://docs.dorkplus.com/pages/faq#i-keep-getting-error-injection-on-the-dumper-is-that-fine)

The "Error" injection is a type of injection like "Union", nothing to worry about, that means it's injecting properly.

All results are 0 or 0/0 on the Dumper, why?[](https://docs.dorkplus.com/pages/faq#all-results-are-0-or-0-0-on-the-dumper-why)

Make sure you've enabled the auto-dumper feature on the Dumper settings. Also you'll need to setup your presets properly, the default presets are for email:pass.

What are pagetypes?[](https://docs.dorkplus.com/pages/faq#what-are-pagetypes)

The pagetypes are important when making dorks, basically it'll allow you to get results with a specific extension like .php, .asp or .aspx. We recommend targeting those 3 but feel free to add more if you know what you're doing:

Copy

    asp
    aspx
    php

My DPM is good but my FPM is low, why?[](https://docs.dorkplus.com/pages/faq#my-dpm-is-good-but-my-fpm-is-low-why)

What does the error message "\[0830/060249.387:ERROR:window\_impl.cc(121)\] Failed to unregister class Chrome\_WidgetWin\_0. Error = 1412" means?[](https://docs.dorkplus.com/pages/faq#what-does-the-error-message-0830-060249.387-error-window_impl.cc-121-failed-to-unregister-class-chro)

It's a log from the UI part of the client. Nothing to worry about so you can just ignore it. It'll happen when you reload the client or clear the data most likely.

My storage is full but I don't have any files uploaded[](https://docs.dorkplus.com/pages/faq#my-storage-is-full-but-i-dont-have-any-files-uploaded)

The cloud storage is for uploads and results, meaning if you don't have any files but your storage is still full it's cause of your task history, move on the /history page and clean your history.

What does IDLE mean on the dumper?[](https://docs.dorkplus.com/pages/faq#what-does-idle-mean-on-the-dumper)

Idle means it's not dumping but it has been injected successfully. The reason for not dumping is cause of the presets or the filter settings such as minimum rows. Make sure to check your settings if you want to edit the presets.

# How to: Fix errors | DorkPlus Documentation

Reload client[](https://docs.dorkplus.com/how-to/how-to-fix-errors#reload-client)

Go to the "**Machines**" page and press the \[...\] button on your machine, then click "**Reload client**" it'll most likely fix the issue.

Browser extensions[](https://docs.dorkplus.com/how-to/how-to-fix-errors#browser-extensions)

Extensions on your browser may interfer with the Dashboard, it can be a crypto wallet, a screenshot tool or whatever, if you face that type of issue you've got 2 options:

*   Use an other fresh browser like Brave or Chrome (recommended)
    
*   Disable extensions
    
*   Use incognito mode
    

Restart the client[](https://docs.dorkplus.com/how-to/how-to-fix-errors#restart-the-client)

In some rare cases you might need to manually reload the client from your machine.

Logout & Log back in[](https://docs.dorkplus.com/how-to/how-to-fix-errors#logout-and-log-back-in)

This might solve some rare issues as well when you've got an error about a token

Clear cache[](https://docs.dorkplus.com/how-to/how-to-fix-errors#clear-cache)

Clearing the cache might help in some rare cases as well.

I'm running the dumper but I don't have any rows[](https://docs.dorkplus.com/how-to/how-to-fix-errors#im-running-the-dumper-but-i-dont-have-any-rows)

*   Make sure you have enabled the Auto-dumper feature from the settings ([Guide here](https://docs.dorkplus.com/pages/dumper-and-db-explorer#dumper)
    )
    
*   Make sure you added presets ([Guide here](https://docs.dorkplus.com/fundamentals/dumper-presets)
    )
    
*   Make sure you have enough links loaded, you'll get about 5% injection rate and about 5-10% dumping rates. Meaning if you load 1K links, you'll get around 50+ injectable and around 5-10 dumped databases (those are rough estimations)
    

My task is stuck what can I do?[](https://docs.dorkplus.com/how-to/how-to-fix-errors#my-task-is-stuck-what-can-i-do)

*   Check if your client is still running
    
*   If so, try reloading the client from the Machines page (click the \[...\] button)
    
*   If it doesn't fix try restarting the client manually from the server
    
*   If it's still not working you'll have to make the task again
    

Timeout error (Dashboard)[](https://docs.dorkplus.com/how-to/how-to-fix-errors#timeout-error-dashboard)

This error may happen cause of various reasons, it can be due to your extensions, browser/OS language, browser itself, AV, ISP, VPN or cache. Try clearing cache, if it doesn't fix then try opening the dashboard on your RDP or your own PC.

I've got the Proxyless plan but it asks for proxies[](https://docs.dorkplus.com/how-to/how-to-fix-errors#ive-got-the-proxyless-plan-but-it-asks-for-proxies)

The proxies option is always there but if you have the proxyless plan you don't need to upload any proxy list, you can leave it empty

When I open the client it instantly close[](https://docs.dorkplus.com/how-to/how-to-fix-errors#when-i-open-the-client-it-instantly-close)

*   Make sure you don't have more than 3 Machines total (inactive + active)
    
*   If you have more than 3 machines delete one and restart the client
    
*   If it doesn't work or you don't have 3 machines try deleting the machine where the client don't start and add the machine back
    

I can't see my API key[](https://docs.dorkplus.com/how-to/how-to-fix-errors#i-cant-see-my-api-key)

Try refreshing the Settings page and see if it's showing actual username, if not then that means we're working on the backend. If you see your username but it doesn't show/copy your API key then open a ticket on our Discord server.

My task isn't starting[](https://docs.dorkplus.com/how-to/how-to-fix-errors#my-task-isnt-starting)

*   Restart the client manually from your server/pc, close it and open it back
    
*   If that doesn't work delete the machine from Machines page and add it back
    

I have a low DPM/FPM on the Scraper[](https://docs.dorkplus.com/how-to/how-to-fix-errors#i-have-a-low-dpm-fpm-on-the-scraper)

If your DPM seems fast but the FPM under 500-1000 that could mean your dorks are returning results you already got which are filtered by our local Anti Public system to prevent you from having the same links over and over. If you want to get higher FPM and don't mind getting results you already scraped then you can delete the file **domains.db** from the same folder where the DPClient.exe is located.

If it doesn't change anything then it might be the quality of your dorks (Dork type) or your keywords not returning good results.

I can't download my results[](https://docs.dorkplus.com/how-to/how-to-fix-errors#i-cant-download-my-results)

If you get an internal server error when downloading your results or just nothing happens then it might be an issue with the task. Try running it again, if the issue persists contact us on Discord.

[](https://docs.dorkplus.com/knowledge#the-scraper-wont-start)

The Scraper won't start

*   Close the client and restart it
    
*   Check your settings (make sure you have at least one engine selected, if you use Google proxyless make sure there is only this engine selected and don't use proxies, if you don't have the Google proxyless make sure to use proxies)
    
*   Check the threads count, 4VCPU/8GB RAM: 50-75 Threads, 8VCPU/16-32GB RAM: 100-200 Threads
    
*   Make sure Windows Defender is disabled
    
*   Check your VPN if you use one
    

### 

[](https://docs.dorkplus.com/knowledge#the-dumper-wont-start)

The dumper won't start

*   Make sure you selected your machine
    
*   Make sure your proxies are working (run the task without and see if it works)
    
*   Check your threads, default settings 50/50 is for 4-6VCPU / 8GB RAM, play with the settings according to your usage
    

### 

[](https://docs.dorkplus.com/knowledge#the-keyword-scraper-dont-work)

The keyword scraper don't work

*   Check the threads, 25-50 for 4-6VCPU, 50-100 for 8VCPU
    
*   Don't use proxies
    
*   Make sure you update the settings to 2 words minimum cause 1 might return too much results for small machines
    

### 

[](https://docs.dorkplus.com/knowledge#the-vuln-scanner-dont-return-any-vuln)

The vuln scanner don't return any vuln

*   Drop the threads to half what you set (50-75 max for 4-6VCPU)
    
*   Check your proxies, run with and without see if there is a difference

I keep getting timeout error

It happens when your IP got a high score meaning it's a bit shady. Make sure to try with a VPN US/CA/EU location, if that doesn't work use a Resi proxy to access the dashboard with like Proxy omega extension.

The dork generator is stuck at loading

The dehasher doesn't dehash my lines

A dehasher will have to find a match based on the hashed string, it won't crack the hash so if it can't dehash your line you'll have to move to the next lines, invest in our Dedicated Dehasher (HPlus) or use hashcat to crack the hash.

I get a low DPM (Dork / minute)

*   Your proxies: slow proxies won't work properly
    
*   Your machine: if the network isn't stable/is slow
    
*   Your threads: too high it won't work fine, too low it won't be as fast as it could
    
*   Your dorks: high FPM means lower DPM

[](https://docs.dorkplus.com/knowledge#i-get-a-low-fpm-filter-minute)

I get a low FPM (Filter / minute)

*   Your dorks quality: bad dorks won't return much links and will lead to a low FPM
    
*   The threads are too high: 4VCPU/8GB RAM go for 50-75 threads, 8VCPU/16GB RAM 100-200
    
*   Local antipublic system: we've got a antipublic system built-into the client which filters the URLs you got in the past to make sure you don't scrape the same links over and over

The dumper presets are wrong

*   If you're using regex strings make sure they're valid
    
*   Don't use must: too much, could skip all lines if you don't use it properly

Making your dorks (queries that are used on the Scraper/Parser to return links)


-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

1.  Identify your target (shopping, gaming, streaming...)
    
2.  Using your brain or AI make 50-100 base keywords and save to a text file (2-4 words, in the language of your target)
    
3.  Make a text file for the page types (php, asp, aspx), you can also use php only if you want to focus on recent results
    
4.  Make a text file for the parameters (id, product, article, collection...), you can find a public list of parameters below
    
5.  Upload all those files to your dashboard ([https://dash.dorkplus.com/](https://dash.dorkplus.com/)
    [upload](https://dash.dorkplus.com/uploads)
    )
    
6.  From your dashboard, top right toolbox icon: open the Dork Generator, keep Google enabled, select your files (keywords, page types and parameters), set the amount to generate (1K keywords for ~25-50K dorks)
    
7.  Hit generate, it'll be saved to your cloud storage

Dumper (to inject a website and dump their database that contains data such as email, password, username...)


-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

We recommend at least 1K SQL vulnerable links to be effective otherwise it'll be a waste of time.

*   Load the links you saved from the vulnerability scanner, select your proxies (10-20 DC/ISP proxies recommended, resi won't work fine), select your machine
    
*   Don't start it yet, go to the settings and update as you want, for the threads 50/50 (main/secondary) if you've got 6-8vcpu and 16gb ram, 25/25 for 4vcpu and 8gb ram, better specs = you'll need to see what works the best
    
*   For the other settings it's up to you, default presets are for email:pass
    
*   Don't forget to hit save if you updated the settings
    
*   Start the dumper and let it finish
    
*   You'll see websites being added, it doesn't mean they're getting dumped, if you see rows being 0/0 that means it didn't find a match from the presets (eg. no email or no pass), you can filter the results by selecting from the dropdown next to Save results button to only show the Dumped ones
    
*   Once it's finished hit the Save results button

Advices

*   Don't just make random keywords, for example if you're targeting gaming, learn about what's currently trending in that niche, browse steam and see what games is trending, what dropped...
    
*   If you want to get the best results you'll need to get the proxyless plan to run the scraper on Google, otherwise you'll get way less results
    
*   When running the modules make sure to keep an eye on your CPU/RAM usages, if they're too high reduce the threads or it won't run properly, if it's low (under 50-60%) you can increase the threads and see how it goes
    
*   Running multiple tasks at the same time is possible but that means you'll need to adjust the threads accordingly
    
*   Resi proxies are good for the Scraper/Parser only, if you run the Vulnerability Scanner or Dumper you'll need ISP/DC proxies, they're faster and more stable

# Scraper/Parser | DorkPlus Documentation

### 

[](https://docs.dorkplus.com/pages#valid-or-filtered)

Valid or Filtered?

The valid links are all links scraped from the search engine(s), meaning they're not sorted and contains a lot of duplicates, links without parameters and other useless lines.

We always recommend people to download filtered links if they want to be effective, save time and get high injectable rates.

The filtered links are cleaned links without duplicates, bad links or empty parameters.

### 

[](https://docs.dorkplus.com/pages#page-count)

Page count

This feature works with all search engines except for Google since we already grab as much pages as possible.

### 

[](https://docs.dorkplus.com/pages#url-check)

URL check

This setting will ensure you don't get links you already scraped before, it's a local AntiPublic system basically.

### 

[](https://docs.dorkplus.com/pages#domain-check)

Domain check

The domain check is here to clean up the duplicates, if you get for example those 2 links:

Copy

    https://website.com/blog?id=123
    https://website.com/docs?id=456

It'll only keep one of these two links.

### 

[](https://docs.dorkplus.com/pages#remove-duplicates)

Remove duplicates

This setting is here to back up the domain check if it's disabled. It'll remove exact same links and filter them out.

### 

[](https://docs.dorkplus.com/pages#keep-unfiltered)

Keep unfiltered

This feature is only useful if you know what to do with links that are raw without parameters. Do not enable if you need links with parameters.
