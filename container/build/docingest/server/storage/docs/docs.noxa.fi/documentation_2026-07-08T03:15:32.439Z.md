# NOXA Fun Overview | NOXA

[Skip to content](https://docs.noxa.fi/launchpad/overview/#_top)

NOXA Fun Overview
=================

NOXA Fun is a revolutionary hybrid token launchpad at [fun.noxa.fi](https://fun.noxa.fi/)
 that combines the best of both worlds: instant V3 DEX trading with graduation mechanics.

Why NOXA Fun?
-------------

[Section titled “Why NOXA Fun?”](https://docs.noxa.fi/launchpad/overview/#why-noxa-fun)

Traditional bonding curve launchpads have issues:

*   Custom trading mechanisms with special token taxes
*   LP migration risk at graduation
*   Tokens aren’t on a real DEX until after graduation

NOXA Fun solves all of this:

*   **Instant DEX trading** - Tokens are tradeable on Uniswap V3 from block one, no LP migration needed
*   **Graduation mechanics** - We keep the gamification degens love
*   **No token taxes** - We use built-in Uniswap fee tiers instead of custom bonding curve taxes
*   **Best UX** - Trade directly on V3 DEX, no waiting

Key Features
------------

[Section titled “Key Features”](https://docs.noxa.fi/launchpad/overview/#key-features)

### Single-Sided V3 LP

[Section titled “Single-Sided V3 LP”](https://docs.noxa.fi/launchpad/overview/#single-sided-v3-lp)

When you create a token on NOXA Fun:

1.  A new ERC-20 token is deployed
2.  Single-sided liquidity is added to a V3 pool with the 1% fee tier
3.  The token is immediately tradeable on the DEX

### Locked Forever

[Section titled “Locked Forever”](https://docs.noxa.fi/launchpad/overview/#locked-forever)

The LP position is locked in our locker contract permanently. It never moves, never gets rugged, never migrates. Unlike bonding curve launchpads where liquidity migration creates risk, NOXA Fun LP stays locked forever.

### Graduation Mechanics

[Section titled “Graduation Mechanics”](https://docs.noxa.fi/launchpad/overview/#graduation-mechanics)

We preserve the gamification that degens love:

*   Tokens have “graduation” milestones based on net buy amount / liquidity in the pool
*   Thresholds are configured per chain depending on chain hype and liquidity
*   **The LP never moves** - graduation is a milestone, not a migration

Best of both worlds: instant DEX trading AND graduation hype.

### Creator Fees

[Section titled “Creator Fees”](https://docs.noxa.fi/launchpad/overview/#creator-fees)

Token creators earn fees from trading activity:

*   Fees accrue from trades on the V3 pool
*   Creators can view accrued fees on the claim page
*   Connect your creator wallet to claim your earned fees
*   Since it’s a V3 pool, fees are paid in wrapped native tokens (e.g., WETH instead of ETH)
*   Easily unwrap your WETH to ETH on NOXA DEX or any other supported protocol

### Multi-DEX Support

[Section titled “Multi-DEX Support”](https://docs.noxa.fi/launchpad/overview/#multi-dex-support)

Launch your token on the DEX of your choice:

*   NOXA DEX
*   Uniswap
*   PancakeSwap
*   Other compatible V3 forks

See the [Supported Chains](https://docs.noxa.fi/launchpad/chains/)
 page for available DEXes per chain.

### Built-in Charts

[Section titled “Built-in Charts”](https://docs.noxa.fi/launchpad/overview/#built-in-charts)

On new chains, DexScreener and other chart providers aren’t available until after official chain launch. NOXA Fun includes built-in charts so you can track price action from day one.

### Token Verification

[Section titled “Token Verification”](https://docs.noxa.fi/launchpad/overview/#token-verification)

Check if other tokens with the same name or symbol already exist. Helps you avoid confusion and identify potential copycats.

### Portfolios & Profiles

[Section titled “Portfolios & Profiles”](https://docs.noxa.fi/launchpad/overview/#portfolios--profiles)

*   View your connected wallet’s portfolio with total value
*   Check other traders’ profiles to see their portfolio value
*   See what tokens they created or bought
*   View their recent trades

### Trending Tokens

[Section titled “Trending Tokens”](https://docs.noxa.fi/launchpad/overview/#trending-tokens)

Discover the hottest tokens with our trending page. Rankings are based on custom logic that factors in:

*   Recent activity and transactions
*   Trading volume
*   Launch date
*   Other criteria

For Developers & Bots
---------------------

[Section titled “For Developers & Bots”](https://docs.noxa.fi/launchpad/overview/#for-developers--bots)

Since tokens launch directly on V3, trading bots, apps, and tools can easily integrate with NOXA Fun launches. No need to build custom integrations for bonding curve mechanics - just use standard Uniswap V3 interfaces.

For Partner DEXes
-----------------

[Section titled “For Partner DEXes”](https://docs.noxa.fi/launchpad/overview/#for-partner-dexes)

When a token launches targeting your DEX, 100% of the trading volume happens on your DEX from day one. Unlike bonding curve launchpads where volume starts on the curve and only moves to your DEX after graduation.

2.  Connect your wallet

How to Launch
=============

Ready to launch your token? Here’s how to do it on NOXA Fun.

Getting Started
---------------

[Section titled “Getting Started”](https://docs.noxa.fi/launchpad/how-to-launch/#getting-started)

1.  Go to the launch page for your chain (e.g., [fun.noxa.fi/megaeth/launch](https://fun.noxa.fi/megaeth/launch)
    )
2.  Connect your wallet
3.  Fill in your token details

You can access the launch page from the NOXA Fun menu on any chain.

Token Details
-------------

[Section titled “Token Details”](https://docs.noxa.fi/launchpad/how-to-launch/#token-details)

### Logo Image (Required)

[Section titled “Logo Image (Required)”](https://docs.noxa.fi/launchpad/how-to-launch/#logo-image-required)

*   Formats: PNG, JPEG, WebP, or GIF
*   Max size: 4.5MB
*   Must be square (1:1 ratio)
*   Minimum: 250x250 pixels
*   Will be resized to 250x250

### Token Name (Required)

[Section titled “Token Name (Required)”](https://docs.noxa.fi/launchpad/how-to-launch/#token-name-required)

*   Up to 60 characters
*   Choose something memorable

### Symbol (Required)

[Section titled “Symbol (Required)”](https://docs.noxa.fi/launchpad/how-to-launch/#symbol-required)

*   Up to 20 characters
*   Usually 3-5 characters (e.g., PEPE, DOGE)

### Description (Optional)

[Section titled “Description (Optional)”](https://docs.noxa.fi/launchpad/how-to-launch/#description-optional)

*   Up to 256 characters
*   Describe what your token is about

### Social Links (Optional)

[Section titled “Social Links (Optional)”](https://docs.noxa.fi/launchpad/how-to-launch/#social-links-optional)

Add your community links:

*   Twitter / X
*   Telegram
*   Discord
*   Website
*   Farcaster

You can use full URLs, usernames, or handles (e.g., @username). Invalid links will be filtered out.

Advanced Settings
-----------------

[Section titled “Advanced Settings”](https://docs.noxa.fi/launchpad/how-to-launch/#advanced-settings)

### Custom Fee Wallet

[Section titled “Custom Fee Wallet”](https://docs.noxa.fi/launchpad/how-to-launch/#custom-fee-wallet)

By default, trading fees go to your connected wallet. You can set a different address to receive:

*   Trading fees from the V3 pool
*   Any tokens from your initial buy

### Initial Buy

[Section titled “Initial Buy”](https://docs.noxa.fi/launchpad/how-to-launch/#initial-buy)

Optionally buy some of your own token at launch. The amount you specify (in the native token, e.g., ETH) will be used to purchase your token immediately after creation.

Launch
------

[Section titled “Launch”](https://docs.noxa.fi/launchpad/how-to-launch/#launch)

Once you’ve filled everything in, click launch and confirm the transaction in your wallet. Your token will be live and tradeable immediately on the selected DEX.

Gas & Fees
----------

[Section titled “Gas & Fees”](https://docs.noxa.fi/dex/trading/#gas--fees)

*   **Swap fee:** Depends on the pool (0.05%, 0.3%, or 1% for V3)

Tips for Degen Trading
----------------------

[Section titled “Tips for Degen Trading”](https://docs.noxa.fi/dex/trading/#tips-for-degen-trading)

1.  **Check liquidity** before large trades - new tokens may have limited depth
2.  **Use reasonable slippage** - too low and your tx fails, too high and you get front-run
3.  **Verify contracts** - always double-check token addresses
4.  **DYOR** - new tokens are risky, only trade what you can afford to lose

# Providing Liquidity | NOXA

[Skip to content](https://docs.noxa.fi/dex/liquidity/#_top)

Providing Liquidity
===================

NOXA DEX supports both V2 and V3 liquidity provision.

V2 Liquidity
------------

[Section titled “V2 Liquidity”](https://docs.noxa.fi/dex/liquidity/#v2-liquidity)

Classic AMM liquidity pools.

### Adding V2 Liquidity

[Section titled “Adding V2 Liquidity”](https://docs.noxa.fi/dex/liquidity/#adding-v2-liquidity)

1.  Go to [dex.noxa.fi](https://dex.noxa.fi/)
     and navigate to Pool
2.  Click “Add Liquidity”
3.  Select V2 pool type
4.  Choose your token pair
5.  Enter amounts (must be balanced by value)
6.  Approve tokens if needed
7.  Add liquidity

You’ll receive LP tokens representing your share of the pool.

### Removing V2 Liquidity

[Section titled “Removing V2 Liquidity”](https://docs.noxa.fi/dex/liquidity/#removing-v2-liquidity)

1.  Go to the Pool section
2.  Find your position
3.  Click “Remove”
4.  Choose the amount to remove (25%, 50%, 100%, or custom)
5.  Confirm the transaction

You’ll receive both tokens back proportional to your share.

V3 Liquidity
------------

[Section titled “V3 Liquidity”](https://docs.noxa.fi/dex/liquidity/#v3-liquidity)

Concentrated liquidity with range orders.

### Adding V3 Liquidity

[Section titled “Adding V3 Liquidity”](https://docs.noxa.fi/dex/liquidity/#adding-v3-liquidity)

1.  Navigate to Pool and select V3
2.  Choose your token pair
3.  Select fee tier (0.05%, 0.3%, or 1%)
4.  Set your price range
5.  Enter the amount of tokens
6.  Approve and add liquidity

You’ll receive an LP NFT representing your position.

### Choosing a Price Range

[Section titled “Choosing a Price Range”](https://docs.noxa.fi/dex/liquidity/#choosing-a-price-range)

*   **Wide range:** More like V2, earns fees across all prices, lower capital efficiency
*   **Narrow range:** Higher capital efficiency, but goes inactive if price moves outside
*   **Full range:** Equivalent to V2 behavior

For new/volatile tokens, wider ranges are safer.

### Fee Tiers

[Section titled “Fee Tiers”](https://docs.noxa.fi/dex/liquidity/#fee-tiers)

| Fee Tier | Best For |
| --- | --- |
| 0.05% | Stable pairs, high volume |
| 0.3% | Standard pairs |
| 1%  | Volatile/exotic pairs, new tokens |

NOXA Fun tokens use the **1% fee tier** by default.

### Managing V3 Positions

[Section titled “Managing V3 Positions”](https://docs.noxa.fi/dex/liquidity/#managing-v3-positions)

*   **Collect fees:** Earned fees can be collected at any time
*   **Adjust range:** Close position and open a new one with different range
*   **Increase liquidity:** Add more to an existing position
*   **Remove liquidity:** Partial or full removal

NOXA Fun LP
-----------

[Section titled “NOXA Fun LP”](https://docs.noxa.fi/dex/liquidity/#noxa-fun-lp)

For tokens launched on NOXA Fun, the initial LP is:

*   Created automatically at launch
*   Locked forever in the locker contract
*   Cannot be removed by anyone (including the token creator)

This locked LP provides permanent baseline liquidity. Additional LP can still be added by anyone.

Risks
-----

[Section titled “Risks”](https://docs.noxa.fi/dex/liquidity/#risks)

*   **Impermanent loss:** Your position value may decrease relative to holding
*   **Smart contract risk:** Using DeFi protocols always carries risk
*   **Token risk:** Providing liquidity to low-quality tokens can result in losses

Only provide liquidity you can afford to lose.

# Building on NOXA Fun | NOXA

[Skip to content](https://docs.noxa.fi/integrations/launchpad/#_top)

Building on NOXA Fun
====================

This page is for developers building apps, bots, and analytics on top of NOXA Fun launches.

Everything you need is **on-chain**. You can read launches, pools, and restrictions directly from the contracts and event logs using any standard RPC.

Tokens Are Standard and Instantly Tradable
------------------------------------------

[Section titled “Tokens Are Standard and Instantly Tradable”](https://docs.noxa.fi/integrations/launchpad/#tokens-are-standard-and-instantly-tradable)

At launch we deploy a standard ERC-20 and add single-sided liquidity to a Uniswap V3-style pool, so the token is live in a real V3 pool from the launch transaction itself.

*   A token launched targeting **Uniswap V3** is tradable on Uniswap V3. Use Uniswap’s normal SwapRouter / QuoterV2.
*   A token launched targeting **NOXA DEX** or any other supported V3 fork trades through that DEX’s standard contracts.
*   There is no migration step and no custom AMM. If you already integrate Uniswap V3, you already integrate NOXA Fun tokens.

Public swaps execute from the block **after** the launch block. The launch block itself is reserved to prevent same-block sniping (see [Trading Restrictions](https://docs.noxa.fi/integrations/launchpad/#trading-restrictions)
).

Discovering Launches
--------------------

[Section titled “Discovering Launches”](https://docs.noxa.fi/integrations/launchpad/#discovering-launches)

The `LauncherFactory` emits `TokenLaunched` for every launch. Filter this event from the factory address for your chain (see [NOXA Fun Contracts](https://docs.noxa.fi/contracts/noxa-fun/)
) to discover launches in real time, or backfill historically by querying logs from the factory’s deployment block.

    event TokenLaunched(    address indexed token,    address indexed deployer,    address indexed dexFactory,    address pairToken,    address pool,    uint256 dexId,    uint256 launchConfigId,    uint256 positionId,    uint256 restrictionsEndBlock,    uint256 initialBuyAmount);

`token` is the new ERC-20, `pool` is the V3 pool to trade against, `pairToken` is the quote token (the chain’s wrapped native, or USDC on Arc), and `dexFactory` identifies which DEX the token launched on.

Verifying a Launch Is Genuine
-----------------------------

[Section titled “Verifying a Launch Is Genuine”](https://docs.noxa.fi/integrations/launchpad/#verifying-a-launch-is-genuine)

Anyone can deploy a lookalike token. To confirm a token actually came from NOXA Fun, query the factory:

    function getLaunchedToken(address token) external view returns (LaunchedToken memory);
    struct LaunchedToken {    address token;    address deployer;    address pairedToken;    address positionManager;    uint256 positionId;    uint256 dexId;    uint256 launchConfigId;    uint256 restrictionsEndBlock;    uint256 supply;    bool    isToken0;    uint24  poolFee;    bool    exists;    uint256 initialBuyAmount;}

Check `exists == true`. The struct also gives you everything you need to locate the pool and position without any extra calls.

Alternatively, read `launchFactory()` on the token and confirm it equals the known factory address for that chain.

Finding the Pool
----------------

[Section titled “Finding the Pool”](https://docs.noxa.fi/integrations/launchpad/#finding-the-pool)

The token exposes its own pool and pairing directly:

    function liquidityPool() external view returns (address); // the V3 poolfunction pairToken()     external view returns (address); // quote token (e.g. WETH, USDC)function poolFee()       external view returns (uint24);  // fee tier, currently 10000 (1%)

`liquidityPool()` resolves the pool from the DEX factory for the `(token, pairToken, poolFee)` triple, so it returns the canonical V3 pool address you swap against.

Trading Restrictions
--------------------

[Section titled “Trading Restrictions”](https://docs.noxa.fi/integrations/launchpad/#trading-restrictions)

Launched tokens enforce anti-snipe restrictions for a short window after launch. Your integration **must** respect these or transfers will revert. Restrictions are active while:

    launchBlock < block.number <= restrictionEndBlock

Read the limits and window from the token:

    function maxWalletLimit()     external view returns (uint256); // max balance per wallet (currently 2% of supply)function maxTxLimit()         external view returns (uint256); // per-tx cap (configured per chain)function restrictionEndBlock() external view returns (uint256); // restrictions lift after this block

Rules while restrictions are active:

*   **Launch block:** buys directly from the pool revert with `LaunchBlockBuyBlocked(address recipient)`. Public trading starts the next block.
*   **Max wallet:** a recipient’s resulting balance cannot exceed `maxWalletLimit()`, or the transfer reverts with `MaxWalletExceeded(address account, uint256 balance, uint256 limit)`.
*   **Max tx:** cumulative buys from the pool per `tx.origin` are capped at 110% of `maxTxLimit()`, or the transfer reverts with `MaxTxExceeded(address originator, uint256 attempted, uint256 limit)`.

After `restrictionEndBlock`, the token behaves as a fully standard ERC-20 with no limits.

Launch restrictions usually operate for ~1 hour after launch.

Minimal Interfaces
------------------

[Section titled “Minimal Interfaces”](https://docs.noxa.fi/integrations/launchpad/#minimal-interfaces)

Drop-in interfaces covering the read paths above:

    interface ILauncherFactory {    function getLaunchedToken(address token) external view returns (LaunchedToken memory);}
    interface ILauncherToken {    function launchFactory() external view returns (address);    function liquidityPool() external view returns (address);    function pairToken() external view returns (address);    function poolFee() external view returns (uint24);    function maxWalletLimit() external view returns (uint256);    function maxTxLimit() external view returns (uint256);    function restrictionEndBlock() external view returns (uint256);}

Contract addresses per chain are on the [NOXA Fun Contracts](https://docs.noxa.fi/contracts/noxa-fun/)
 page, and chain IDs are on [Supported Chains](https://docs.noxa.fi/launchpad/chains/)
.

# DEX Integration | NOXA

[Skip to content](https://docs.noxa.fi/integrations/dex/#_top)

DEX Integration
===============

NOXA Fun supports multiple DEXes per chain. If you run a DEX and want to be added as a launch target, this page explains the requirements and process.

Requirements
------------

[Section titled “Requirements”](https://docs.noxa.fi/integrations/dex/#requirements)

Your DEX must be:

1.  **Uniswap V3 fork** (unmodified contracts), OR
2.  **PancakeSwap V3 fork** (unmodified contracts)

We do not support:

*   V2-only DEXes (no concentrated liquidity)
*   Custom AMM implementations
*   DEXes with modified core contracts

### Why Unmodified?

[Section titled “Why Unmodified?”](https://docs.noxa.fi/integrations/dex/#why-unmodified)

NOXA Fun creates LP positions programmatically. We need to know exactly how the contracts behave. Modified forks introduce unknown behavior that could cause issues.

Unmodified means:

*   Same function signatures as Uniswap/PancakeSwap V3
*   Same events
*   Same pool creation process
*   Same NFT position manager

What We Need From You
---------------------

[Section titled “What We Need From You”](https://docs.noxa.fi/integrations/dex/#what-we-need-from-you)

To add your DEX to NOXA Fun:

### 1\. Contract Addresses
