Fair launch a token for your agent. Trading fees flow back to fund compute. Self-sustaining from day one.

Community-built skills for trading, social, and DeFi. One-click install. Your agent gets smarter over time.

# Token Launching Overview | Bankr Documentation

[Skip to main content](https://docs.bankr.bot/token-launching/overview/#__docusaurus_skipToContent_fallback)

On this page

Launch a token for your AI agent and earn trading fees automatically. This is how agents fund themselves.

Why Launch a Token?[​](https://docs.bankr.bot/token-launching/overview/#why-launch-a-token "Direct link to Why Launch a Token?")

---------------------------------------------------------------------------------------------------------------------------------

When you launch a token through Bankr:

1.  **Liquidity pool is created** — Your token is immediately tradeable
2.  **Trading fees accumulate** — Every trade generates fees
3.  **Fees flow to you** — Claim your earnings anytime
4.  **Fund your compute** — Use fees to pay for your agent's API costs

Supported Networks[​](https://docs.bankr.bot/token-launching/overview/#supported-networks "Direct link to Supported Networks")

| Solana | Yes | Yes | Yes |

-------------------------------------------------------------------------------------------------------------------------------------------------------------------

    "deploy a token called MyAgent with symbol AGENT on base""launch a token called CoolBot on solana""deploy a token with 20% vaulted for 30 days on base"

Launching via Social (Built-in Virality)[​](https://docs.bankr.bot/token-launching/overview/#launching-via-social-built-in-virality "Direct link to Launching via Social (Built-in Virality)")

-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

    @bankrbot deploy a token called ViralAgent on solana

This creates instant social proof and discoverability for your token.

Deployment Limits[​](https://docs.bankr.bot/token-launching/overview/#deployment-limits "Direct link to Deployment Limits")

----------------------------------------------------------------------------------------------------------------------------

| User Type | Tokens Per Day |
| --- | --- |
| Standard | 1   |
| Bankr Club | 10  |

Gas is sponsored within these limits. Additional launches require paying gas yourself (~0.02 SOL on Solana).

Fee Structure[​](https://docs.bankr.bot/token-launching/overview/#fee-structure "Direct link to Fee Structure")

----------------------------------------------------------------------------------------------------------------

### Base[​](https://docs.bankr.bot/token-launching/overview/#base "Direct link to Base")

Trading fees are split:

| Recipient | Share |
| --- | --- |
| Deployer | 60% |
| Bankr | 40% |

Fees accumulate in your token and WETH from the 1% fee liquidity pool.

### Solana[​](https://docs.bankr.bot/token-launching/overview/#solana "Direct link to Solana")

Fees are structured differently across the token lifecycle:

**During Bonding Curve:**

*   1% platform fee (to Bankr)
*   0.5% creator fee (to your wallet)

**After Migration to CPMM Pool:**

*   50% of LP tokens locked to you (creator)
*   40% of LP tokens locked to Bankr
*   10% of LP tokens burned

You earn ongoing trading fees from your locked LP position.

Token Supply[​](https://docs.bankr.bot/token-launching/overview/#token-supply "Direct link to Token Supply")

-------------------------------------------------------------------------------------------------------------

*   **Base:** Fixed supply of 100 billion tokens (not mintable after deployment)
*   **Solana:** Configurable supply with default 6 decimals

# Platform Overview | Bankr Documentation

[Skip to main content](https://docs.bankr.bot/getting-started/overview/#__docusaurus_skipToContent_fallback)

On this page

Bankr is a Web3 AI companion platform that integrates LLM technology with DeFi protocols. It enables AI agents to execute blockchain transactions through natural language commands while Bankr handles security, routing, and execution.

Core Capabilities[​](https://docs.bankr.bot/getting-started/overview/#core-capabilities "Direct link to Core Capabilities")

----------------------------------------------------------------------------------------------------------------------------

### Natural Language Trading[​](https://docs.bankr.bot/getting-started/overview/#natural-language-trading "Direct link to Natural Language Trading")

Your agent sends plain English commands. Bankr interprets them and executes the appropriate blockchain transactions.

    "swap $50 of ETH to USDC on base""buy $10 of BONK on solana""set a limit order to buy BNKR if it drops 10%"

### Multi-Chain Support[​](https://docs.bankr.bot/getting-started/overview/#multi-chain-support "Direct link to Multi-Chain Support")

Bankr operates across five blockchain networks:

*   **Base** — Primary EVM chain with gas sponsorship
*   **Ethereum** — ETH mainnet for high-value operations
*   **Polygon** — Low-cost EVM transactions
*   **Unichain** — Uniswap's native chain
*   **Solana** — High-speed SVM transactions

### Security First[​](https://docs.bankr.bot/getting-started/overview/#security-first "Direct link to Security First")

Every transaction passes through Bankr's Sentinel security system, which checks for:

*   Malicious contract addresses
*   Phishing attempts
*   Unusual transaction patterns
*   Prompt injection attacks

### Token Launching[​](https://docs.bankr.bot/getting-started/overview/#token-launching "Direct link to Token Launching")

Deploy tokens on Base or Solana with built-in:

*   Liquidity pool creation
*   Fee sharing arrangements
*   Vaulting and vesting options

Base[​](https://docs.bankr.bot/getting-started/supported-chains/#base "Direct link to Base")

---------------------------------------------------------------------------------------------

Base is the primary chain for Bankr with the most features enabled.

**Capabilities:**

*   Token swaps via Uniswap V2/V3/V4 and Aerodrome
*   Token launching with fee sharing
*   Limit orders, stop orders, DCA, TWAP
*   NFT buying, selling, and minting
*   Leveraged trading via Avantis
*   Gas sponsorship for transactions

**Block Explorer:** [Basescan](https://basescan.org/)

Ethereum[​](https://docs.bankr.bot/getting-started/supported-chains/#ethereum "Direct link to Ethereum")

---------------------------------------------------------------------------------------------------------

Ethereum mainnet for high-value operations.

**Capabilities:**

*   Token swaps via Uniswap V2/V3/V4
*   NFT buying, selling, and minting

**Note:** Gas is not sponsored on Ethereum mainnet due to high costs.

**Block Explorer:** [Etherscan](https://etherscan.io/)

Polygon[​](https://docs.bankr.bot/getting-started/supported-chains/#polygon "Direct link to Polygon")

------------------------------------------------------------------------------------------------------

Low-cost EVM chain with Polymarket support.

**Capabilities:**

*   Token swaps via Uniswap V2/V3/V4
*   Polymarket betting
*   NFT buying, selling, and minting
*   Gas sponsorship for transactions

**Block Explorer:** [Polygonscan](https://polygonscan.com/)

Unichain[​](https://docs.bankr.bot/getting-started/supported-chains/#unichain "Direct link to Unichain")

---------------------------------------------------------------------------------------------------------

Uniswap's native L2 chain.

**Capabilities:**

*   Token swaps via Uniswap V2/V3/V4
*   NFT buying, selling, and minting
*   Gas sponsorship for transactions

**Block Explorer:** [Uniscan](https://uniscan.xyz/)

Solana[​](https://docs.bankr.bot/getting-started/supported-chains/#solana "Direct link to Solana")

---------------------------------------------------------------------------------------------------

High-speed SVM chain for fast trading.

**Capabilities:**

*   Token swaps via Jupiter aggregator
*   Token launching via Raydium LaunchLab
*   Cross-chain swaps

**Note:** Gas sponsorship is limited on Solana (1/day standard, 10/day Bankr Club for token launches).

**Block Explorer:** [Orb Markets](https://www.orbmarkets.io/)

Specifying Chains[​](https://docs.bankr.bot/getting-started/supported-chains/#specifying-chains "Direct link to Specifying Chains")

------------------------------------------------------------------------------------------------------------------------------------

When sending prompts, you can specify the chain:

    "swap $10 of USDC to ETH on base""buy $5 of BONK on solana""buy $20 of MATIC on polygon"

If no chain is specified, Bankr will:

1.  Use the chain where your balance is highest
2.  Default to Base for EVM operations
3.  Default to Solana for SOL-related operations

Cross-Chain Operations[​](https://docs.bankr.bot/getting-started/supported-chains/#cross-chain-operations "Direct link to Cross-Chain Operations")

---------------------------------------------------------------------------------------------------------------------------------------------------

Bankr supports cross-chain swaps between EVM chains using bridge aggregators:

    "swap $50 USDC from polygon to ETH on base"

Cross-chain operations between EVM and Solana are not currently supported.

# Fee Splitting | Bankr Documentation

[Skip to main content](https://docs.bankr.bot/token-launching/fee-splitting/#__docusaurus_skipToContent_fallback)

On this page

Route trading fees to collaborators, treasury wallets, or partners.

How Fee Splitting Works[​](https://docs.bankr.bot/token-launching/fee-splitting/#how-fee-splitting-works "Direct link to How Fee Splitting Works")

---------------------------------------------------------------------------------------------------------------------------------------------------

When your token is traded, fees are generated. By default, your 60% share (Base) or creator fees (Solana) go to your deployer wallet. With fee splitting, you can route your share to a different address.

Base Fee Splitting[​](https://docs.bankr.bot/token-launching/fee-splitting/#base-fee-splitting "Direct link to Base Fee Splitting")

------------------------------------------------------------------------------------------------------------------------------------

On Base, specify a different reward owner at deployment:

    "deploy TeamToken with fees going to 0x1234... on base"

This routes your 60% share of trading fees to the specified address. The 40% Bankr platform fee is separate.

### Fee Recipient vs Vault Recipient[​](https://docs.bankr.bot/token-launching/fee-splitting/#fee-recipient-vs-vault-recipient "Direct link to Fee Recipient vs Vault Recipient")

You can set different addresses for:

| Recipient | Receives | Changeable? |
| --- | --- | --- |
| Fee recipient | Trading fees (60% share) | Yes (can be transferred) |
| Vault recipient | Vested tokens | No (immutable) |

    "deploy a token with fees to 0xABC... and vault to 0xDEF... on base"

### Transferring Fee Ownership[​](https://docs.bankr.bot/token-launching/fee-splitting/#transferring-fee-ownership "Direct link to Transferring Fee Ownership")

After deployment, the fee recipient can transfer their rights:

    "transfer fee ownership for 0x1234... to 0x5678..."

**Warning:** This is irreversible. Both the recipient address and admin rights transfer together.

Solana Fee Splitting[​](https://docs.bankr.bot/token-launching/fee-splitting/#solana-fee-splitting "Direct link to Solana Fee Splitting")

------------------------------------------------------------------------------------------------------------------------------------------

On Solana, specify a fee recipient at launch:

    "launch a token on solana with fees going to ABC123..."

The fee recipient arrangement is **permanent** once created:

**During Bonding Curve:**

*   Receives 0.5% of all trades (the "share fee")
*   Both creator and fee recipient can claim accumulated fees
*   Fees are split: 99.9% to recipient, 0.1% to creator as referrer fee

**After Migration:**

*   Fee recipient can claim the Fee Key NFT
*   Say "claim my fees for TokenName" to receive the NFT
*   The NFT represents rights to ongoing LP trading fees

warning

Once a fee recipient is specified at launch on Solana, the deployer **cannot** transfer the Fee Key NFT to anyone else. It can only go to the designated fee recipient.

# Vaulting and Vesting | Bankr Documentation

[Skip to main content](https://docs.bankr.bot/token-launching/vaulting-vesting/#__docusaurus_skipToContent_fallback)

On this page

Lock a portion of your token supply with time-based release schedules.

What is Vaulting?[​](https://docs.bankr.bot/token-launching/vaulting-vesting/#what-is-vaulting "Direct link to What is Vaulting?")

-----------------------------------------------------------------------------------------------------------------------------------

Vaulting locks a percentage of your token supply. This:

*   Signals long-term commitment to holders
*   Prevents immediate dumps
*   Builds trust with your community

What is Vesting?[​](https://docs.bankr.bot/token-launching/vaulting-vesting/#what-is-vesting "Direct link to What is Vesting?")

--------------------------------------------------------------------------------------------------------------------------------

Vesting releases locked tokens gradually over time. You can configure:

*   **Cliff period** — Time before any tokens unlock
*   **Vesting duration** — Time over which tokens gradually release

Base Token Vaulting[​](https://docs.bankr.bot/token-launching/vaulting-vesting/#base-token-vaulting "Direct link to Base Token Vaulting")

------------------------------------------------------------------------------------------------------------------------------------------

On Base, you can vault 0-90% of the token supply:

    "deploy a token called MyAgent with 30% vaulted on base""deploy TeamToken with 20% vault, 7 day lockup, vesting over 30 days on base"

### Configuration Options[​](https://docs.bankr.bot/token-launching/vaulting-vesting/#configuration-options "Direct link to Configuration Options")

| Parameter | Description | Default |
| --- | --- | --- |
| Vault percentage | 0-90% of supply | 0% (no vault) |
| Lockup/cliff | Minimum 7 days | 7 days |
| Vesting duration | Days to fully vest | 0 (immediate after cliff) |
| Vault recipient | Who receives vested tokens | Deployer |

### Example Configurations[​](https://docs.bankr.bot/token-launching/vaulting-vesting/#example-configurations "Direct link to Example Configurations")

**Simple lockup (no vesting):**

    "deploy a token with 25% vaulted for 14 days on base"

After 14 days, all vaulted tokens unlock at once.

**Lockup with vesting:**

    "deploy a token with 30% vault, 7 day cliff, 60 day vesting on base"

After 7 days, tokens begin unlocking linearly over 60 days.

Solana Token Vesting[​](https://docs.bankr.bot/token-launching/vaulting-vesting/#solana-token-vesting "Direct link to Solana Token Vesting")

---------------------------------------------------------------------------------------------------------------------------------------------

On Solana, vesting uses the Raydium LaunchLab mechanism:

| Parameter | Description |
| --- | --- |
| `totalLockedAmount` | Number of tokens to lock (in token units with decimals) |
| `cliffPeriod` | Seconds before vesting starts (e.g., 30 days = 2,592,000 seconds) |
| `unlockPeriod` | Seconds over which tokens vest linearly |

    "launch a token on solana with 50% locked, 30 day cliff, 90 day vesting"

note

Solana vesting parameters are specified in **seconds**, not days. Bankr handles the conversion when you use natural language like "30 day cliff".

Important Notes[​](https://docs.bankr.bot/token-launching/vaulting-vesting/#important-notes "Direct link to Important Notes")

------------------------------------------------------------------------------------------------------------------------------

*   **Vault recipient is immutable** — On Base, you cannot change who receives vested tokens after deployment
*   **Minimum lockup** — Base requires at least 7 days
*   **No vault by default** — If you don't specify, no tokens are locked

-----------------------------------------------------------------------------------------------------------------------

    "claim my fees for TokenName""claim fees for 0x1234..."

Fee Types by Chain[​](https://docs.bankr.bot/token-launching/claiming-fees/#fee-types-by-chain "Direct link to Fee Types by Chain")

------------------------------------------------------------------------------------------------------------------------------------

### Base[​](https://docs.bankr.bot/token-launching/claiming-fees/#base "Direct link to Base")

When you claim on Base, you receive two assets:

| Asset | Description |
| --- | --- |
| Your token | From swap fees |
| WETH | From trading fees |

Both accumulate in the liquidity pool and are claimed together.

### Solana[​](https://docs.bankr.bot/token-launching/claiming-fees/#solana "Direct link to Solana")

Solana fees depend on the token's lifecycle stage:

**During Bonding Curve:**

*   Fees are paid in **SOL**
*   0.5% creator fee from each trade
*   Claim with: `"claim my fees for TokenName"`

**After Migration to CPMM:**

*   You hold a Fee Key NFT representing LP rights
*   Fees come from LP trading activity
*   Claim with: `"claim CPMM fees for TokenName"`

tip

If your token has migrated but you're trying to claim bonding curve fees, Bankr will guide you to use the correct claim method.

Claim Frequency[​](https://docs.bankr.bot/token-launching/claiming-fees/#claim-frequency "Direct link to Claim Frequency")

---------------------------------------------------------------------------------------------------------------------------

You can claim at any time. There's no minimum threshold, but consider gas costs when claiming small amounts.

**Recommended approach:**

*   Check fees periodically: `"show my unclaimed fees"`
*   Claim when accumulated amount justifies gas cost
*   On Base, gas is low so claim more frequently
*   On Solana, claim transactions cost ~0.005 SOL

Viewing All Your Tokens[​](https://docs.bankr.bot/token-launching/claiming-fees/#viewing-all-your-tokens "Direct link to Viewing All Your Tokens")

---------------------------------------------------------------------------------------------------------------------------------------------------

See all tokens you've deployed and their fee status:

    "show all my deployed tokens""list my tokens with fees""show my Fee Key NFTs"  # Solana only

Troubleshooting[​](https://docs.bankr.bot/token-launching/claiming-fees/#troubleshooting "Direct link to Troubleshooting")

---------------------------------------------------------------------------------------------------------------------------

**"No fees to claim"**

*   Your token may not have had recent trading activity
*   Fees might have already been claimed
*   On Solana, check if the token has migrated (use different claim method)

**"Not authorized to claim"**

*   You must be the fee recipient for the token
*   Check if fee ownership was transferred

**"Has a fee sharing arrangement"** (Solana)

*   Use the shared claim flow which splits fees automatically
*   Both creator and fee recipient can initiate claims

**Transaction fails**

*   Ensure you have enough gas for the claim transaction
*   On Solana, ensure your wallet has some SOL for fees (~0.005 SOL)

# Prompt Endpoint | Bankr Documentation

[Skip to main content](https://docs.bankr.bot/agent-api/prompt-endpoint/#__docusaurus_skipToContent_fallback)

On this page

Submit natural language commands to the Bankr AI agent for processing.

Endpoint[​](https://docs.bankr.bot/agent-api/prompt-endpoint/#endpoint "Direct link to Endpoint")

--------------------------------------------------------------------------------------------------

    POST /agent/prompt

Request[​](https://docs.bankr.bot/agent-api/prompt-endpoint/#request "Direct link to Request")

-----------------------------------------------------------------------------------------------

### Headers[​](https://docs.bankr.bot/agent-api/prompt-endpoint/#headers "Direct link to Headers")

| Header | Value | Required |
| --- | --- | --- |
| `Content-Type` | `application/json` | Yes |
| `X-API-Key` | Your API key | Yes |

### Body[​](https://docs.bankr.bot/agent-api/prompt-endpoint/#body "Direct link to Body")

    {  "prompt": "swap $10 of ETH to USDC on base"}

| Field | Type | Description | Required |
| --- | --- | --- | --- |
| `prompt` | string | Natural language command (max 10,000 characters) | Yes |
| `threadId` | string | Continue an existing conversation thread | No  |

When `threadId` is provided, the agent loads prior messages from that thread so it has conversation context. If omitted, a new thread is created automatically.

Response[​](https://docs.bankr.bot/agent-api/prompt-endpoint/#response "Direct link to Response")

--------------------------------------------------------------------------------------------------

### Success (202 Accepted)[​](https://docs.bankr.bot/agent-api/prompt-endpoint/#success-202-accepted "Direct link to Success (202 Accepted)")

    {  "success": true,  "jobId": "abc123def456",  "threadId": "thr_XYZ789",  "status": "pending",  "message": "Job created successfully"}

| Field | Type | Description |
| --- | --- | --- |
| `success` | boolean | Whether the request was successful |
| `jobId` | string | Unique identifier for tracking this job |
| `threadId` | string | Conversation thread ID (reuse to continue the conversation) |
| `status` | string | Current status (`"pending"`) |
| `message` | string | Human-readable message |

### Error Responses[​](https://docs.bankr.bot/agent-api/prompt-endpoint/#error-responses "Direct link to Error Responses")

**Invalid Request (400)**

    {  "error": "Invalid request",  "message": "Request body must include a prompt"}

**Prompt Too Long (400)**

    {  "error": "Prompt too long",  "message": "Prompt must be 10,000 characters or less"}

**Authentication Required (401)**

    {  "error": "Authentication required",  "message": "Please provide a valid API key"}

**Agent Access Not Enabled (403)**

    {  "error": "Agent API access not enabled",  "message": "Enable agent access for your API key at bankr.bot/api"}

-----------------------------------------------------------------------------

### Login[​](https://docs.bankr.bot/agent-api/cli/#login "Direct link to Login")

    bankr login

Opens [bankr.bot/api](https://bankr.bot/api)
 in your browser so you can copy your API key, then prompts you to paste it. The key is stored in `~/.bankr/config.json`.

#### Non-interactive flags[​](https://docs.bankr.bot/agent-api/cli/#non-interactive-flags "Direct link to Non-interactive flags")

For AI agents or scripts that can't use interactive prompts:

    export BANKR_API_KEY=your_api_key_hereexport BANKR_LLM_KEY=your_llm_key_here  # optional, falls back to BANKR_API_KEY

# Submit Endpoint | Bankr Documentation

[Skip to main content](https://docs.bankr.bot/agent-api/submit-endpoint/#__docusaurus_skipToContent_fallback)

On this page

Submit transactions directly to the blockchain with optional confirmation waiting.

Endpoint[​](https://docs.bankr.bot/agent-api/submit-endpoint/#endpoint "Direct link to Endpoint")

--------------------------------------------------------------------------------------------------

    POST /agent/submit

Overview[​](https://docs.bankr.bot/agent-api/submit-endpoint/#overview "Direct link to Overview")

--------------------------------------------------------------------------------------------------

The submit endpoint allows you to submit raw EVM transactions directly to the blockchain. Unlike the [Prompt Endpoint](https://docs.bankr.bot/agent-api/prompt-endpoint)
 which uses natural language, this endpoint accepts explicit transaction parameters.

Key features:

*   **Direct transaction submission** — Bypass the AI agent for pre-built transactions
*   **Synchronous response** — Returns transaction hash immediately
*   **Optional confirmation** — Wait for on-chain confirmation or return immediately
*   **Full control** — Specify exact gas, nonce, and calldata

Request[​](https://docs.bankr.bot/agent-api/submit-endpoint/#request "Direct link to Request")

-----------------------------------------------------------------------------------------------

### Headers[​](https://docs.bankr.bot/agent-api/submit-endpoint/#headers "Direct link to Headers")

| Header | Value | Required |
| --- | --- | --- |
| `Content-Type` | `application/json` | Yes |
| `X-API-Key` | Your API key | Yes |

### Body[​](https://docs.bankr.bot/agent-api/submit-endpoint/#body "Direct link to Body")

    {  "transaction": {    "to": "0x...",    "chainId": 8453,    "value": "1000000000000000000",    "data": "0x..."  },  "description": "Transfer 1 ETH",  "waitForConfirmation": true}

### Transaction Object[​](https://docs.bankr.bot/agent-api/submit-endpoint/#transaction-object "Direct link to Transaction Object")

| Field | Type | Description | Required |
| --- | --- | --- | --- |
| `to` | string | Destination address | Yes |
| `chainId` | number | Chain ID (8453 for Base, 1 for Ethereum, etc.) | Yes |
| `value` | string | Value in wei (as string) | No  |
| `data` | string | Calldata (hex string starting with `0x`) | No  |
| `gas` | string | Gas limit | No  |
| `gasPrice` | string | Legacy gas price in wei | No  |
| `maxFeePerGas` | string | EIP-1559 max fee per gas | No  |
| `maxPriorityFeePerGas` | string | EIP-1559 priority fee | No  |
| `nonce` | number | Transaction nonce (auto-filled if omitted) | No  |

### Additional Fields[​](https://docs.bankr.bot/agent-api/submit-endpoint/#additional-fields "Direct link to Additional Fields")

| Field | Type | Description | Default |
| --- | --- | --- | --- |
| `description` | string | Human-readable description for logging | \-  |
| `waitForConfirmation` | boolean | Wait for transaction confirmation | `true` |

Response[​](https://docs.bankr.bot/agent-api/submit-endpoint/#response "Direct link to Response")

--------------------------------------------------------------------------------------------------

### Success with Confirmation (200 OK)[​](https://docs.bankr.bot/agent-api/submit-endpoint/#success-with-confirmation-200-ok "Direct link to Success with Confirmation (200 OK)")

When `waitForConfirmation: true` (default):

    {  "success": true,  "transactionHash": "0x...",  "status": "success",  "blockNumber": "12345678",  "gasUsed": "21000",  "signer": "0x...",  "chainId": 8453}

### Success without Confirmation (200 OK)[​](https://docs.bankr.bot/agent-api/submit-endpoint/#success-without-confirmation-200-ok "Direct link to Success without Confirmation (200 OK)")

When `waitForConfirmation: false`:

    {  "success": true,  "transactionHash": "0x...",  "status": "pending",  "signer": "0x...",  "chainId": 8453}

### Response Fields[​](https://docs.bankr.bot/agent-api/submit-endpoint/#response-fields "Direct link to Response Fields")

| Field | Type | Description |
| --- | --- | --- |
| `success` | boolean | `true` if submission succeeded |
| `transactionHash` | string | The transaction hash |
| `status` | string | `"success"`, `"reverted"`, or `"pending"` |
| `blockNumber` | string | Block number (if confirmed) |
| `gasUsed` | string | Gas used (if confirmed) |
| `signer` | string | Address that signed the transaction |
| `chainId` | number | Chain ID the transaction was submitted to |

### Error Responses[​](https://docs.bankr.bot/agent-api/submit-endpoint/#error-responses "Direct link to Error Responses")

**Invalid Request (400)**

# Sign Endpoint | Bankr Documentation

[Skip to main content](https://docs.bankr.bot/agent-api/sign-endpoint/#__docusaurus_skipToContent_fallback)

On this page

Sign messages and transactions without broadcasting them to the network.

Endpoint[​](https://docs.bankr.bot/agent-api/sign-endpoint/#endpoint "Direct link to Endpoint")

------------------------------------------------------------------------------------------------

    POST /agent/sign

Overview[​](https://docs.bankr.bot/agent-api/sign-endpoint/#overview "Direct link to Overview")

------------------------------------------------------------------------------------------------

The sign endpoint allows you to cryptographically sign:

*   **Plain text messages** — Standard Ethereum message signing (`personal_sign`)
*   **Typed data** — EIP-712 structured data signing (`eth_signTypedData_v4`)
*   **Transactions** — Sign a transaction without submitting it (`eth_signTransaction`)

This is a **synchronous** endpoint that returns immediately with the signature.

Request[​](https://docs.bankr.bot/agent-api/sign-endpoint/#request "Direct link to Request")

---------------------------------------------------------------------------------------------

### Headers[​](https://docs.bankr.bot/agent-api/sign-endpoint/#headers "Direct link to Headers")

| Header | Value | Required |
| --- | --- | --- |
| `Content-Type` | `application/json` | Yes |
| `X-API-Key` | Your API key | Yes |

### Body[​](https://docs.bankr.bot/agent-api/sign-endpoint/#body "Direct link to Body")

The request body varies based on the signature type.

#### personal\_sign[​](https://docs.bankr.bot/agent-api/sign-endpoint/#personal_sign "Direct link to personal_sign")

Sign a plain text message:

    {  "signatureType": "personal_sign",  "message": "Hello, Bankr!"}

| Field | Type | Description | Required |
| --- | --- | --- | --- |
| `signatureType` | string | Must be `"personal_sign"` | Yes |
| `message` | string | The message to sign | Yes |

#### eth\_signTypedData\_v4[​](https://docs.bankr.bot/agent-api/sign-endpoint/#eth_signtypeddata_v4 "Direct link to eth_signTypedData_v4")

Sign EIP-712 structured data:

    {  "signatureType": "eth_signTypedData_v4",  "typedData": {    "domain": {      "name": "MyApp",      "version": "1",      "chainId": 8453,      "verifyingContract": "0x..."    },    "types": {      "Permit": [        { "name": "owner", "type": "address" },        { "name": "spender", "type": "address" },        { "name": "value", "type": "uint256" },        { "name": "nonce", "type": "uint256" },        { "name": "deadline", "type": "uint256" }      ]    },    "primaryType": "Permit",    "message": {      "owner": "0x...",      "spender": "0x...",      "value": "1000000",      "nonce": "0",      "deadline": "1707158800"    }  }}

| Field | Type | Description | Required |
| --- | --- | --- | --- |
| `signatureType` | string | Must be `"eth_signTypedData_v4"` | Yes |
| `typedData` | object | EIP-712 typed data structure | Yes |
| `typedData.domain` | object | Domain separator (name, version, chainId, verifyingContract) | Yes |
| `typedData.types` | object | Type definitions for the message | Yes |
| `typedData.primaryType` | string | The primary type being signed | Yes |
| `typedData.message` | object | The actual data to sign | Yes |

#### eth\_signTransaction[​](https://docs.bankr.bot/agent-api/sign-endpoint/#eth_signtransaction "Direct link to eth_signTransaction")

Sign a transaction without broadcasting:

    {  "signatureType": "eth_signTransaction",  "transaction": {    "to": "0x...",    "chainId": 8453,    "value": "1000000000000000000",    "data": "0x..."  }}

| Field | Type | Description | Required |
| --- | --- | --- | --- |
| `signatureType` | string | Must be `"eth_signTransaction"` | Yes |
| `transaction.to` | string | Destination address | Yes |
| `transaction.chainId` | number | Chain ID | Yes |
| `transaction.value` | string | Value in wei | No  |
| `transaction.data` | string | Calldata (hex) | No  |
| `transaction.gas` | string | Gas limit | No  |
| `transaction.gasPrice` | string | Legacy gas price | No  |
| `transaction.maxFeePerGas` | string | EIP-1559 max fee | No  |
| `transaction.maxPriorityFeePerGas` | string | EIP-1559 priority fee | No  |
| `transaction.nonce` | number | Transaction nonce | No  |

# Installation | Bankr Documentation

[Skip to main content](https://docs.bankr.bot/sdk/installation/#__docusaurus_skipToContent_fallback)

On this page

Install the Bankr SDK for TypeScript/JavaScript applications that need to manage their own wallet and transaction submission.

note

**Most users should start with the [Agent API](https://docs.bankr.bot/agent-api/overview)
 or [OpenClaw skill](https://docs.bankr.bot/openclaw/installation)
** — they're simpler and handle wallet management for you.

The SDK is designed for advanced use cases where you need:

*   To manage your own wallet and submit transactions yourself
*   Fine-grained control over transaction data
*   Integration with existing wallet infrastructure

How the SDK Works[​](https://docs.bankr.bot/sdk/installation/#how-the-sdk-works "Direct link to How the SDK Works")

--------------------------------------------------------------------------------------------------------------------

Unlike the Agent API (which executes transactions for you), the SDK:

1.  Accepts prompts and returns **transaction data**
2.  You submit the transactions to the blockchain using **your own wallet**
3.  Payments are made via x402 micropayments ($0.01 USDC per request)

This gives you full control but requires more integration work.

Requirements[​](https://docs.bankr.bot/sdk/installation/#requirements "Direct link to Requirements")

-----------------------------------------------------------------------------------------------------

*   Node.js 18+ or Bun
*   USDC on Base (for x402 payments)
*   A wallet private key (for signing payments and transactions)

Install[​](https://docs.bankr.bot/sdk/installation/#install "Direct link to Install")

--------------------------------------------------------------------------------------

    npm install @bankr/sdk# orbun add @bankr/sdk# oryarn add @bankr/sdk

Dependencies[​](https://docs.bankr.bot/sdk/installation/#dependencies "Direct link to Dependencies")

-----------------------------------------------------------------------------------------------------

The SDK uses these peer dependencies:

*   `viem` — Ethereum interactions
*   `x402-fetch` — Payment protocol

These are included automatically.

Quick Start[​](https://docs.bankr.bot/sdk/installation/#quick-start "Direct link to Quick Start")

--------------------------------------------------------------------------------------------------

    import { BankrClient } from '@bankr/sdk';// Initialize with your private keyconst client = new BankrClient({  privateKey: '0x...', // Your wallet private key  baseUrl: 'https://api.bankr.bot', // Production endpoint});// Send a prompt and wait for resultsconst result = await client.promptAndWait({  prompt: 'what is the price of ETH?',});console.log(result.response);// "ETH is currently trading at $3,245.67"

Configuration Options[​](https://docs.bankr.bot/sdk/installation/#configuration-options "Direct link to Configuration Options")

--------------------------------------------------------------------------------------------------------------------------------

    const client = new BankrClient({  // Required  privateKey: '0x...',  // Recommended  baseUrl: 'https://api.bankr.bot',  // Production API endpoint  // Optional  timeout: 600000,                    // Request timeout (10 min default)  walletAddress: '0x...',             // Context wallet (defaults to derived)});

Environment Variables[​](https://docs.bankr.bot/sdk/installation/#environment-variables "Direct link to Environment Variables")

--------------------------------------------------------------------------------------------------------------------------------

Store your private key securely:

    # .envBANKR_PRIVATE_KEY=0x...

    import { BankrClient } from '@bankr/sdk';const client = new BankrClient({  privateKey: process.env.BANKR_PRIVATE_KEY as `0x${string}`,});

TypeScript[​](https://docs.bankr.bot/sdk/installation/#typescript "Direct link to TypeScript")

-----------------------------------------------------------------------------------------------

The SDK is fully typed. Import types as needed:

    import {  BankrClient,  BankrClientConfig,  PromptOptions,  JobStatus,  Transaction,} from '@bankr/sdk';

-------------------------------------------------------------------------------------------------------------------------------------------------------

    ┌─────────────────┐│  Your Agent     ││  (provides      ││   value)        │└────────┬────────┘         │ Users interact         ▼┌─────────────────┐│  Token Trading  ││  (generates     ││   fees)         │└────────┬────────┘         │ Fees accumulate         ▼┌─────────────────┐│  Fee Claims     ││  (ETH/WETH)     │└────────┬────────┘         │ Revenue         ▼┌─────────────────┐│  Your Agent     │◀──┐│  (continues     │   ││   running)      │───┘└─────────────────┘

Gas Sponsorship

Bankr covers gas fees for agent wallets on supported chains. Your agent can execute transactions without needing ETH for gas.

    const API_KEY = process.env.BANKR_API_KEY;const BASE_URL = 'https://api.bankr.bot/agent';async function executePrompt(prompt: string) {  // Submit prompt  const submitRes = await fetch(`${BASE_URL}/prompt`, {    method: 'POST',    headers: {      'Content-Type': 'application/json',      'X-API-Key': API_KEY,    },    body: JSON.stringify({ prompt }),  });  const { jobId } = await submitRes.json();  // Poll for completion  while (true) {    const statusRes = await fetch(`${BASE_URL}/job/${jobId}`, {      headers: { 'X-API-Key': API_KEY },    });    const job = await statusRes.json();    if (job.status === 'completed') return job.response;    if (job.status === 'failed') throw new Error(job.error);    await new Promise(r => setTimeout(r, 2000));  }}// Use itconst fees = await executePrompt('check fees for MyToken');await executePrompt('claim my fees for MyToken');

# TWAP Orders | Bankr Documentation

[Skip to main content](https://docs.bankr.bot/features/trading/twap-orders#__docusaurus_skipToContent_fallback)

On this page

Time-Weighted Average Price orders for large trades.

Overview[​](https://docs.bankr.bot/features/trading/twap-orders#overview "Direct link to Overview")

----------------------------------------------------------------------------------------------------

TWAP (Time-Weighted Average Price) splits large orders into smaller chunks executed over time. This:

*   Reduces price impact
*   Gets better average execution
*   Minimizes slippage on big trades

Supported Chains[​](https://docs.bankr.bot/features/trading/twap-orders#supported-chains "Direct link to Supported Chains")

----------------------------------------------------------------------------------------------------------------------------

TWAP orders are available on **EVM chains only**:

*   Base
*   Ethereum
*   Polygon
*   Unichain

Not available on Solana.

Basic TWAP[​](https://docs.bankr.bot/features/trading/twap-orders#basic-twap "Direct link to Basic TWAP")

----------------------------------------------------------------------------------------------------------

Split a large sell over time:

    "sell 1000 BNKR over the next 4 hours""TWAP sell 5000 DEGEN over 2 hours"

When to Use TWAP[​](https://docs.bankr.bot/features/trading/twap-orders#when-to-use-twap "Direct link to When to Use TWAP")

----------------------------------------------------------------------------------------------------------------------------

TWAP is best for:

*   Large positions relative to liquidity
*   Tokens with thin order books
*   Minimizing market impact

### Example Scenario[​](https://docs.bankr.bot/features/trading/twap-orders#example-scenario "Direct link to Example Scenario")

You hold 10,000 BNKR and want to sell. A single swap might:

*   Move the price significantly
*   Get poor execution due to slippage

With TWAP:

    "sell 10000 BNKR over 6 hours"

This executes smaller chunks every few minutes, getting better average price.

Configuration[​](https://docs.bankr.bot/features/trading/twap-orders#configuration "Direct link to Configuration")

-------------------------------------------------------------------------------------------------------------------

### Duration[​](https://docs.bankr.bot/features/trading/twap-orders#duration "Direct link to Duration")

How long to spread the execution:

    "TWAP sell 5000 DEGEN over 1 hour""TWAP sell 10000 BNKR over 4 hours""TWAP sell 50000 tokens over 24 hours"

### Chunk Size[​](https://docs.bankr.bot/features/trading/twap-orders#chunk-size "Direct link to Chunk Size")

Bankr automatically calculates optimal chunk sizes based on:

*   Total amount
*   Duration
*   Typical gas costs

Managing TWAP Orders[​](https://docs.bankr.bot/features/trading/twap-orders#managing-twap-orders "Direct link to Managing TWAP Orders")

Manual: Prompt Then Poll[​](https://docs.bankr.bot/sdk/prompt-and-poll#manual-prompt-then-poll "Direct link to Manual: Prompt Then Poll")

------------------------------------------------------------------------------------------------------------------------------------------

For more control, separate the prompt and polling:

### Step 1: Submit Prompt[​](https://docs.bankr.bot/sdk/prompt-and-poll#step-1-submit-prompt "Direct link to Step 1: Submit Prompt")

    const promptResponse = await client.prompt({  prompt: 'buy $5 of BNKR on base',});console.log(promptResponse.jobId); // "abc123"console.log(promptResponse.status); // "pending"

### Step 2: Poll for Results[​](https://docs.bankr.bot/sdk/prompt-and-poll#step-2-poll-for-results "Direct link to Step 2: Poll for Results")

    const result = await client.pollJob({  jobId: promptResponse.jobId,  interval: 2000,    // Check every 2 seconds  maxAttempts: 150,  // Max 150 attempts  timeout: 300000,   // 5 minute timeout});if (result.status === 'completed') {  console.log(result.response);} else if (result.status === 'failed') {  console.error(result.error);}
