# Volt Documentation

Actors
======

The **Actors** library provides functions for working with Roblox's parallel Luau actors and Lua state management.

[Overview](https://docs.volt.bz/docs/actors#overview)

------------------------------------------------------

These functions allow you to:

*   Get and manage actors in the game
*   Run code on specific actor threads
*   Create communication channels between actors
*   Check parallel execution state
*   Access and control Lua execution states
*   Monitor actor state creation

[Available Functions](https://docs.volt.bz/docs/actors#available-functions)

----------------------------------------------------------------------------

| Function | Description |
| --- | --- |
| [`create_comm_channel`](https://docs.volt.bz/docs/actors/create_comm_channel) | Create a communication channel |
| [`get_comm_channel`](https://docs.volt.bz/docs/actors/get_comm_channel) | Get an existing communication channel |
| [`getactors`](https://docs.volt.bz/docs/actors/getactors) | Get all actors in the game |
| [`getactorstates`](https://docs.volt.bz/docs/actors/getactorstates) | Get all active LuaStateProxy objects |
| [`getluastate`](https://docs.volt.bz/docs/actors/getluastate) | Get LuaStateProxy for actor or script |
| [`getgamestate`](https://docs.volt.bz/docs/actors/getgamestate) | Get the default game Lua state |
| [`isparallel`](https://docs.volt.bz/docs/actors/isparallel) | Check if running in parallel |
| [`on_actor_added`](https://docs.volt.bz/docs/actors/on_actor_added) | Event fired when actor is ready |
| [`on_actor_state_created`](https://docs.volt.bz/docs/actors/on_actor_state_created) | Event fired when actor state is created |
| [`run_on_actor`](https://docs.volt.bz/docs/actors/run_on_actor) | Execute code on an actor |

[What are Actors?](https://docs.volt.bz/docs/actors#what-are-actors)

---------------------------------------------------------------------

Actors are Roblox's way of enabling parallel Luau execution. They allow scripts to run concurrently on multiple threads, improving performance for computationally intensive tasks.

[Communication Channels](https://docs.volt.bz/docs/actors#communication-channels)

----------------------------------------------------------------------------------

Communication channels allow actors to send messages to each other. Use `create_comm_channel` to create a channel and `get_comm_channel` to retrieve it from another actor.

create\_comm\_channel
=====================

Creates a communication channel for sending messages between actors.

[Syntax](https://docs.volt.bz/docs/actors/create_comm_channel#syntax)

----------------------------------------------------------------------

    create_comm_channel(name: string?) -> (string, Channel)

[Parameters](https://docs.volt.bz/docs/actors/create_comm_channel#parameters)

------------------------------------------------------------------------------

| Parameter | Type | Description |
| --- | --- | --- |
| `name` | `string` | (Optional) Name for the channel |

[Returns](https://docs.volt.bz/docs/actors/create_comm_channel#returns)

------------------------------------------------------------------------

| Type | Description |
| --- | --- |
| `string` | The channel identifier |
| `Channel` | The channel object |

[Description](https://docs.volt.bz/docs/actors/create_comm_channel#description)

--------------------------------------------------------------------------------

`create_comm_channel` creates a new communication channel that can be used to send and receive messages between actors and the main thread.

[Channel Object](https://docs.volt.bz/docs/actors/create_comm_channel#channel-object)

--------------------------------------------------------------------------------------

| Property/Method | Description |
| --- | --- |
| `Event` | Signal to connect for receiving |
| `Fire(...)` | Send a message through the channel |

[Example](https://docs.volt.bz/docs/actors/create_comm_channel#example)

------------------------------------------------------------------------

    local id, channel = create_comm_channel()
    
    -- Listen for messages
    channel.Event:Connect(function(message)
        print("Received:", message)
    end)
    
    -- Send to an actor
    run_on_actor(someActor, [[\
        local channelId = ...\
        local channel = get_comm_channel(channelId)\
        channel:Fire("Hello from actor!")\
    ]], id)

[Bidirectional Communication](https://docs.volt.bz/docs/actors/create_comm_channel#bidirectional-communication)

----------------------------------------------------------------------------------------------------------------

    -- Main thread
    local id, channel = create_comm_channel()
    channel.Event:Connect(function(response)
        print("Actor responded:", response)
    end)
    
    -- Actor sends back
    run_on_actor(actor, [[\
        local id = ...\
        local channel = get_comm_channel(id)\
        channel:Fire("Message received!")\
    ]], id)

[Related Functions](https://docs.volt.bz/docs/actors/create_comm_channel#related-functions)

--------------------------------------------------------------------------------------------

*   [`get_comm_channel`](https://docs.volt.bz/docs/actors/get_comm_channel)
     - Retrieve a channel
*   [`run_on_actor`](https://docs.volt.bz/docs/actors/run_on_actor)
     - Run code on an actor

get\_comm\_channel
==================

Retrieves an existing communication channel by its identifier.

[Syntax](https://docs.volt.bz/docs/actors/get_comm_channel#syntax)

-------------------------------------------------------------------

    get_comm_channel(id: string) -> Channel

[Parameters](https://docs.volt.bz/docs/actors/get_comm_channel#parameters)

---------------------------------------------------------------------------

| Parameter | Type | Description |
| --- | --- | --- |
| `id` | `string` | The channel identifier |

[Returns](https://docs.volt.bz/docs/actors/get_comm_channel#returns)

---------------------------------------------------------------------

| Type | Description |
| --- | --- |
| `Channel` | The channel object |

[Description](https://docs.volt.bz/docs/actors/get_comm_channel#description)

-----------------------------------------------------------------------------

`get_comm_channel` retrieves a communication channel that was created with `create_comm_channel`. This is typically used inside an actor to get a reference to a channel created on the main thread.

[Example](https://docs.volt.bz/docs/actors/get_comm_channel#example)

---------------------------------------------------------------------

    -- On main thread
    local id, channel = create_comm_channel()
    channel.Event:Connect(print)
    
    -- Inside actor (via run_on_actor)
    run_on_actor(actor, [[\
        local channelId = ...\
        local channel = get_comm_channel(channelId)\
        channel:Fire("Hello from actor!")\
    ]], id)

[Notes](https://docs.volt.bz/docs/actors/get_comm_channel#notes)

-----------------------------------------------------------------

*   The channel ID is passed from the main thread to the actor
*   Use this inside actors to communicate back to the main thread

[Related Functions](https://docs.volt.bz/docs/actors/get_comm_channel#related-functions)

-----------------------------------------------------------------------------------------

*   [`create_comm_channel`](https://docs.volt.bz/docs/actors/create_comm_channel)
     - Create a channel
*   [`run_on_actor`](https://docs.volt.bz/docs/actors/run_on_actor)
     - Run code on an actor

getactorstates
==============

Returns all active LuaStateProxy objects.

[Syntax](https://docs.volt.bz/docs/actors/getactorstates#syntax)

-----------------------------------------------------------------

    getactorstates() -> {LuaStateProxy}

[Returns](https://docs.volt.bz/docs/actors/getactorstates#returns)

-------------------------------------------------------------------

| Type | Description |
| --- | --- |
| `{LuaStateProxy}` | Array of all active LuaStateProxy objects |

[Description](https://docs.volt.bz/docs/actors/getactorstates#description)

---------------------------------------------------------------------------

`getactorstates` returns all active `LuaStateProxy` objects in the game. This includes both actor states and the main game state. Each `LuaStateProxy` represents a unique Lua execution environment.

[Example](https://docs.volt.bz/docs/actors/getactorstates#example)

-------------------------------------------------------------------

    local states = getactorstates()
    print("Total Lua states:", #states)
    
    for i, state in ipairs(states) do
        print(i, "State ID:", state.Id, "Is Actor:", state.IsActorState)
    end

[Filter Actor States](https://docs.volt.bz/docs/actors/getactorstates#filter-actor-states)

-------------------------------------------------------------------------------------------

    local states = getactorstates()
    local actorStates = {}
    
    for _, state in ipairs(states) do
        if state.IsActorState then
            table.insert(actorStates, state)
        end
    end
    
    print("Actor states:", #actorStates)

[Description](https://docs.volt.bz/docs/actors/getgamestate#description)

[Example](https://docs.volt.bz/docs/actors/getgamestate#example)

    local gameState = getgamestate()

    local gameState = getgamestate()

    ]])

[Compare States](https://docs.volt.bz/docs/actors/getgamestate#compare-states)

-------------------------------------------------------------------------------

    local gameState = getgamestate()
    local currentState = getluastate()
    
    if gameState.Id == currentState.Id then
        print("Running on game state")
    else
        print("Running on different state")
    end

    end

        end
        
        modifier(mt)
        
        if wasReadonly then
            setreadonly(mt, true)
        end
    end

    end)

    local function safeLoadstring(code)
        local func, compileErr = loadstring(code)
        if not func then
            return nil, "Compile error: " .. tostring(compileErr)
        end
    
        local success, result = pcall(func)
        if not success then
            return nil, "Runtime error: " .. tostring(result)
        end
    
        return result
    end
    
    local result, err = safeLoadstring("return 1 + 1")

        print("Result:", result)
