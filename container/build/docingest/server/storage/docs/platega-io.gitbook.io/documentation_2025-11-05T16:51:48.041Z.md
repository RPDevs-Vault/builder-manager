# Создание ссылки на оплату | platega | API

**Создание ссылки на оплату**

`POST` /`**app.platega.io/transaction/process**`

Данный эндпоинт шлет запрос нашему API на создание транзакции.

Ниже вы сможете ознакомиться:

*   с заголовками, которые нужно передать для осуществления запроса
    
*   со списком доступных методов оплат для транзакции
    
*   с примерами запроса и ответами от него
    
*   реализацией на различных языках программирования
    

**Заголовки запроса**

Name

Value

Content-Type

`application/json`

X-MerchantId

<MerchantId>

X-Secret

<Api Key>

**Список доступных методов оплат**

Название метода оплаты

Значение (передается в запрос)

СБП / QR

2 - НСПК / QR

CardRu

10 - Карточный 2дс, оплата картами МИР

International

12 - Международный эквайринг

**Пример тела запроса**

Copy

    {
      "paymentMethod": 2, // метод оплаты
      "id": "3fa85f64-5717-4562-b3fc-2c963f66aza6", // айди транзакции (в формате UUID)
      "paymentDetails": {
        "amount": 970, // сумма платежа 
        "currency": "RUB" // валюта
      },
      "description": "test", // назначение платежа
      "return": "https://google.com" // редирект url при успешном платеже
      "failedUrl": "https://yourdomain/fail" // редирект при не успешном платеже
      "payload":"1111"  // дополнительное поле для инциализации информации в вашей системе
    }

Принцип формирования тела запроса для QR / СБП и P2P метода не отличаются друг от друга. Единственное различие в передаче поля paymentMethod нужной цифры, которая указывает на нужный метод оплаты. В случае с QR / СБП мы всегда передаем цифру 2. Все остальные методы, включая метод 1, 2 - 9, связанны с P2P методами

**Ответы**

200 (СБП / QR)

200 (P2P)

400(P2P)

Copy

    {
      "paymentMethod": "SBPQR",
      "transactionId": "3fa85f64-5717-4562-b3fc-2c463f66afa6",
      "redirect": "https://pay.platega.io?qrsbp",
      "return": "https://google.com",
      "paymentDetails": "100 RUB",
      "status": "PENDING",
      "expiresIn": "00:15:00",
      "merchantId": "1a021d91-9b26-4762-b303-5d4aac74e921",
      "usdtRate": 93.45 // курс по которому происходит конвертация в USDT в момент оплаты
    }

Copy

    {
      "paymentMethod": null,
      "transactionId": "3fa85f64-5717-4562-b4fc-2c163f66afa6",
      "redirect": "https://pay.platega.io?id=3fa85f64-5717-4562-b4fc-2c163f66afa6&mh=1a021d91-9b26-4762-b303-5d4aac74e927",
      "return": "https://google.com",
      "paymentDetails": "2000 RUB",
      "status": "PENDING",
      "expiresIn": "00:15:00",
      "merchantId": "1a021d91-9b26-4762-b303-5d4aac74e927",
      "usdtRate": 93.45
    }
    

Copy

      "message": "No available requisites"
    
    } // Данный статус код и сообщение говорят нам о том, что система не смогла найти реквизитов на подходяшую сумму. Для исправления ситуации попробуйте еще раз запросить реквизиты на другую сумму.Советуем запрашивать реквизиты на суммы 1001, 2002, 3001
    {
      "statusCode": 400,

    } // данный статус код и сообщение говорят нам о том, что транзакция с таким айди уже сушествует в системе. Для исправления ситуации измените айди транзакции и сформируйте запрос по новой

# Получение курсов | platega | API

**Endpoint**

Copy

    GET app.platega.io/rates/payment_method_rate

**Описание** Запрос возвращает текущий курс обмена для указанного платёжного метода и валюты.

**Параметры запроса (Query Params)**

Параметр

Тип

Описание

`merchantId`

UUID

ID мерчанта

`paymentMethod`

integer

ID платёжного метода

`currencyFrom`

string

Исходная валюта (например, `RUB`)

`currencyTo`

string

Целевая валюта (например, `USDT`)

**Заголовки (Headers)**

Заголовок

Тип

Описание

`accept`

string

Формат ответа (`text/plain` или `application/json`)

`X-MerchantId`

UUID

ID мерчанта

`X-Secret`

UUID

Секретный ключ мерчанта

**Пример запроса**

Copy

    curl -X 'GET' \
      'https://app.platega.io/rates/payment_method_rate?merchantId=3fa85f64-5717-4562-b3fc-2c463f66afa6&paymentMethod=2&currencyFrom=RUB&currencyTo=USDT' \
      -H 'accept: text/plain' \
      -H 'X-MerchantId: 3fa85f64-5717-f562-b3fc-2c463f66afa6' \
      -H 'X-Secret: -'

**Пример ответа (успешный)**

Copy

    {
      "paymentMethod": 2,
      "currencyFrom": "RUB",
      "currencyTo": "USDT",
      "rate": 0.0105,
      "updatedAt": "2025-08-11T10:15:00Z"
    }

# Проверка статуса оплаты платежа | platega | API

**Список PaymentStatus:**

*   **PENDING** - ожидание оплаты ⌛️
    
*   **CONFIRMED** - подтверждение оплаты ✅
    
*   **EXPIRED** - истек срок оплаты платежа 🕐
    
*   **CANCELED** - отмененный платеж ❌
    
*   **FAILED** - ошибка создания платежа ⛔️
    

`**GET**``**app.platega.io/transaction/{id}**`

Пример ответа:

Copy

    {
      "id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
      "status": "string",
      "paymentDetails": {
        "amount": 2000,
        "currency": "string"
      },
      "merchantName": "string",
      "mechantId": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
      "comission": 0,
      "paymentMethod": "string",
      "expiresIn": "string",
      "return": "string",
      "comissionUsdt": 1.64044944,
      "amountUsdt": 10.8988764,
      "qr": "string",
      "payformSuccessUrl": "string",
      "payload": "string",
      "comissionType": 1,
      "externalId": "0000a4f3-0000-0000-b8ac-fcb675a0000a",
      "description": "string"
    }
    

> При успешной оплате система отправляет статус **CONFIRMED**.
> 
> При неуспешной оплате система отправляет статус **CANCELED**

# Callback | platega | API

![](https://platega-io.gitbook.io/platega.io-api-dokumentaciya/~gitbook/image?url=https%3A%2F%2F2863853083-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252F5zvDCmctvNn3Cgh2ugHL%252Fuploads%252F3EGqQrEoPoQaEoFfmI3Y%252Fimage.png%3Falt%3Dmedia%26token%3D2fd26098-ad19-4288-b675-5a5c42cd3900&width=768&dpr=4&quality=100&sign=bdc3cf56&sv=2)

Настройки -> Callback Urls

Для отправки callback'а необходимо принимать 2 параметра в заголовках запроса (headers)

Key

Value

X-MerchantId

<Ваш MerchantId>

X-Secret

<Ваш API ключ>

**Метод:** **POST**

Copy

    {
        "id": "00000000-0000-0000-0000-000000000000", // id транзакции
        "amount": 0.0, // сумма
        "currency": "string", // валюта транзакции
        "status": "string", // статус транзакции
        "paymentMethod": 0 // метод оплаты
    }

> При успешной оплате система отправляет статус **CONFIRMED**
> 
> При неуспешной оплате система отправляет статус **CANCELED**

Если наше API не получит успешный ответ в течение 60 секунд, запрос будет отменен по таймауту.

Транзакция будет отправлена повторно 3 раза, с интервалом 5 минут между попытками.
