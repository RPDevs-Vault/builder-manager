# Introduction | Spoofing Database

![Page cover](https://spoofing-database.gitbook.io/free-spoofing-database-docs/~gitbook/image?url=https%3A%2F%2Fstatic-2c.gitbook.com%2F_next%2Fstatic%2Fmedia%2Fdefault-page-cover.a2ccd7e9.svg&width=1248&dpr=3&quality=100&sign=8db2ad43&sv=2)

Permanent spoofing refers to a method where hardware IDs (HWIDs) are spoofed in such a way that the changes are permanent. This technique modifies the HWIDs at a deeper system level, ensuring that they persist even after system reboots or Windows reinstallation. It offers a long-term solution because the spoofed IDs remain intact, even after software updates or system restores.

circle-check

**Most of these methods work with Secure Boot and TPM enabled. To be sure, check the top of the method’s page to see if it specifies that it supports Secure Boot and TPM.**

triangle-exclamation

**To use these methods, you will need to reinstall Windows. Make sure to back up any important files from your hard drive beforehand.**

# Introduction | Spoofing Database

Temp Spoofing (Temporary Spoofing) is a method of changing your system’s hardware identifiers (HWID) only for the current session. These changes are not permanent and will be reset after a reboot. It's a safer and faster way to spoof your system, ideal for testing or avoiding detection without modifying your BIOS or system files permanently.

triangle-exclamation

**Most of these methods won’t work if Secure Boot and TPM are enabled.** **To be sure, check the top of the method’s page to see if it specifies that it supports Secure Boot and TPM.**

circle-exclamation

**Please note that these tutorials do not include steps for fully removing system traces. For optimal safety and effectiveness, a clean Windows reinstall is strongly recommended.**

# EFI Spoofing | Spoofing Database

triangle-exclamation

This method does not work with Secure Boot enabled. Make sure to disable it in your BIOS settings before proceeding.

* * *

[hashtag](https://spoofing-database.gitbook.io/free-spoofing-database-docs/perm-spoofing/efi-spoofing#guide-how-to-spoof-your-hwid-using-a-efi-shell)

Guide: How to spoof your HWID using a EFI Shell


----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

circle-info

Before spoofing, make sure to review your current serial numbers and take a screenshot of them. Once you've finished spoofing, you can go back and compare the values to verify that the changes were successfully applied.

* * *

### 

[hashtag](https://spoofing-database.gitbook.io/free-spoofing-database-docs/perm-spoofing/efi-spoofing#step-0-prepare)

Step 0: Prepare

Before we begin spoofing, we first need get and install all the required tools:

1.  EFI Spoofer: [Downloadarrow-up-right](https://drive.google.com/file/d/1t5Pvb93lT8t-8U5QymkIWssUOA3o1Rn-/view?usp=sharing)
    
2.  HWID Checker: [Downloadarrow-up-right](https://drive.google.com/file/d/1-o5rMvjfbeTy9FN958zWBFInTor8PgYh/view?usp=sharing)
    
3.  USB flash drive [Buyarrow-up-right](https://www.amazon.com/SanDisk-128GB-Ultra-Flash-Drive/dp/B081QSJNRJ/ref=sr_1_9?sr=8-9)
    

After **downloading all the necessary tools**, proceed to **enter your BIOS** and adjust the **following settings:**

1.  Disable Secure Boot and clear the Secure Boot keys
    
2.  Enable CSM (Compatibility Support Module)
    
3.  Disable Fast Boot
    

Next you need to Format your USB flash drive to FAT32:

1.  Right-click on your USB flash drive in File Explorer — in my case, it's named **"ESD-USB (D:)"**.
    
2.  Now press the button called **"Format...".**
    
3.  Under **"File system"** select **"FAT32"**
    
4.  Now press **"Start"** and wait for the process to complete.
    

* * *

### 

[hashtag](https://spoofing-database.gitbook.io/free-spoofing-database-docs/perm-spoofing/efi-spoofing#step-1-prepare-usb-flash-drive)

Step 1: Prepare USB flash drive

To change your serials, start by downloading the EFI Spoofer linked above. Extract the contents of the ZIP file to the **root directory** of your USB flash drive (see image below for reference).

![](https://spoofing-database.gitbook.io/free-spoofing-database-docs/~gitbook/image?url=https%3A%2F%2F4106604860-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252FhaLdsYZWMtYUliMbBzss%252Fuploads%252Ff7TRYiv0jD2VKeB7u8Fo%252FAnnotation%25202025-08-05%2520223150.png%3Falt%3Dmedia%26token%3Df1270717-1253-4c95-b1f8-abe3fcda8715&width=768&dpr=3&quality=100&sign=15bc534d&sv=2)

### 

[hashtag](https://spoofing-database.gitbook.io/free-spoofing-database-docs/perm-spoofing/efi-spoofing#step-2-change-serials)

Step 2: Change Serials

Go into the USB flash drive and do the following:

1.  Open the folder **"efi"** and in the folder open the folder **"boot".**
    
2.  Now open the file **"startup.nsh"**
    
3.  Replace each instance of **"YOUR-SERIAL"** with your desired serial number. Refer to the image below for guidance.
    
4.  Save everything and exit out
    

#### 

[hashtag](https://spoofing-database.gitbook.io/free-spoofing-database-docs/perm-spoofing/efi-spoofing#after-setting-up-your-usb-flash-drive-you-need-to-boot-from-it)

After setting up your USB flash drive, you need to boot from it:

1.  Enter your BIOS settings (if you're unsure how, search on YouTube: **"how to enter BIOS \[your motherboard brand\]"**).
    
2.  Now go to the **"BOOT"** tab in your BIOS and set your USB drive as the first device in the boot order. If you're not sure how to do this, search on YouTube: **"how to change boot order \[your motherboard brand\]"**.
    
3.  After that exit out of the BIOS

2.  `AFUWINx64.exe bios.rom /O`
    

After typing these commands, you should see something like this appear:

![](https://spoofing-database.gitbook.io/free-spoofing-database-docs/~gitbook/image?url=https%3A%2F%2F4106604860-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252FhaLdsYZWMtYUliMbBzss%252Fuploads%252FemrDg3JxtR61RWf4gyse%252FAnnotation%25202025-08-05%2520192041.png%3Falt%3Dmedia%26token%3De74b8d08-5f1f-4503-9f70-945b169247eb&width=768&dpr=3&quality=100&sign=6a50dd3&sv=2)

Step 3: Editing your Serials

Start by launching DMIEdit. The first step will be to modify two key fields: the CPU Serial Number and the System Serial Number. These values are often required for system identification

#### 

[hashtag](https://spoofing-database.gitbook.io/free-spoofing-database-docs/perm-spoofing/afuwin-method#system-serial-number)

System Serial Number:

1.  Go to **"\[Type 001\] — System Information"**
    
2.  Double Click **"Serial Number"** and under **"String Value: xxx",** change your Serial Number

CPU Serial Number:

1.  Go to **"\[Type 004\] — Processor Information"**
    
2.  Double Click **"Serial Number"** and under **"String Value: xxx",** change your Serial Number
    
3.  Next, click the icon that looks like a lightning bolt and is labeled 'All'. This will apply the changes you've made.
    

* * *

Now we need to update the Motherboard Serial Number and the System UUID. These two fields are more complex to modify than the others, but they are also the most important serials for proper system identification.

#### 

[hashtag](https://spoofing-database.gitbook.io/free-spoofing-database-docs/perm-spoofing/afuwin-method#system-uuid)

System UUID:

1.  Start by going into **"\[Type 001\] — System Information"** in DMIEdit
    
2.  Double Click **"UUID"**
    
3.  Copy your current UUID under "**Hexadecimal Value: xxx"**
    
4.  Go into your configured HxD and press `CTRL + R` to replace
    
5.  In the Replace Window select: **"Hex-values"**
    
6.  In **"Search for: "** paste your current UUID
    
7.  Go back to DMIEdit and press **"Auto Generates system UUID"** and copy your new UUID
    
8.  Go back to HxD and in **"Replace with: "** paste your new UUID
    
9.  Under **"Search direction"** select **"All"** and press **"Replace All"**
    
10.  Press `CTRL + S` to save your new changes
    

#### 

[hashtag](https://spoofing-database.gitbook.io/free-spoofing-database-docs/perm-spoofing/afuwin-method#motherboard-serial-number)

Motherboard Serial Number:

circle-exclamation

Your new serial number must be the same length as the original. For example, if your old serial is 8 digits long, your new one must also be exactly 8 digits. Example: Old Serial: 12345678, New Serial: 87654321

1.  Start by going into **"\[Type 002\] — Base Board/Module Information"** in DMIEdit
    
2.  Double Click **"Serial Number"** and copy your current Serial Number
    
3.  Go into HxD with your bios.rom loaded and press `CTRL + R` to replace
    
4.  in the Replace Window select: "Text-string"
    
5.  In **"Search for: "** paste your current Serial Number
    
6.  In **"Replace with: "** type a new Serial Number the same length as the old one (see above)
    
7.  Under **"Search direction"** select **"All"** and press **"Replace All"**
    
8.  Press `CTRL + S` to save your new changes

[hashtag](https://spoofing-database.gitbook.io/free-spoofing-database-docs/perm-spoofing/afuwin-method#step-4-flash-your-new-bios.rom)

Step 4: Flash your new bios.rom

After you've successfully modified all necessary serial numbers, the next step is to flash the updated BIOS to your system. This process is essential, as it applies the changes you've made by writing them directly to the system firmware. Without flashing, the new serials will not be stored permanently and your system will continue to use the old values.

Flashing the BIOS essentially overwrites the existing firmware with the modified version, ensuring that your changes — such as the CPU Serial Number, System UUID, and Motherboard Serial Number — are properly saved and recognized by the system during startup.

circle-exclamation

For the next step, you'll need to use a specific version of AFUWIN: **version 3.05.04**. This is crucial because it's the only known version that includes support for the `/GAN` command, which is required to force-flash the modified BIOS onto your system.

Start by opening a Administrator Console Prompt and type in the following commands:

1.  `cd "your current AFUWIN directory"`
    
2.  `AFUWINx64.exe bios.rom /GAN`
    

You will see something like this if you did everything right:

![](https://spoofing-database.gitbook.io/free-spoofing-database-docs/~gitbook/image?url=https%3A%2F%2F4106604860-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252FhaLdsYZWMtYUliMbBzss%252Fuploads%252FKFQVYlSCzQc5Kkuf0UtV%252Fasdasda.png%3Falt%3Dmedia%26token%3Dcb1bdf58-eaa8-4120-9c99-61cf9587a98c&width=768&dpr=3&quality=100&sign=352825fb&sv=2)

If you followed all the steps correctly, the flashing process should complete successfully and display a message such as **"done"**. This indicates that your modified BIOS has been written to the system.

### 

[hashtag](https://spoofing-database.gitbook.io/free-spoofing-database-docs/perm-spoofing/afuwin-method#step-5-post-spoofing)

Step 5: Post Spoofing

Now that your BIOS has been successfully spoofed, you can move on to other system identifiers, such as your **MAC address**, **Volume ID**, or **Disk Serial Number**. These values are often used by software and anti-cheat systems for hardware identification.

I'll cover how to spoof each of these as well. So if there's already a dedicated page or section on these topics, feel free to go ahead and complete them next.

circle-check

You can now restore your original BIOS settings if you wish. This includes any custom configurations you had before flashing, such as Secure Boot, Secure Boot Keys, CSM and Fast Boot.

Guide: How to Change your Hard Disk Volume ID


-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

* * *

circle-info

Before spoofing, make sure to review your current Volume ID and take a screenshot of it. Once you've finished spoofing, you can go back and compare the values to verify that the changes were successfully applied.

* * *

### 

[hashtag](https://spoofing-database.gitbook.io/free-spoofing-database-docs/disk-spoofing#step-0-prepare)

Step 0: Prepare

Before we begin changing our Volume ID, we need to gather all the necessary tools:

1.  **VolumeID:** [Downloadarrow-up-right](https://download.sysinternals.com/files/VolumeId.zip)
    
2.  **HWID Checker:** [Downloadarrow-up-right](https://drive.google.com/file/d/1-o5rMvjfbeTy9FN958zWBFInTor8PgYh/view?usp=sharing)
    

* * *

### 

[hashtag](https://spoofing-database.gitbook.io/free-spoofing-database-docs/disk-spoofing#step-1-setting-up-volumeid)

Step 1: Setting up VolumeID

After downloading VolumeID, we need to set it up properly to avoid errors.

1.  Open the freshly installed **"Volumeid.zip"** and Drag-and-Drop the **"Volumeid.exe"** into a folder, and copy the path
    
2.  Run Command Prompt as Administrator and navigate to the VolumeID folder using: `cd "path-to-VolumeID"`
    

* * *

### 

[hashtag](https://spoofing-database.gitbook.io/free-spoofing-database-docs/disk-spoofing#step-2-changing-serials)

Step 2: Changing Serials

Go into the already setuped Command Prompt and do the following:

1.  **Type the following command:** `**volumeid.exe [drive letter]: XXXX-XXXX**` _Replace_ `_[drive letter]_` _with your actual drive (e.g., C) and_ `_XXXX-XXXX_` _with a valid ID using characters 0–9 and A–F._
    
2.  Now repeat the first step for each drive you have.

# Introduction | Spoofing Database

Temp Spoofing (Temporary Spoofing) is a method of changing your system’s hardware identifiers (HWID) only for the current session. These changes are not permanent and will be reset after a reboot. It's a safer and faster way to spoof your system, ideal for testing or avoiding detection without modifying your BIOS or system files permanently.

triangle-exclamation

**Most of these methods won’t work if Secure Boot and TPM are enabled.** **To be sure, check the top of the method’s page to see if it specifies that it supports Secure Boot and TPM.**

circle-exclamation

**Please note that these tutorials do not include steps for fully removing system traces. For optimal safety and effectiveness, a clean Windows reinstall is strongly recommended.**
