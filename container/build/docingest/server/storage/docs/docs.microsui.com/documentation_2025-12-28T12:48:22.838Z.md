# Introduction | MicroSui Docs

[Skip to main content](https://docs.microsui.com/docs/intro#__docusaurus_skipToContent_fallback)

On this page

This documentation is developer-oriented.

The **framework** has two main components: a **_C-library repository_** and a **_set of Hardware Board-specific Implementations_**.

The **_MicroSui C-library repository_**, containing the core and generic code **written in C Language**, serves as the foundation for all the **hardware implementations**, some of which may require minor adaptations to the library.

Main Components[​](https://docs.microsui.com/docs/intro#main-components "Direct link to Main Components")

----------------------------------------------------------------------------------------------------------

### MicroSui C-library[​](https://docs.microsui.com/docs/intro#microsui-c-library "Direct link to MicroSui C-library")

The MicroSui Framework includes **[microsui-lib](https://github.com/MicroSui/microsui-lib)
** repository, a lightweight implementation for running _Sui Clients_ in _C_.

It’s designed to be compatible with virtually **any device or computer**. That said, let’s be brutally honest: there’s little reason to use it on powerful systems where robust solutions like _Mysten’s TypeScript SDK_ are available. In those cases, it is strongly recommended to use the tools offered by _Mysten Labs_ — for obvious reasons.

`microsui-lib` is truly valuable in constrained environments devices where firmware **MUST** be written in C. That’s where the library truly shines, offering a unique solution where no alternatives exist.

In most cases, developers will use this repository as a library whenever they want to build a hardware implementation.

### Hardware Implementations[​](https://docs.microsui.com/docs/intro#hardware-implementations "Direct link to Hardware Implementations")

The _MicroSui C-lib_ alone is not enough — the framework’s real value lies in making it easier for developers to build their own solutions.

That’s why _MicroSui_ provides **hardware implementations** with ready-to-use examples for some of the most popular boards and microcontrollers used worldwide and across industries.

So far, these are the available implementations:

*   **[microsui-esp32](https://github.com/MicroSui/microsui-esp32)

# Overview | MicroSui Docs

[Skip to main content](https://docs.microsui.com/docs/hardware-implementations/microsui-esp32/overview#__docusaurus_skipToContent_fallback)

This hardware implementation of **_MicroSui_** is based on the well-known **_ESP32_** — a powerful microcontroller with integrated _Wi-Fi_ and _Bluetooth_, ideal for _IoT_ and _embedded applications_ thanks to its high performance and low power consumption.

In this implementation, we use the `esp-idf` framework, which is the official one provided by **_Espressif_**, the manufacturer of this chip. We also use **_PlatformIO_** as the development environment due to the many advantages it offers.

This project is configured for the _C3_ version of the chip (_ESP32-C3_). However, the code is fully compatible with any _ESP32_ version. If you wish to use a different version, simply change the board name in the _PlatformIO configuration_.

GitHub Repo Link: [https://github.com/MicroSui/microsui-esp32](https://github.com/MicroSui/microsui-esp32)

# Offline Sign from Sui Bech32 private key | MicroSui Docs

[Skip to main content](https://docs.microsui.com/docs/hardware-implementations/microsui-esp32/examples/offline-sign-bech#__docusaurus_skipToContent_fallback)

On this page

### Overview[​](https://docs.microsui.com/docs/hardware-implementations/microsui-esp32/examples/offline-sign-bech#overview "Direct link to Overview")

This example adapted for the ESP32 includes a sample prebuilt message in hexadecimal format, which will be signed using a Sui-compatible Bech32 format private key (starting with `suiprivkey1...`) provided by the user. The key will be decoded internally before signing.

The result will be a hexadecimal signature that can be used, together with the message, to construct a Sui transaction.

# microsui_sign_message | MicroSui Docs

[Skip to main content](https://docs.microsui.com/docs/microsui-clibrary/functions/sign.c/microsui_sign_message#__docusaurus_skipToContent_fallback)

On this page

### Function[​](https://docs.microsui.com/docs/microsui-clibrary/functions/sign.c/microsui_sign_message#function "Direct link to Function")

    int microsui_sign_message(uint8_t sui_sig[97], const char* message_hex, const uint8_t private_key[32]);

### Description[​](https://docs.microsui.com/docs/microsui-clibrary/functions/sign.c/microsui_sign_message#description "Direct link to Description")

Generates a valid Sui signature for a given hexadecimal message using the Ed25519 scheme. The output is a compact Sui-compatible signature structure.

### Location[​](https://docs.microsui.com/docs/microsui-clibrary/functions/sign.c/microsui_sign_message#location "Direct link to Location")

Located in `/src/sign.c`

### Parameters[​](https://docs.microsui.com/docs/microsui-clibrary/functions/sign.c/microsui_sign_message#parameters "Direct link to Parameters")

| Name | Type | Description |
| --- | --- | --- |
| `sui_sig` | `uint8_t[97]` | Output buffer for the generated Sui signature. Format:  <br>• Byte 0: `0x00` (Ed25519 scheme)  <br>• Bytes 1–64: Ed25519 signature  <br>• Bytes 65–96: Ed25519 public key |
| `message_hex` | `const char*` | Null-terminated string representing the message in hexadecimal (2 hex chars per byte). |
| `private_key` | `const uint8_t[32]` | Raw 32-byte Ed25519 private key used for signature generation. |

### Returns[​](https://docs.microsui.com/docs/microsui-clibrary/functions/sign.c/microsui_sign_message#returns "Direct link to Returns")

*   `0` on success.

### Example usage[​](https://docs.microsui.com/docs/microsui-clibrary/functions/sign.c/microsui_sign_message#example-usage "Direct link to Example usage")

Given:

*   A hexadecimal message: `"48656c6c6f2053756921"` (which is `"Hello Sui!"`)
*   A 32-byte Ed25519 private key

You can generate a signature as follows:

    uint8_t sui_sig[97];  // We initialize it emptystatus = microsui_sign_message(sui_sig, message_hex, private_key)if status == 0:    // Signature is now stored in sui_sig

# microsui_decode_sui_privkey | MicroSui Docs

[Skip to main content](https://docs.microsui.com/docs/microsui-clibrary/functions/cryptography.c/microsui_decode_sui_privkey#__docusaurus_skipToContent_fallback)

On this page

### Function[​](https://docs.microsui.com/docs/microsui-clibrary/functions/cryptography.c/microsui_decode_sui_privkey#function "Direct link to Function")

    int microsui_decode_sui_privkey(const char *privkey_bech, uint8_t *privkey_bytes_out);

### Description[​](https://docs.microsui.com/docs/microsui-clibrary/functions/cryptography.c/microsui_decode_sui_privkey#description "Direct link to Description")

Decodes a Sui-formatted Bech32 private key string (e.g. `suiprivkey1...`) into its raw 32-byte Ed25519 private key representation. The input string must have a valid checksum and the expected prefix.

### Location[​](https://docs.microsui.com/docs/microsui-clibrary/functions/cryptography.c/microsui_decode_sui_privkey#location "Direct link to Location")

Located in `/src/cryptography.c`

### Parameters[​](https://docs.microsui.com/docs/microsui-clibrary/functions/cryptography.c/microsui_decode_sui_privkey#parameters "Direct link to Parameters")

| Name | Type | Description |
| --- | --- | --- |
| `privkey_bech` | `const char*` | Null-terminated Bech32-encoded private key string with the prefix `suiprivkey`. |
| `privkey_bytes_out` | `uint8_t[32]` | Output buffer for the decoded 32-byte Ed25519 private key. |

### Returns[​](https://docs.microsui.com/docs/microsui-clibrary/functions/cryptography.c/microsui_decode_sui_privkey#returns "Direct link to Returns")

*   `0` on success.
*   `-1` if decoding fails (e.g. invalid checksum, bad prefix, incorrect length or internal conversion failure).

### Example usage[​](https://docs.microsui.com/docs/microsui-clibrary/functions/cryptography.c/microsui_decode_sui_privkey#example-usage "Direct link to Example usage")

    const char *pk_bech32 = "suiprivkey1.....";uint8_t pk_bytes_output[32];status = microsui_decode_sui_privkey(pk_bech32, pk_bytes_output);if (status == 0)    // 32-byte Ed25519 pk is now stored in pk_bytes_output

# microsui_encode_sui_privkey | MicroSui Docs

[Skip to main content](https://docs.microsui.com/docs/microsui-clibrary/functions/cryptography.c/microsui_encode_sui_privkey#__docusaurus_skipToContent_fallback)

On this page

### Function[​](https://docs.microsui.com/docs/microsui-clibrary/functions/cryptography.c/microsui_encode_sui_privkey#function "Direct link to Function")

    int microsui_encode_sui_privkey(const uint8_t *privkey_bytes, char *privkey_bech_output);

### Description[​](https://docs.microsui.com/docs/microsui-clibrary/functions/cryptography.c/microsui_encode_sui_privkey#description "Direct link to Description")

Encodes a 32-byte Ed25519 private key into a Bech32 string with the Sui prefix `suiprivkey`. The output includes the `0x00` scheme flag byte and a valid 6-character Bech32 checksum, producing a fully compatible Sui private key string.

### Location[​](https://docs.microsui.com/docs/microsui-clibrary/functions/cryptography.c/microsui_encode_sui_privkey#location "Direct link to Location")

Located in `/src/cryptography.c`

### Parameters[​](https://docs.microsui.com/docs/microsui-clibrary/functions/cryptography.c/microsui_encode_sui_privkey#parameters "Direct link to Parameters")

| Name | Type | Description |
| --- | --- | --- |
| `privkey_bytes` | `const uint8_t[32]` | Raw 32-byte Ed25519 private key to encode. |
| `privkey_bech_output` | `char*` | Output buffer where the resulting Bech32 string will be written (null-terminated). The length must be 71 bytes. |

### Returns[​](https://docs.microsui.com/docs/microsui-clibrary/functions/cryptography.c/microsui_encode_sui_privkey#returns "Direct link to Returns")

*   `0` on success.
*   `-1` if encoding fails (e.g. due to bad input, insufficient buffer size or internal conversion failure).

### Example usage[​](https://docs.microsui.com/docs/microsui-clibrary/functions/cryptography.c/microsui_encode_sui_privkey#example-usage "Direct link to Example usage")

    const uint8_t pk_bytes[32] = { /* your 32-byte private key */ };char pk_bech32_output[71];  // We initialize it emptystatus = microsui_encode_sui_privkey(pk_bytes, pk_bech32_output)if status == 0:    // Sui compatible Bech32 pk is now stored in pk_bech32_output
