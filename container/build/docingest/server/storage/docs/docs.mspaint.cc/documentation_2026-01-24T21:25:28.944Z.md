# Getting Started

Getting Started
===============

Getting Started with Obsidian

* * *

![Obsidian](https://docs.mspaint.cc/_next/image?url=%2F_next%2Fstatic%2Fmedia%2Fobsidian.c84b794c.png&w=3840&q=75)

[Introduction](https://docs.mspaint.cc/obsidian#introduction)

--------------------------------------------------------------

Welcome to the documentation for Obsidian, a clean Roblox UI Library made by deivid for the [mspaint script hub](https://www.mspaint.cc/)
.

_This is why the Library has an API modeled after the [LinoriaLib](https://github.com/mstudio45/linorialib)
 UI library (as that was the previous UI Library for mspaint)._

Any changes that are done to the Obsidian library will be reflected in our [LinoriaLib UI Fork](https://github.com/mstudio45/linorialib)
.

[Why Obsidian?](https://docs.mspaint.cc/obsidian#why-obsidian)

---------------------------------------------------------------

*   Clean and simple UI
*   Full Mobile and PC support
*   Customizable themes
*   Built-in Save Manager
*   Maintained UI Library

[Installation](https://docs.mspaint.cc/obsidian#installation)

--------------------------------------------------------------

Obsidian is available to use on **Roblox Studio**, **Executors** and even **Next.js + Tailwind**. Please refer to the [installation guide](https://docs.mspaint.cc/obsidian/installation)
.

[Quick Start](https://docs.mspaint.cc/obsidian#quick-start)

------------------------------------------------------------

First understand the [structure](https://docs.mspaint.cc/obsidian/structure)
 of Obsidian **(This is important if you are not familiar with LinoriaLib)**

Then create a [window](https://docs.mspaint.cc/obsidian/core/library#Window)
 and start adding [tabs](https://docs.mspaint.cc/obsidian/structure/tabs)
 and [groupboxes](https://docs.mspaint.cc/obsidian/structure/groupboxes)
 to it. If you are less into reading and more into checking code, the Obsidian example script can be found [here](https://github.com/deividcomsono/Obsidian/blob/main/Example.lua)
.

[Suggestions and Bug Reports](https://docs.mspaint.cc/obsidian#suggestions-and-bug-reports)

--------------------------------------------------------------------------------------------

If you have any suggestions or bug reports, please feel free to reach out to us on our [open source discord server variant](https://discord.gg/AZp8cR8DaU)
. We also read through [issues](https://github.com/deividcomsono/Obsidian/issues)
 and [pull requests](https://github.com/deividcomsono/Obsidian/pulls)
 regularly.

# Structuring

Structuring
===========

Structuring is the process of organizing your Obsidian UI.

Obsidian uses a UI Library API similar to [LinoriaLib](https://github.com/mstudio45/linorialib)
. This means that our LinoriaLib fork and Obsidian are compatible with each other.

To structure your Obsidian UI, you first need to understand the underlying organization of Obsidian.

[Tabs](https://docs.mspaint.cc/obsidian/structure#tabs)

--------------------------------------------------------

Tabs are the main way to seperate each category of elements for your UI. Each tab has a name and optionally a description and icon. The tabs show up on the left side of the Obsidian window.

![tab structure](https://docs.mspaint.cc/_next/image?url=%2F_next%2Fstatic%2Fmedia%2FTabStructure.56560dd3.png&w=3840&q=75)

* * *

[Groupboxes & Tabboxes](https://docs.mspaint.cc/obsidian/structure#groupboxes--tabboxes)

-----------------------------------------------------------------------------------------

Groupboxes and tabboxes are organized inside of the tab content area. The tab content area is divided into two columns, left and right. Each column can have multiple groupbox and/or tabbox inside of it.

![groupbox structure](https://docs.mspaint.cc/_next/image?url=%2F_next%2Fstatic%2Fmedia%2FTabContentStructure.0250fd5d.png&w=3840&q=75)

Inside each groupbox/tabbox, you can add any UI Element in the UI elements section.

![groupbox elements](https://docs.mspaint.cc/_next/image?url=%2F_next%2Fstatic%2Fmedia%2FGroupboxElements.a6b75150.png&w=3840&q=75)

# ThemeManager

ThemeManager
============

Using the ThemeManager addon to manage themes in Obsidian

Table of Contents

[What is the ThemeManager?](https://docs.mspaint.cc/obsidian/core/addons/thememanager#what-is-the-thememanager)

----------------------------------------------------------------------------------------------------------------

The ThemeManager addon ships with a library of built-in color schemes and helper utilities to create, persist and reuse custom themes for your Obsidian UI.

[Quick Start](https://docs.mspaint.cc/obsidian/core/addons/thememanager#quick-start)

-------------------------------------------------------------------------------------

Add the ThemeManager alongside your library and attach it to a tab or groupbox so users can tweak colors on the fly:

    local repo = "https://raw.githubusercontent.com/deividcomsono/Obsidian/main/"
    local Library = loadstring(game:HttpGet(repo .. "Library.lua"))()
    local ThemeManager = loadstring(game:HttpGet(repo .. "addons/ThemeManager.lua"))()
     
    -- register the library so ThemeManager can manipulate controls
    ThemeManager:SetLibrary(Library)
     
    -- optional: keep themes inside a custom folder
    ThemeManager:SetFolder("MyScriptHub")
     
    local Tab = Window:AddTab("UI Settings", "settings")
     
    -- automatically creates the color pickers, theme lists and buttons
    ThemeManager:ApplyToTab(Tab)
     
    -- apply whatever theme is flagged as default (built-in or custom)
    ThemeManager:LoadDefault()

### [Built-In Themes](https://docs.mspaint.cc/obsidian/core/addons/thememanager#built-in-themes)

`ThemeManager.BuiltInThemes` exposes packaged schemes such as Default, BBot, Fatality, Nord, Dracula, Catppuccin and more. These are available instantly from the “Theme list” dropdown created by `ApplyToTab`.

[API Reference](https://docs.mspaint.cc/obsidian/core/addons/thememanager#api-reference)

-----------------------------------------------------------------------------------------

### [Methods](https://docs.mspaint.cc/obsidian/core/addons/thememanager#methods)

#### [SetLibrary](https://docs.mspaint.cc/obsidian/core/addons/thememanager#setlibrary)

Registers the UI library instance so ThemeManager can read and update color pickers.

    ThemeManager:SetLibrary(Library)

| Arg Idx | Argument Description | Type | Default |
| --- | --- | --- | --- |
| 1   | Library to use for the ThemeManager | Library (table) | nil |

#### [SetFolder](https://docs.mspaint.cc/obsidian/core/addons/thememanager#setfolder)

Overrides the base folder used to store theme JSON files and default markers.

    ThemeManager:SetFolder(folder)

| Arg Idx | Argument Description | Type | Default |
| --- | --- | --- | --- |
| 1   | Folder path relative to workspace | string | `"ObsidianLibSettings"` |

#### [ApplyTheme](https://docs.mspaint.cc/obsidian/core/addons/thememanager#applytheme)

Applies either a built-in theme or a custom theme saved on disk.

    ThemeManager:ApplyTheme(name)

| Arg Idx | Argument Description | Type | Default |
| --- | --- | --- | --- |
| 1   | Theme name (built-in or custom) | string | nil |

#### [ThemeUpdate](https://docs.mspaint.cc/obsidian/core/addons/thememanager#themeupdate)

Pushes the current picker values back into the library scheme. Automatically called when color pickers change, but exposed if you need to trigger it manually.

    ThemeManager:ThemeUpdate()

#### [GetCustomTheme](https://docs.mspaint.cc/obsidian/core/addons/thememanager#getcustomtheme)

Loads a custom theme JSON file and returns the decoded table.

    local theme = ThemeManager:GetCustomTheme(name)

| Arg Idx | Argument Description | Type | Default |
| --- | --- | --- | --- |
| 1   | Theme file name without extension | string | nil |

Returns `table?`; `nil` when the file does not exist or cannot be decoded.

#### [LoadDefault](https://docs.mspaint.cc/obsidian/core/addons/thememanager#loaddefault)

Reads the saved default marker and applies the relevant theme (built-in or custom).

    ThemeManager:LoadDefault()

#### [SaveDefault](https://docs.mspaint.cc/obsidian/core/addons/thememanager#savedefault)

Stores the provided theme name as the default choice for future sessions.

    ThemeManager:SaveDefault(name)

| Arg Idx | Argument Description | Type | Default |
| --- | --- | --- | --- |
| 1   | Theme name to persist | string | nil |

#### [SetDefaultTheme](https://docs.mspaint.cc/obsidian/core/addons/thememanager#setdefaulttheme)

Replaces the built-in `Default` theme with custom color definitions before rendering the UI. Call this **before** `ApplyToTab`.

    ThemeManager:SetDefaultTheme(theme)

| Arg Idx | Argument Description | Type | Default |
| --- | --- | --- | --- |
| 1   | Table containing `FontColor`, `MainColor`, `AccentColor`, `BackgroundColor`, `OutlineColor`, `FontFace` | table | nil |

#### [SaveCustomTheme](https://docs.mspaint.cc/obsidian/core/addons/thememanager#savecustomtheme)

Serializes the current picker values to `<folder>/themes/<name>.json`.

    ThemeManager:SaveCustomTheme(name)

| Arg Idx | Argument Description | Type | Default |
| --- | --- | --- | --- |
| 1   | Theme name without file extension | string | nil |

#### [Delete](https://docs.mspaint.cc/obsidian/core/addons/thememanager#delete)

Removes a saved custom theme file from disk.

    local success, err = ThemeManager:Delete(name)

| Arg Idx | Argument Description | Type | Default |
| --- | --- | --- | --- |
| 1   | Theme name without file extension | string | nil |

Returns `boolean, string?` describing the result.

#### [ReloadCustomThemes](https://docs.mspaint.cc/obsidian/core/addons/thememanager#reloadcustomthemes)

Re-scans the theme directory and returns a list of custom theme names.

    local names = ThemeManager:ReloadCustomThemes()

Returns `{ string }` of theme names.

#### [CreateThemeManager](https://docs.mspaint.cc/obsidian/core/addons/thememanager#createthememanager)

Builds the ThemeManager controls inside a provided groupbox (color pickers, dropdowns and action buttons).

    ThemeManager:CreateThemeManager(groupbox)

| Arg Idx | Argument Description | Type | Default |
| --- | --- | --- | --- |
| 1   | Groupbox returned by `AddLeftGroupbox`/`AddRightGroupbox` | Groupbox (table) | nil |

#### [CreateGroupBox](https://docs.mspaint.cc/obsidian/core/addons/thememanager#creategroupbox)

Helper that creates the standard “Themes” groupbox on the left side of a tab.

    local groupbox = ThemeManager:CreateGroupBox(tab)

| Arg Idx | Argument Description | Type | Default |
| --- | --- | --- | --- |
| 1   | Tab returned by `Window:AddTab` | Tab (table) | nil |

Returns the created groupbox so you can customize it further before calling `CreateThemeManager`.

#### [ApplyToTab](https://docs.mspaint.cc/obsidian/core/addons/thememanager#applytotab)

Creates a groupbox within the tab and installs the ThemeManager UI into it.

    ThemeManager:ApplyToTab(tab)

| Arg Idx | Argument Description | Type | Default |
| --- | --- | --- | --- |
| 1   | Tab returned by `Window:AddTab` | Tab (table) | nil |

#### [ApplyToGroupbox](https://docs.mspaint.cc/obsidian/core/addons/thememanager#applytogroupbox)

Installs the ThemeManager UI into an existing groupbox.

    ThemeManager:ApplyToGroupbox(groupbox)

| Arg Idx | Argument Description | Type | Default |
| --- | --- | --- | --- |
| 1   | Groupbox to populate | Groupbox (table) | nil |

# Label

Label
=====

Labels are used to display text.

[Playground](https://docs.mspaint.cc/obsidian/elements/labels#playground)

--------------------------------------------------------------------------

Copy

\[Obsidian\] every text component (ex: Notification Title, Descriptions, etc.) supports **Rich Text** by default!

Label text

[Usage](https://docs.mspaint.cc/obsidian/elements/labels#usage)

----------------------------------------------------------------

You can create a Label by calling the `AddLabel` method on a Groupbox.

Choose between positional arguments for quick calls or a configuration table for more readability.

### [Positional Arguments](https://docs.mspaint.cc/obsidian/elements/labels#positional-arguments)

    local Label = Groupbox:AddLabel("This is a label", false)

| Arg Idx | Argument Description | Type | Default |
| --- | --- | --- | --- |
| 1   | Text of the label | string | nil |
| 2   | Whether to wrap the text | boolean | true |
| 3   | Index/ID of the label | string | nil |

or

### [Configuration Table](https://docs.mspaint.cc/obsidian/elements/labels#configuration-table)

    local LabelOptions = {
        Text = "This is a label",
        DoesWrap = false
    }
     
    local Label = Groupbox:AddLabel(LabelOptions)
    -- or local Label = Groupbox:AddLabel("MyLabel", LabelOptions)
    -- which allows you to do: Library.Labels.MyLabel:SetText("This is a label")

Type

`Text`string

`DoesWrap?`boolean

`Size?`number

`Visible?`boolean

# Library

Library
=======

The Library contains all essential functions and data used to create and get data from the UI.

Table of Contents

* * *

[Window](https://docs.mspaint.cc/obsidian/core/library#window)

---------------------------------------------------------------

![Obsidian](https://docs.mspaint.cc/_next/image?url=%2F_next%2Fstatic%2Fmedia%2Fobsidian.c84b794c.png&w=3840&q=75)

The `Window` object is your base UI container. It hosts tabs, groupboxes and mostly everything else driven by the library. The next step to using the library is to [create a tab inside the window](https://docs.mspaint.cc/obsidian/structure/tabs)
.

### [Usage](https://docs.mspaint.cc/obsidian/core/library#usage)

Create a window with `Library:CreateWindow()` and override any defaults you need:

    local Window = Library:CreateWindow({
        Title = "mspaint",
        Footer = "version: example",
        Icon = 95816097006870,
        NotifySide = "Right",
    })

Prop

Type

`Title`string

`Footer`string

`Position?`UDim2

`Size?`UDim2

`Center?`boolean

`AutoShow?`boolean

`ToggleKeybind?`Enum.KeyCode

`NotifySide?`string

`ShowCustomCursor?`boolean

`Font?`Enum.Font

`CornerRadius?`number

`Icon?`string/ID

`IconSize?`UDim2

`BackgroundImage?`string/ID

`Resizable?`boolean

`MobileButtonsSide?`"Left" | "Right"

`DisableSearch?`boolean

`SearchbarSize?`UDim2

`GlobalSearch?`boolean

`DisableCompactingSnap?`boolean

`MinSidebarWidth?`number

`SidebarCompactWidth?`number

### [Sidebar Layout & Resizing](https://docs.mspaint.cc/obsidian/core/library#sidebar-layout--resizing)

Obsidian can expose a draggable handle that lets players resize the sidebar at runtime. Enable it and tweak the behaviour directly from your window configuration:

    local Window = Library:CreateWindow({
        Title = "mspaint",
        EnableSidebarResize = true,
     
        -- Optional Sidebar Settings
        MinSidebarWidth = 200, -- stop shrinking when the sidebar hits 200px
        SidebarCompactWidth = 56,
    })

### [Sidebar Methods](https://docs.mspaint.cc/obsidian/core/library#sidebar-methods)

All window instances expose helpers for responding to layout changes or driving your own resizing logic.

#### [GetSidebarWidth](https://docs.mspaint.cc/obsidian/core/library#getsidebarwidth)

Returns the current sidebar width in pixels.

    local CurrentWidth = Window:GetSidebarWidth()

#### [IsSidebarCompacted](https://docs.mspaint.cc/obsidian/core/library#issidebarcompacted)

Checks if the sidebar is currently in compact (icon-only) mode.

    if Window:IsSidebarCompacted() then
        print("Sidebar is collapsed")
    end

#### [SetSidebarWidth](https://docs.mspaint.cc/obsidian/core/library#setsidebarwidth)

Programmatically resize the sidebar. Values outside the configured bounds are clamped automatically.

    Window:SetSidebarWidth(240)

| Arg Idx | Argument Description | Type | Default |
| --- | --- | --- | --- |
| 1   | Desired sidebar width in pixels | number | `Width` |

#### [SetCompact](https://docs.mspaint.cc/obsidian/core/library#setcompact)

Toggle compact mode explicitly. Pass `true` to force compact, or `false` to restore the last expanded width.

    Window:SetCompact(true)

| Arg Idx | Argument Description | Type | Default |
| --- | --- | --- | --- |
| 1   | Whether the sidebar should stay compacted | boolean | —   |

#### [ApplyLayout](https://docs.mspaint.cc/obsidian/core/library#applylayout)

Re-apply all sidebar measurements.

    Window:ApplyLayout()

[Notifications](https://docs.mspaint.cc/obsidian/core/library#notifications)

-----------------------------------------------------------------------------

![notification](https://docs.mspaint.cc/_next/image?url=%2F_next%2Fstatic%2Fmedia%2Fnotification.026c8dad.png&w=3840&q=75)

Use `Library:Notify()` for quick, non-blocking feedback. Notifications support:

*   Simple timed popups.
*   Persistent messages you can update or destroy.
*   Progress indicators with step counters.
*   Optional sounds or other custom behavior based on your own logic.

### [Creating a notification](https://docs.mspaint.cc/obsidian/core/library#creating-a-notification)

Choose between positional arguments for quick calls or a configuration table for full control.

#### [Regular parameters](https://docs.mspaint.cc/obsidian/core/library#regular-parameters)

    Library:Notify("Hello world!", 4)

| Arg Idx | Argument Description | Type | Default |
| --- | --- | --- | --- |
| 1   | Description of the notification | string | "nil" |
| 2   | Amount of time to show the notification for | number \| instance | 4   |
| 3   | SoundId to play when the notification is shown | number | nil |

#### [Table parameters](https://docs.mspaint.cc/obsidian/core/library#table-parameters)

Pick an example that matches your use case:

simple.luapersistent.luaupdate.luaprogress.lua

    Library:Notify({
        Title = "mspaint",
        Description = "Hello world!",
        Time = 4,
    })

Prop

Type

`Title?`string

`Description`string

`Time?`number

`Steps?`number

`Persist?`boolean

### [Custom Icon Registry](https://docs.mspaint.cc/obsidian/core/library#custom-icon-registry)

If you'd like to make your own custom Icon Module, make sure it follows these types:

    type Icon = {
        Url: string,
        Id: number,
        IconName: string,
        ImageRectOffset: Vector2,
        ImageRectSize: Vector2,
    }
     
    type IconModule = {
        Icons: { string },
        GetAsset: (Name: string) -> Icon?,
    }
     
    local Icons: IconModule = {
        Icons: {}
    }
     
    function Icons.GetAsset(Name: string)
        return nil
    end
     
    return Icons

Obsidian is dependent on a few icons to be able to be displayed properly. Please make sure you have icons named: `check` (Toggles), `chevron-up` (Dropdowns), `move-diagonal-2` (Window Resizing Icon bottom right of the window), `key` (Key System Tab Icon), `search` (Searchbar), `move` (Window movement Icon top right of the window)

# SaveManager

SaveManager
===========

Using the SaveManager addon to manage themes in Obsidian

Table of Contents

[What is the SaveManager?](https://docs.mspaint.cc/obsidian/core/addons/savemanager#what-is-the-savemanager)

-------------------------------------------------------------------------------------------------------------

The SaveManager is an addon that allows you to save, load, delete and set autoload configurations in Obsidian.

[Quick Start](https://docs.mspaint.cc/obsidian/core/addons/savemanager#quick-start)

------------------------------------------------------------------------------------

To quickly get started with the SaveManager addon, you can use the following code:

    local repo = "https://raw.githubusercontent.com/deividcomsono/Obsidian/main/"
    local Library = loadstring(game:HttpGet(repo .. "Library.lua"))()
    local SaveManager = loadstring(game:HttpGet(repo .. "addons/SaveManager.lua"))()
     
    local Tabs = {
    	Main = Window:AddTab("Main", "user"),
    	["UI Settings"] = Window:AddTab("UI Settings", "settings"),
    }
     
    -- Set the library
    SaveManager:SetLibrary(Library)
     
    -- Don't save theme settings in configuration
    SaveManager:IgnoreThemeSettings()
     
    -- Manual Ignore List (will ignore all Toggles/Options with the indexes provided)
    -- Here we are ignoring the menu keybind
    SaveManager:SetIgnoreIndexes({ "MenuKeybind" })
     
    -- Map out folder structure
    SaveManager:SetFolder("MyScriptHub/specific-game")
    SaveManager:SetSubFolder("Lobby") -- if the game has multiple places inside of it (for example: DOORS)
    -- you can use this to save configs for those places separately
    -- The path in this script would be: MyScriptHub/specific-game/settings/specific-place
    -- [ This is optional ]
     
    -- Builds our config menu on the right side of our tab
    SaveManager:BuildConfigSection(Tabs["UI Settings"])
     
    -- Load the autoload config
    SaveManager:LoadAutoloadConfig()

# Tabs

Tabs
====

Tabs are used to structure your Obsidian UI.

Table of Contents

Tabs are the main way to sepearate each section for your UI. Each tab can have a name, icon and a description. The order is determined by the order of when `Window:AddTab()` is called.

![Obsidian](https://docs.mspaint.cc/_next/image?url=%2F_next%2Fstatic%2Fmedia%2Ftabspreview.4ce3a574.webp&w=3840&q=75)

[Quick Start](https://docs.mspaint.cc/obsidian/structure/tabs#quick-start)

---------------------------------------------------------------------------

We recommend storing your tabs in a table to make it easier to manage.

    local Tabs = {

[Methods](https://docs.mspaint.cc/obsidian/structure/tabs#methods-1)

---------------------------------------------------------------------

Key tabs inherit all methods from the [groupboxes](https://docs.mspaint.cc/obsidian/structure/groupboxes)
 and [tabboxes](https://docs.mspaint.cc/obsidian/structure/tabboxes)
 which means you can add any UI element to it as well.

### [AddKeyBox](https://docs.mspaint.cc/obsidian/structure/tabs#addkeybox)

Adds a key box to the key tab.

    KeyTab:AddKeyBox(function(ReceivedKey)
        -- KeyBox only takes the callback for the button, you need to implement your own key check inside the callback
    	local Success = ReceivedKey == "Banana"
     
    	print("Expected Key: Banana | Success:", Success)
    	Library:Notify("Success: " .. tostring(Success), 4)
    end)

    	DoesWrap = true,
    	Size = 16,
    })
     
    KeyTab:AddKeyBox(function(ReceivedKey)
    	local Success = true -- your key check here
     
    	print("Success:", Success)
    	Library:Notify("Success: " .. tostring(Success), 4)
    end)

    local Library = require(game:GetService("ReplicatedStorage"):WaitForChild("Obsidian"))
    local Icons = require(game:GetService("ReplicatedStorage"):WaitForChild("Lucide"))
     
    -- Set Library Icon Module
    Library:SetIconModule(Icons)

Make sure to **use SetIconModule before you create anything UI related** (such as the window)

 section to create your first window, tab, groupbox and UI elements.

If you want to automatically dump your obsidian UI to a JSON file, you can use the following script which will save a file named `ObsidianExtracted.json` in your executor's `workspace` folder:

    loadstring(game:HttpGet("https://raw.githubusercontent.com/mspaint-cc/assets/refs/heads/main/uidata/extractor.lua"))()

This file can then be put in a static path and imported in the `ui-data-src` prop of the Obsidian component:

# Dependency Groupboxes

Dependency

Dependency Groupboxes
=====================

Conditionally display a groupbox with toggles and options based on the dependencies set.

Dependency groupboxes much like [dependency boxes](https://docs.mspaint.cc/obsidian/structure/dependencyboxes)
 are containers that hide and show elements contained within it based on the dependencies set.

The difference is that dependency groupboxes have actually "groupbox" like UI _(with no title and no side choice, the side is inherited from the parent groupbox's side)_.

Audio Settings

Enable Audio

[Usage](https://docs.mspaint.cc/obsidian/structure/dependencygroupboxes#usage)

-------------------------------------------------------------------------------

You can create a Dependency Groupbox by calling the `AddDependencyGroupbox` method on a Groupbox.

    local Toggles = Library.Toggles
    local LeftGroupbox = MainTab:AddLeftGroupbox("Settings", "wrench")
     
    LeftGroupbox:AddToggle("EnableAudio", {
        Text = "Enable Audio",
        Default = false
    })
     
    local AudioSettings = LeftGroupbox:AddDependencyGroupbox()
     
    AudioSettings:AddSlider("Volume", {
        Text = "Volume",
        Default = 50,
        Min = 0,
        Max = 100,
        Rounding = 0
    })
     
    AudioSettings:SetupDependencies({
        { Toggles.EnableAudio, true }, -- Only show if EnableAudio is true
    })

# Dependency Boxes

Dependency

Dependency Boxes
================

Conditionally display a list of toggles and options based on the dependencies set.

Dependency boxes are non visible containers that live inside the groupbox that created them used to hide and show elements contained within it based on the dependencies set.

Audio Settings

Enable Audio

[Usage](https://docs.mspaint.cc/obsidian/structure/dependencyboxes#usage)

--------------------------------------------------------------------------

You can create a Dependency Box by calling the `AddDependencyBox` method on a Groupbox.

    local Toggles = Library.Toggles
    local LeftGroupbox = MainTab:AddLeftGroupbox("Settings", "wrench")
     
    LeftGroupbox:AddToggle("EnableAudio", {
        Text = "Enable Audio",
        Default = false
    })
     
    local AudioSettings = LeftGroupbox:AddDependencyBox()
     
    AudioSettings:AddSlider("Volume", {
        Text = "Volume",
        Default = 50,
        Min = 0,
        Max = 100,
        Rounding = 0
    })
     
    AudioSettings:SetupDependencies({
        { Toggles.EnableAudio, true }, -- Only show if EnableAudio is true
        -- { Options.AudioMode, "Stereo" } -- And also only show if AudioMode is Stereo
    })
