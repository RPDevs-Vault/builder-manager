# Vector Engine Documentation | Vector Lua Engine

Welcome to the **Vector Engine** scripting documentation. The Vector Engine is a powerful Lua scripting system that powers Project Vector's script execution layer, providing direct memory access, rendering, and game automation capabilities.

* * *

### 

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine#what-is-vector-engine)

What is Vector Engine?

Vector Engine is the scripting core of Project Vector, providing:

*   **Direct Memory Access** - Read and write game memory
    
*   **High Performance** - LuaJIT compilation with custom optimizations
    
*   **Complete Game Access** - Full instance hierarchy and property control
    
*   **Screen Rendering** - ImGui-based overlay system for ESP, HUD, menus
    
*   **Async Execution** - Background threading without frame blocking
    
*   **Real-time Scripting** - Per-frame callbacks for instant feedback
    

It's the framework that enables Project Vector's advanced features through Lua scripting.

Complete Example: Advanced ESP with Health

Copy

    function on_frame()
        local me = entity.get_local_player()
        if not me then return end
        
        -- Draw ESP for each player
        for _, player in ipairs(entity.get_players()) do
            if player.IsAlive then
                local bounds = player:get_bounds()
                if bounds.valid then
                    -- Draw bounding box
                    draw.box(bounds.x, bounds.y, bounds.w, bounds.h, {1, 0, 0, 1})
                    
                    -- Draw player name CENTERED at top
                    local name_width = draw.get_text_size(player.Name, 12)
                    local name_x = bounds.x + (bounds.w - name_width) / 2
                    draw.text(name_x, bounds.y - 15, player.Name, {1, 1, 1, 1}, 12)
                    
                    -- Draw distance
                    local pos = player.Position
                    local my_pos = me.Position
                    local dx = pos.X - my_pos.X
                    local dy = pos.Y - my_pos.Y
                    local dz = pos.Z - my_pos.Z
                    local dist = math.sqrt(dx*dx + dy*dy + dz*dz)
                    
                    -- Draw distance CENTERED at bottom
                    local dist_text = math.floor(dist) .. "m"
                    local dist_width = draw.get_text_size(dist_text, 11)
                    local dist_x = bounds.x + (bounds.w - dist_width) / 2
                    draw.text(dist_x, bounds.y + bounds.h + 5, dist_text, {1, 1, 1, 1}, 11)
                    
                    -- Draw health bar on LEFT side
                    draw.health_bar(bounds.x, bounds.y, bounds.h, player.Health, player.MaxHealth)
                end
            end
        end
    end

# Thread API Documentation | Vector Lua Engine

The Thread API allows you to run code asynchronously in the background at specified intervals. Threads run independently from the main rendering loop, allowing you to perform periodic tasks without blocking the frame.

* * *

### 

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/api/thread-api-documentation#overview)

Overview

**What the Thread API provides:**

*   Run callbacks at specified intervals
    
*   Background execution (doesn't block rendering)
    
*   Manage multiple concurrent threads
    
*   Control thread lifecycle
    
*   Dynamic interval adjustment
    

* * *

### 

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/api/thread-api-documentation#core-functions)

Core Functions

#### 

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/api/thread-api-documentation#thread.create)

thread.Create()

Create a new background thread with a callback function.

Copy

    thread_id = thread.Create(callback_function, interval_ms)

**Parameters:**

*   `callback_function` (function) - Function to call repeatedly
    
*   `interval_ms` (number, optional) - Interval in milliseconds, default: 100, minimum: 1
    

**Returns:**

*   `thread_id` (number) - Unique thread ID for control
    

**Example:**

Copy

    function on_frame()
        -- Create a thread that runs every 500ms
        local my_thread = thread.Create(function()
            print("Thread running!")
        end, 500)
        
        print("Thread created with ID: " .. my_thread)
    end

* * *

#### 

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/api/thread-api-documentation#thread.stop)

thread.Stop()

Stop a specific thread by ID.

Copy

    thread.Stop(thread_id)

**Parameters:**

*   `thread_id` (number) - ID returned from thread.Create()
    

**Example:**

Copy

    function on_frame()
        -- Create thread
        local my_thread = thread.Create(function()
            print("Periodic task")
        end, 1000)
        
        -- Stop it later
        thread.Stop(my_thread)
        print("Thread stopped")
    end

**Parameters:**

*   `thread_id` (number) - ID to check

Example 5: Thread Management

Copy

    function on_frame()
        -- Create multiple worker threads
        local workers = {}
        
        for i = 1, 3 do
            local worker = thread.Create(function()
                print("Worker " .. i .. " active")
            end, 1000 * i)
            
            table.insert(workers, worker)
        end
        
        -- Check status
        for i, worker_id in ipairs(workers) do
            if thread.IsRunning(worker_id) then
                print("Worker " .. i .. " is running")
            end
        end
        
        -- Stop specific worker
        if #workers > 0 then
            thread.Stop(workers[1])
            print("Stopped first worker")
        end
    end

Important Notes

#### 

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/api/thread-api-documentation#thread-safety)

Thread Safety

*   Threads run in background without blocking rendering
    
*   Multiple threads can run simultaneously
    
*   Safe for reading game state
    
*   Each thread has its own execution context
    

#### 

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/api/thread-api-documentation#performance)

Performance

*   Intervals are in milliseconds (1000 = 1 second)
    
*   Minimum interval is 1ms (not recommended)
    
*   Smaller intervals = higher CPU usage
    
*   Choose appropriate intervals for your task
    

#### 

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/api/thread-api-documentation#best-practices-for-threads)

Best Practices for Threads

*   Use threads for background monitoring, not frequent updates
    
*   Don't use threads for real-time ESP or aimbot (use on\_frame instead)
    
*   Call thread.StopAll() when scripts unload
    
*   Check if objects exist before accessing in threads
    
*   Handle errors gracefully in thread callbacks
    

#### 

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/api/thread-api-documentation#when-to-use-threads-vs-on_frame)

When to Use Threads vs on\_frame

**Use on\_frame for:**

*   Rendering (drawing)
    
*   Real-time input handling
    
*   Frame-dependent logic
    

**Use threads for:**

*   Periodic data collection
    
*   Background monitoring
    
*   Heavy computations that don't need to run every frame
    
*   Scheduled tasks

# Entity API Documentation | Vector Lua Engine

The Entity API provides safe, cached access to player objects and their game data. All functions support **both** `**snake_case**` **and** `**camelCase**` naming conventions.

* * *

### 

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/api/entity-api-documentation#overview)

Overview

**What the Entity API provides:**

*   Player enumeration and filtering
    
*   Cached player data (updated every 500ms)
    
*   Live position/health readings
    
*   Bone/part screen-space projections
    
*   Character hierarchy access
    
*   Both R6 and R15 rig support
    
*   Bounding box calculations
    
*   Distance calculations
    

* * *

### 

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/api/entity-api-documentation#core-functions)

Core Functions

#### 

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/api/entity-api-documentation#entity.get_players-entity.getplayers)

entity.get\_players() / entity.GetPlayers()

Get all valid players currently in game.

Copy

    -- Both work identically
    local players = entity.get_players()      -- snake_case
    local players = entity.GetPlayers()       -- camelCase

**Returns:** `table` - Array of player objects (1-indexed)

**Example:**

Copy

    function on_frame()
        local players = entity.get_players()
        print("Players in game: " .. #players)
        
        for i, player in ipairs(players) do
            print(i .. ": " .. player.Name .. " (" .. player.Team .. ")")
        end
    end

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/api/entity-api-documentation#live-properties)

Live Properties

Read fresh from memory every access:

Copy

    player.Health            -- number: Current HP
    player.MaxHealth         -- number: Maximum HP
    player.IsAlive           -- boolean: Health > 0
    player.Position          -- Vector3: Root part world position
    player.Velocity          -- Vector3: Linear velocity
    player.HeadPosition      -- Vector3: Head world position

**Example:**

Copy

    function on_frame()
        local me = entity.get_local_player()
        if not me then return end
        
        for _, player in ipairs(entity.get_players()) do
            if player.IsAlive then
                local health = player.Health
                local pos = player.Position
                
                local dx = me.Position.X - pos.X
                local dy = me.Position.Y - pos.Y
                local dz = me.Position.Z - pos.Z
                local dist = math.sqrt(dx*dx + dy*dy + dz*dz)
                
                print(player.Name .. ": " .. math.floor(health) .. "HP @ " .. math.floor(dist) .. "m")
            end
        end
    end

# Menu API Documentation | Vector Lua Engine

The Menu API allows you to create professional, interactive user interfaces with tabs, groups, and customizable elements. All UI elements support callbacks, conditional visibility, and dynamic updates.

### 

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/api/menu-api-documentation#overview)

Overview

The Menu system is organized hierarchically:

*   **Tabs** - Top-level containers (full or half width)
    
*   **Groups** - Organize elements within tabs
    
*   **Elements** - Individual UI components (checkboxes, sliders, buttons, etc.)
    

Elements are accessed by unique IDs and support real-time value reading/writing and change callbacks.

* * *

### 

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/api/menu-api-documentation#tabs)

Tabs

Tabs are the main navigation containers at the top level.

#### 

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/api/menu-api-documentation#add_tab-name-icon-size)

add\_tab(name, icon, size)

Create a new tab in the menu.

**Parameters:**

*   `name` (string) - Tab name
    
*   `icon` (string) - Single character icon
    
*   `size` (string) - "full" (default) or "half" (side-by-side layout)
    

**Example:**

Copy

    menu.add_tab("Main", "M", "full")
    menu.add_tab("Settings", "S", "half")
    menu.add_tab("Advanced", "A", "half")

* * *

### 

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/api/menu-api-documentation#groups)

Groups

Groups organize elements within a tab. Create multiple groups to section your UI logically.

#### 

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/api/menu-api-documentation#add_group-tab-name-width-same_line)

add\_group(tab, name, width, same\_line)

Create a group in a tab.

**Parameters:**

*   `tab` (string) - Tab name to add group to
    
*   `name` (string) - Group name
    
*   `width` (number, optional) - Custom width (0 = auto)
    
*   `same_line` (boolean, optional) - Render on same line as previous group
    

**Example:**

Copy

    menu.add_group("Main", "ESP", 0, false)
    menu.add_group("Main", "Aimbot", 0, false)
    menu.add_group("Main", "Settings", 200, true)

* * *

### 

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/api/menu-api-documentation#checkbox)

Checkbox

Simple boolean toggle with optional inline color picker and hotkey.

#### 

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/api/menu-api-documentation#add_checkbox-tab-group-id-label-default-options)

add\_checkbox(tab, group, id, label, default, options)

Create a checkbox element.

**Parameters:**

*   `tab` (string) - Tab name
    
*   `group` (string) - Group name
    
*   `id` (string) - Unique identifier
    
*   `label` (string) - Display label
    
*   `default` (boolean) - Default value
    
*   `options` (table, optional) - Configuration table
    

**Options:**

*   `colorpicker` (table) - Enable inline color picker `{r, g, b, a}`
    
*   `key` (number) - VK code for attached hotkey
    
*   `parent` (string) - Parent element ID (conditional visibility)
    
*   `show_mode` (boolean) - Show hotkey mode selector (default: true)
    

**Returns:** Element ID (string)

**Example:**

Copy

    -- Simple checkbox
    menu.add_checkbox("Main", "ESP", "esp_enabled", "Enable ESP", false)
    
    -- Checkbox with inline color picker
    menu.add_checkbox("Main", "ESP", "npc_esp", "NPC ESP", true, {
        colorpicker = {1, 0, 0, 1}  -- Red color picker
    })
    
    -- Checkbox with hotkey
    menu.add_checkbox("Main", "Aimbot", "aimbot_toggle", "Aimbot", false, {
        key = 0x02,  -- Right mouse button
        show_mode = true
    })
    
    -- Conditional checkbox (only visible when parent is enabled)
    menu.add_checkbox("Main", "ESP", "esp_detailed", "Show Details", true, {
        parent = "esp_enabled"
    })

**Getting/Setting Values:**

Copy

    local enabled = menu.get("esp_enabled")  -- Returns: boolean
    menu.set("esp_enabled", true)
    
    -- Get/set attached hotkey
    local key = menu.get_key("aimbot_toggle")  -- Returns: VK code
    menu.set_key("aimbot_toggle", 0x4D)  -- M key
    
    -- Get/set color (if colorpicker enabled)
    local color = menu.get_color("npc_esp")  -- Returns: {r, g, b, a}
    menu.set_color("npc_esp", {0, 1, 0, 1})  -- Green

**Callback:**

Copy

    menu.set_callback("esp_enabled", function(value)
        if value then
            print("ESP enabled")
        else
            print("ESP disabled")
        end
    end)

# Utility API Documentation | Vector Lua Engine

The Utility API provides essential functions for screen conversion, input handling, and timing. These utilities enable you to interact with screen coordinates, retrieve timing information, and manage frame-rate independent calculations without blocking the rendering loop.

* * *

### 

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/api/utility-api-documentation#overview)

Overview

**What the Utility API provides:**

*   Convert 3D world coordinates to 2D screen coordinates
    
*   Get screen and window dimensions
    
*   Retrieve current mouse position
    
*   Access frame timing information
    
*   Monitor frames per second
    
*   Manage frame-rate independent calculations
    

* * *

### 

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/api/utility-api-documentation#utility.world_to_screen-x-y-z)

utility.world\_to\_screen(x, y, z)

Convert 3D world coordinates to 2D screen coordinates. Both utility.world\_to\_screen() and utility.WorldToScreen() are supported.

Copy

    local sx, sy, visible = utility.world_to_screen(100, 50, -200)
    if visible then
        draw.text(sx, sy, "Target!", {1, 1, 1, 1}, 14)
    end

**Parameters:**

*   `x` (number) - World X coordinate
    
*   `y` (number) - World Y coordinate
    
*   `z` (number) - World Z coordinate
    

**Returns:**

*   `screen_x` (number) - Screen X position in pixels
    
*   `screen_y` (number) - Screen Y position in pixels
    
*   `on_screen` (boolean) - Whether the point is visible on screen
    

**Notes:**

*   Returns screen coordinates even if off-screen (useful for arrow indicators)
    
*   Coordinates are relative to game window (0,0 = top-left)
    
*   Supports Vector3 objects as single parameter: `utility.world_to_screen(vector3_object)`
    

* * *

### 

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/api/utility-api-documentation#utility.get_screen_size)

utility.get\_screen\_size()

Get the current screen or window dimensions. Both utility.get\_screen\_size() and utility.GetScreenSize() are supported.

Copy

    local w, h = utility.get_screen_size()
    print("Resolution: " .. w .. "x" .. h)
    
    -- Center of screen
    local cx, cy = w / 2, h / 2
    draw.circle(cx, cy, 5, {1, 1, 1, 1}, 16)

**Parameters:** None

**Returns:**

*   `width` (number) - Screen width in pixels
    
*   `height` (number) - Screen height in pixels
    

**Notes:**

*   Updates automatically when window is resized
    
*   Useful for relative positioning and adaptive layouts
    

* * *

### 

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/api/utility-api-documentation#utility.get_mouse_pos)

utility.get\_mouse\_pos()

Get the current mouse position in screen coordinates. Both utility.get\_mouse\_pos() and utility.GetMousePos() are supported.

Copy

    local mx, my = utility.get_mouse_pos()
    draw.circle(mx, my, 3, {1, 0, 0, 1}, 16)

**Parameters:** None

**Returns:**

*   `x` (number) - Mouse X position in pixels
    
*   `y` (number) - Mouse Y position in pixels
    

**Notes:**

*   Position is relative to screen (0,0 = top-left)
    
*   Updated every frame
    
*   Requires window focus to work properly
    

* * *

### 

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/api/utility-api-documentation#utility.get_delta_time)

utility.get\_delta\_time()

Get the time elapsed since the last frame in seconds. Both utility.get\_delta\_time() and utility.GetDeltaTime() are supported.

Copy

    local dt = utility.get_delta_time()
    local speed = 100  -- pixels per second
    local distance = speed * dt
    player.x = player.x + distance

**Parameters:** None

**Returns:**

*   `delta_time` (number) - Time in seconds (clamped 0.0001s - 0.1s)
    

**Notes:**

*   Automatically clamped to prevent extreme values
    
*   Essential for frame-rate independent movement and animations
    
*   Use for smooth, consistent behavior across different frame rates
    

* * *

### 

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/api/utility-api-documentation#utility.get_tick_count)

utility.get\_tick\_count()

Get the system tick count in milliseconds since boot. Both utility.get\_tick\_count() and utility.GetTickCount() are supported.

Copy

    local start = utility.get_tick_count()
    
    -- ... perform action ...
    
    local elapsed = utility.get_tick_count() - start
    print("Operation took: " .. elapsed .. "ms")

**Parameters:** None

**Returns:**

*   `ticks` (number) - Milliseconds since system boot
    

**Notes:**

*   Returns 64-bit value (won't overflow for practical use)
    
*   Millisecond precision, independent of frame rate
    
*   Better for cooldowns and fixed intervals than delta\_time
    

* * *

### 

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/api/utility-api-documentation#utility.get_fps)

utility.get\_fps()

Get the current frames per second (smoothed average). Both utility.get\_fps() and utility.GetFPS() are supported.

Copy

    local fps = utility.get_fps()
    draw.text(10, 10, "FPS: " .. math.floor(fps), {1, 1, 1, 1}, 12)
    
    -- Adapt quality based on performance
    if fps < 60 then
        disable_expensive_features()
    end

**Parameters:** None

**Returns:**

*   `fps` (number) - Frames per second (float, smoothed over 60 frames)
    

**Notes:**

*   Smoothed to prevent jitter (averaged over 1 second at 60 FPS)
    
*   Useful for performance monitoring and adaptive quality
    
*   Not instantaneous due to smoothing

Common Usage Patterns

**Frame-Rate Independent Movement:**

Copy

    function update_position(object, velocity)
        local dt = utility.get_delta_time()
        object.x = object.x + velocity.x * dt
        object.y = object.y + velocity.y * dt
    end

**Cooldown System:**

Copy

    local last_action = 0
    
    function can_act(cooldown_ms)
        local now = utility.get_tick_count()
        return now - last_action > cooldown_ms
    end
    
    function perform_action()
        if can_act(500) then
            last_action = utility.get_tick_count()
            -- execute action
        end
    end

**Screen-Based Targeting:**

Copy

    function on_frame()
        local w, h = utility.get_screen_size()
        local mx, my = utility.get_mouse_pos()
        
        -- Screen center
        local cx, cy = w / 2, h / 2
        
        -- Mouse distance from center
        local dx, dy = mx - cx, my - cy
        local dist = math.sqrt(dx*dx + dy*dy)
        
        if dist < 100 then
            draw.circle(cx, cy, 100, {1, 1, 1, 0.2}, 64)
        end
    end

**World to Screen Rendering:**

Copy

    function on_frame()
        -- Example world positions
        local targets = {
            {x = 0, y = 5, z = 10},
            {x = 50, y = 0, z = -20},
            {x = -30, y = 15, z = 5}
        }
        
        for _, target in ipairs(targets) do
            local sx, sy, visible = utility.world_to_screen(target.x, target.y, target.z)
            
            if visible then
                draw.circle(sx, sy, 5, {0, 1, 0, 1}, 16)
                draw.text(sx + 10, sy, "Target", {0, 1, 0, 1}, 12)
            end
        end
    end

    function on_frame()
        local workspace = game.Workspace
        
        -- Get actual children first to see what exists
        local children = workspace:GetChildren()

            -- Use the first child's name to test FindFirstChild

# Draw API Documentation | Vector Lua Engine

The Draw API provides real-time rendering functions for drawing shapes, text, and UI elements directly on screen. All drawing happens in world space with automatic coordinate validation.

* * *

### 

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/api/draw-api-documentation#overview)

Overview

The Draw system renders to ImGui's background draw list:

*   All coordinates are in **screen space** (pixel coordinates)
    
*   Colors are RGBA with values **0.0-1.0** (not 0-255)
    
*   Automatic black outlines for readability
    
*   Validated coordinates prevent crashes
    

* * *

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/api/draw-api-documentation#line)

Line

Draw a line between two points.

Copy

    draw.line(x1, y1, x2, y2, color, thickness)

**Parameters:**

*   `x1, y1` (number) - Starting point
    
*   `x2, y2` (number) - Ending point
    
*   `color` (table) - RGBA color `{r, g, b, a}` (0.0-1.0)
    
*   `thickness` (number, optional) - Line width, default: 1.0
    

**Example:**

Copy

    function on_frame()
        -- Simple line
        draw.line(100, 100, 500, 200, {1, 0, 0, 1})
        
        -- Thick line
        draw.line(100, 100, 500, 200, {0, 1, 0, 1}, 3.0)
        
        -- Crosshair at screen center
        local sw, sh = draw.get_screen_size()
        local cx, cy = sw / 2, sh / 2
        local size = 20
        draw.line(cx - size, cy, cx + size, cy, {1, 1, 1, 1}, 1.5)
        draw.line(cx, cy - size, cx, cy + size, {1, 1, 1, 1}, 1.5)
    end

# Vector Engine Documentation | Vector Lua Engine

Welcome to the **Vector Engine** scripting documentation. The Vector Engine is a powerful Lua scripting system that powers Project Vector's script execution layer, providing direct memory access, rendering, and game automation capabilities.

* * *

### 

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/introduction#what-is-vector-engine)

What is Vector Engine?

Vector Engine is the scripting core of Project Vector, providing:

*   **Direct Memory Access** - Read and write game memory
    
*   **High Performance** - LuaJIT compilation with custom optimizations
    
*   **Complete Game Access** - Full instance hierarchy and property control
    
*   **Screen Rendering** - ImGui-based overlay system for ESP, HUD, menus
    
*   **Async Execution** - Background threading without frame blocking
    
*   **Real-time Scripting** - Per-frame callbacks for instant feedback
    

It's the framework that enables Project Vector's advanced features through Lua scripting.

# Thread API Documentation | Vector Lua Engine

The Thread API allows you to run code asynchronously in the background at specified intervals. Threads run independently from the main rendering loop, allowing you to perform periodic tasks without blocking the frame.

* * *

### 

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/api#overview)

Overview

**What the Thread API provides:**

*   Run callbacks at specified intervals
    
*   Background execution (doesn't block rendering)
    
*   Manage multiple concurrent threads
    
*   Control thread lifecycle
    
*   Dynamic interval adjustment
    

* * *

### 

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/api#core-functions)

Core Functions

#### 

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/api#thread.create)

thread.Create()

Create a new background thread with a callback function.

Copy

    thread_id = thread.Create(callback_function, interval_ms)

**Parameters:**

*   `callback_function` (function) - Function to call repeatedly
    
*   `interval_ms` (number, optional) - Interval in milliseconds, default: 100, minimum: 1
    

**Returns:**

*   `thread_id` (number) - Unique thread ID for control
    

**Example:**

Copy

    function on_frame()
        -- Create a thread that runs every 500ms
        local my_thread = thread.Create(function()
            print("Thread running!")
        end, 500)
        
        print("Thread created with ID: " .. my_thread)
    end

thread.Stop()

Stop a specific thread by ID.

    thread.Stop(thread_id)

**Parameters:**

*   `thread_id` (number) - ID returned from thread.Create()

Important Notes

#### 

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/api#thread-safety)

Thread Safety

*   Threads run in background without blocking rendering
    
*   Multiple threads can run simultaneously
    
*   Safe for reading game state
    
*   Each thread has its own execution context
    

#### 

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/api#performance)

Performance

*   Intervals are in milliseconds (1000 = 1 second)
    
*   Minimum interval is 1ms (not recommended)
    
*   Smaller intervals = higher CPU usage
    
*   Choose appropriate intervals for your task
    

#### 

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/api#best-practices-for-threads)

Best Practices for Threads

*   Use threads for background monitoring, not frequent updates
    
*   Don't use threads for real-time ESP or aimbot (use on\_frame instead)
    
*   Call thread.StopAll() when scripts unload
    
*   Check if objects exist before accessing in threads
    
*   Handle errors gracefully in thread callbacks
    

#### 

[hashtag](https://project-vector-1.gitbook.io/vector-lua-engine/api#when-to-use-threads-vs-on_frame)

When to Use Threads vs on\_frame

**Use on\_frame for:**

*   Rendering (drawing)
    
*   Real-time input handling
    
*   Frame-dependent logic
    

**Use threads for:**

*   Periodic data collection
    
*   Background monitoring
    
*   Heavy computations that don't need to run every frame
    
*   Scheduled tasks

            end, 5000)  -- Every 5 seconds
        end
