Integrate Circle’s MCP server to let your LLM or AI-assisted IDE generate and fix code for crypto apps using Circle’s offerings, including Wallets, Contracts, CCTP, Bridge Kit, and Gateway.

[​](https://developers.circle.com/ai/mcp#quick-setup)

Quick setup
--------------------------------------------------------------------

Add Circle’s MCP server to your client with the following details:

*   **Server Name**: `circle`
*   **Server URL**: `https://api.circle.com/v1/codegen/mcp`

[​](https://developers.circle.com/ai/mcp#installation-with-your-ide)

Installation with your IDE
--------------------------------------------------------------------------------------------------

Select your MCP client to view detailed setup instructions:

> **Note:** If you are using a client that is not listed here, you can still use the Circle MCP server by manually adding the server URL to your client’s configuration.

*   Cursor
    
*   Claude Code
    
*   Windsurf
    
*   Kiro
    

Download and install [Cursor](https://cursor.com/)
 if you haven’t already.[1-Click Set Up](cursor://anysphere.cursor-deeplink/mcp/install?name=circle&config=eyJ1cmwiOiJodHRwczovL2FwaS5jaXJjbGUuY29tL3YxL2NvZGVnZW4vbWNwIn0%3D)
Manual steps:

1.  Open a project in Cursor and navigate to **Cursor Settings**.
2.  In the settings menu, go to the **MCP** section.
3.  Click **New MCP Server**. This will open your `mcp.json` configuration file.
4.  Add the following configuration:
    
    Report incorrect code
    
    Copy
    
        {
          "mcpServers": {
            "circle": {
              "url": "https://api.circle.com/v1/codegen/mcp"
            }
          }
        }
        
    
5.  Return to the MCP settings page and enable the server using the toggle switch next to `circle`. Start generating code for Circle Wallets, Contracts, CCTP, and Gateway.

For more information on how to use MCP with Cursor, see the [Cursor MCP documentation](https://cursor.com/docs/context/mcp)
.

Download and install [Claude Code](https://docs.claude.com/en/docs/claude-code/overview)
 if you haven’t already.

1.  Using the Claude Code command line add the MCP server with the following command:

Report incorrect code

Copy

    claude mcp add --transport http circle https://api.circle.com/v1/codegen/mcp --scope user
    

2.  Verify the server is added by running the following command:

Report incorrect code

Copy

    claude mcp get circle
    

3.  Start generating code for Circle Wallets, Contracts, CCTP, and Gateway.

For more information on how to use MCP with Claude Code, see the [Claude Code MCP documentation](https://docs.claude.com/en/docs/claude-code/mcp)
.

Download and install [Windsurf](https://docs.windsurf.com/windsurf/getting-started)
 if you haven’t already.

1.  Open the `~/.codeium/windsurf/mcp_config.json` file and add the following:
    
    Report incorrect code
    
    Copy
    
        {
          "mcpServers": {
            "circle": {
              "url": "https://api.circle.com/v1/codegen/mcp"
            }
          }
        }
        
    
2.  Enable the Circle MCP server in your MCP settings. Start generating code for Circle Wallets, Contracts, CCTP, and Gateway.

For more information on how to use MCP with Windsurf, see the [Windsurf MCP documentation](https://docs.windsurf.com/windsurf/cascade/mcp)
.

Download and install [Kiro](https://kiro.dev/)
 if you haven’t already.

1.  Open Kiro and go to **Preferences/Settings** → search for “MCP” → enable MCP support.
2.  Create or open the MCP config file:
    *   **Workspace-level**: `./.kiro/settings/mcp.json` (recommended for project-specific)
    *   **User-level**: `~/.kiro/settings/mcp.json`
3.  Add a server configuration entry for Circle:
    
    Report incorrect code
    
    Copy
    
        {
          "mcpServers": {
            "circle": {
              "command": "npx",
              "args": ["-y", "@circle/mcp-server"],
              "env": {
                "CIRCLE_BASE_URL": "https://api.circle.com/v1/codegen/mcp"
              },
              "disabled": false
            }
          }
        }
        
    
4.  Save and then restart Kiro (or reload the MCP server list) so the new server appears in the MCP tab.
5.  In Kiro’s side panel → **MCP Servers** tab → you should see “circle-mcp” listed. Start generating code for Circle Wallets, Contracts, CCTP, and Gateway.

Kiro’s general MCP setup: [Kiro Docs - MCP](https://kiro.dev/docs/mcp/)
.

#### Path Parameters

[​](https://developers.circle.com/api-reference/cctp/all/get-attestation#parameter-message-hash)

messageHash

string

required

Message hash for the message being bridged. This can be generated using the `keccak256` hash of the message bytes emitted by the `MessageSent` event.

Example:

`"0x912f22a13e9ccb979b621500f6952b2afd6e75be7eadaed93fc2625fe11c52a2"`

#### Response

200

application/json

Successfully retrieved either the signed attestation or the message is still pending block confirmations.

Signed or pending attestation.

[​](https://developers.circle.com/api-reference/cctp/all/get-attestation#response-status)

status

enum<string>

required

Status of the attestation, whether it is signed or awaiting more block confirmations.

Available options:

`complete`,

`pending_confirmations`

[​](https://developers.circle.com/api-reference/cctp/all/get-attestation#response-attestation-one-of-0)

attestation

string | null

Signed attestation corresponding to the given `messageHash` parameter. This is null if the event has been seen but the attestation is still pending block confirmations.

Example:

`"0x6edd90f4a0ad0212fd9fbbd5058a25aa8ee10ce77e4fc143567bbe73fb6e164f384a3e14d350c8a4fc50b781177297e03c16b304e8d7656391df0f59a75a271f1b"`
