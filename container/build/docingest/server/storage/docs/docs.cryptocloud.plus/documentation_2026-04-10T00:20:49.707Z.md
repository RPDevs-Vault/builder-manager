# Авторизация запросов | CryptoCloud

[hashtag](https://docs.cryptocloud.plus/ru/api-reference-v2/#zagolovok-authorization)

Заголовок Authorization

Для авторизации каждого запроса к API в заголовке `Authorization` необходимо передать ваш API KEY следующим образом:

Copy

    Authorization: Token <API KEY>

# Принцип работы интеграции | CryptoCloud

Интеграция с CryptoCloud предполагает взаимодействие вашей платформы с нашим API для автоматизации финансовых операций. Вот как это работает:

1.  **Авторизация запросов:**
    
    *   Все запросы к API должны быть авторизованы с использованием ваших API ключей.
        
    
2.  **Создание счета:**
    
    *   Вы отправляете запрос на создание счета, указывая необходимые параметры и сумму.
        
    
3.  **Получение ссылки или реквизитов на оплату:**
    
    *   В ответ на ваш запрос система генерирует и отправляет вам уникальную ссылку или реквизиты на оплату.
        
    
4.  **Оплата пользователем:**
    
    *   Ваш пользователь производит оплату по ссылке или указанным реквизитам.
        
    
5.  **Уведомление о поступлении средств:**
    
    *   После поступления средств на счет, мы отправляем POSTBACK на URL для уведомлений, указанный в настройках вашего проекта.
        
    *   Вы должны обработать это уведомление и, при необходимости, предпринять соответствующие действия на стороне вашей платформы.
        
    
6.  **Проверка счета:**
    
    *   Вы можете самостоятельно проверять статус счета, используя его INV номер через соответствующий запрос к API.
        
    

Этот процесс обеспечивает безопасное и эффективное взаимодействие между вашей платформой и сервисом CryptoCloud, позволяя автоматизировать процесс приема платежей и управления счетами.

# Получение API ключей | CryptoCloud

Для работы с API CryptoCloud вам потребуется получить уникальные API ключи. Эти ключи идентифицируют ваш проект и обеспечивают безопасное взаимодействие между вашей платформой и сервисом CryptoCloud. Вот как вы можете получить свои API ключи:

1.  **Вход в личный кабинет:**
    
    *   Зарегистрируйтесь и войдите в свой личный кабинет на платформе CryptoCloud.
        
    
2.  **Создание нового проекта:**
    
    *   Перейдите в раздел «Мои проекты» в левом меню.
        
    *   Нажмите кнопку «Создать новый проект».
        
    *   В пункте «Как вы хотите принимать платежи» выберите «На своем сайте».
        
    *   Заполните все необходимые параметры.
        
    
3.  **Получение ключей:**
    
    *   После создания проекта перейдите в его настройки.
        
    *   В правой части экрана найдите раздел с вашим API KEY, SHOP ID и SECRET KEY.

# Тестирование интеграции | CryptoCloud

После добавления для вашего проекта автоматически включается тестовый режим. Он необходим для проверки корректности интеграции.

Текущий режим работы проекта можно отслеживать в разделе «Мои проекты».

circle-info

При включенном тестовом режиме вы можете создавать счета любым доступным способом и подтверждать их без оплаты. Данные счета не будут влиять на ваш баланс.

1.  Для тестирования интеграции необходимо создать счет, перейдя к оплате на вашем сайте или другой платформе, с которой вы проводили интеграцию. Создавая счет вручную в разделе «Платежи», протестировать интеграцию с вашей платформой не получится.
    

![](https://docs.cryptocloud.plus/ru/~gitbook/image?url=https%3A%2F%2F1986541005-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252F5U1THZAib81UsHLz0L7L%252Fuploads%252FkrDMn4ReB4OxjE0nwIhC%252Ftest%2520payment%25201.jpg%3Falt%3Dmedia%26token%3D068e6969-de57-4c25-b1c4-d40c57f693ae&width=768&dpr=3&quality=100&sign=7e114324&sv=2)

1.  Для подтверждения тестового счета и отправки уведомления об оплате необходимо перейти в раздел «Платежи», включить тумблер «Показать тестовые счета», найти нужный счет и кликнуть по нему.
    

![](https://docs.cryptocloud.plus/ru/~gitbook/image?url=https%3A%2F%2F1986541005-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252F5U1THZAib81UsHLz0L7L%252Fuploads%252FILjBlAZqVckliF60N4xw%252F7%2520ru.png%3Falt%3Dmedia%26token%3Dac120cc8-234e-4816-ad5d-3b820dab7d32&width=768&dpr=3&quality=100&sign=6c745da8&sv=2)

После открытия страницы с информацией о счете вы увидите кнопку «Подтвердить счет без оплаты».

![](https://docs.cryptocloud.plus/ru/~gitbook/image?url=https%3A%2F%2F1986541005-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252F5U1THZAib81UsHLz0L7L%252Fuploads%252Fo1Nk6Ksb8q50r4ooDvFi%252F8%2520ru.png%3Falt%3Dmedia%26token%3D0ee01701-adf2-4ffd-bd6a-c97c0a3bf00d&width=768&dpr=3&quality=100&sign=22b9c223&sv=2)

1.  При клике на данную кнопку мы отправим уведомление об успешной оплате на «URL для уведомлений» вашего проекта. После успешной доставки уведомления статус счета в личном кабинете и на платежной странице автоматически изменится на «Оплачен». Средства на баланс зачислены не будут.
    

![](https://docs.cryptocloud.plus/ru/~gitbook/image?url=https%3A%2F%2F1986541005-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252F5U1THZAib81UsHLz0L7L%252Fuploads%252FMXiAtksqD3SgICpGMRwe%252F9%2520ru.png%3Falt%3Dmedia%26token%3Decddb438-4a06-43ad-83bd-552d00d493b3&width=768&dpr=3&quality=100&sign=2832354c&sv=2)

В случае, если после нажатия на кнопку «Подтвердить счет без оплаты» статус на странице созданного счета не изменился, необходимо внести корректировки в настройки интеграции. В личном кабинете статус созданного счета автоматически изменится на «Создан». После внесения изменений в настройки интеграции отправить уведомление можно повторно.

Для просмотра статуса уведомления вы можете перейти в раздел «История POSTBACK». При клике на конкретное уведомление будет открыто окно с отправленным запросом и полученным ответом от вашего сервера. После выхода из тестового режима в данном разделе можно повторно отправить уведомление по уже оплаченному счету.

После того как интеграция будет настроена и протестирована, убедитесь в корректности заполнения всех полей в проекте. Затем отправьте заявку на выход из тестового режима — её рассмотрение займет до 24 часов с момента отправки. Статус заявки можно отслеживать в личном кабинете.

При выключенном тестовом режиме вы можете создавать счета любым доступным способом, но единственным способом их подтверждения будет полная оплата счета. Суммы данных счетов будут зачисляться на ваш баланс.

При необходимости вы можете включить тестовый режим в любое время. Для последующей активации приема платежей необходимо повторно направить заявку на выход из тестового режима.

# Инструкция по оплате криптовалютой | CryptoCloud

circle-info

Публикация инструкции по оплате способствует увеличению конверсии, особенно, со стороны клиентов с небольшим опытом в осуществлении криптовалютных операций.

* * *

Мы рады предложить вам удобный и безопасный способ совершения платежей с помощью криптовалют. \[Название компании\] принимает оплату в Tether (USDT), Bitcoin (BTC), Ethereum (ETH), Litecoin (LTC) и других криптовалютах.

Ниже представлено простое руководство, которое поможет вам в процессе оплаты.

### 

[hashtag](https://docs.cryptocloud.plus/ru/start/how-to-pay-instruction#id-1.-sozdanie-zakaza)

**1\. Создание заказа**

Создайте заказ на нашем сайте и перейдите к его оплате. В методах оплаты выберите «CryptoCloud». Вы будете перенаправлены на страницу созданного счета.

### 

[hashtag](https://docs.cryptocloud.plus/ru/start/how-to-pay-instruction#id-2.-vybor-kriptovalyuty-dlya-oplaty)

2\. Выбор криптовалюты для оплаты

Из списка поддерживаемых выберите криптовалюту и сеть, которые вы хотите использовать для оплаты. Также укажите электронный адрес для получения уведомления о платеже.

![Выбор криптовалюты для оплаты](https://docs.cryptocloud.plus/ru/~gitbook/image?url=https%3A%2F%2F1986541005-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252F5U1THZAib81UsHLz0L7L%252Fuploads%252Fj5H5zmpO6DxnwZTBOQHX%252F18%2520-%2520pay%2520ru.jpg%3Falt%3Dmedia%26token%3Dae7df797-8ada-4521-9c8e-ea1cfa0efba7&width=768&dpr=3&quality=100&sign=4d28d1d0&sv=2)

Убедитесь в том, что на вашем криптовалютном кошельке имеется достаточная сумма для осуществления платежа. Помните о том, что отправка средств в криптовалюте предполагает взимание комиссии сети.

Далее нажмите кнопку «Оплатить».

### 

[hashtag](https://docs.cryptocloud.plus/ru/start/how-to-pay-instruction#id-3.-poluchenie-adresa-platezha)

3\. Получение адреса платежа

На следующем шаге процесса оплаты будет отображен уникальный платежный адрес для осуществления транзакции, сопровождаемый QR-кодом.

![Получение адреса платежа](https://docs.cryptocloud.plus/ru/~gitbook/image?url=https%3A%2F%2F1986541005-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252F5U1THZAib81UsHLz0L7L%252Fuploads%252FUqyLm1dHWeBwxL9n06Yo%252F19%2520-%2520pay%2520ru.jpg%3Falt%3Dmedia%26token%3D3c641267-75f7-4f22-bc07-3f1e3afbc51a&width=768&dpr=3&quality=100&sign=560c43a0&sv=2)

Скопируйте платежный адрес или отсканируйте QR-код с помощью мобильного приложения вашего криптокошелька или биржи. Следите за точностью, так как отправка криптовалюты на неправильный адрес может привести к потере средств.

### 

[hashtag](https://docs.cryptocloud.plus/ru/start/how-to-pay-instruction#id-4.-sovershenie-platezha-iz-vashego-koshelka)

4\. Совершение платежа из вашего кошелька

Откройте свой криптовалютный кошелек, с которого вы собираетесь отправить платеж. В разделе «Отправить» или «Вывод» выберите соответствующую криптовалюту и сеть.

Введите скопированный адрес (или отсканируйте QR-код) вашего кошелька. Введите точную сумму криптовалюты, необходимую для совершения транзакции.

![Совершение платежа из вашего кошелька](https://docs.cryptocloud.plus/ru/~gitbook/image?url=https%3A%2F%2F1986541005-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252F5U1THZAib81UsHLz0L7L%252Fuploads%252FLITNVmLlwasCrYEr65M2%252Fru%2520%281%29.jpg%3Falt%3Dmedia%26token%3D7386dabc-8fed-4db9-8648-9abf3feace2b&width=768&dpr=3&quality=100&sign=5d793c0b&sv=2)

Помните о комиссии за транзакцию, взимаемой вашим кошельком или сетью блокчейн, которую, возможно, придется добавить к общей сумме отправки. При отправке средств необходимо проверить то, что в сервис CryptoCloud дойдет верная сумма после вычета всех комиссий.

Большинство провайдеров добавляют комиссию к необходимой сумме, но есть кошельки и биржи, в которых сумма комиссии вычитается из общей суммы отправки.

Если вы отправите неверную сумму, система определит ваш платеж и рассчитает минимальную сумму, которую необходимо доплатить.

### 

[hashtag](https://docs.cryptocloud.plus/ru/start/how-to-pay-instruction#id-5.-podtverzhdenie-i-otpravka-platezha)

5\. Подтверждение и отправка платежа

Счет доступен для оплаты в течение ограниченного количества времени. Для вашего удобства на странице оплаты вы увидите таймер обратного отсчета.

Если вы отправили средства, а счет завершился, то он подтвердится сразу, как только средства будут получены. В случае, если ваш счет истекает, а вы еще не оплатили, советуем создать или запросить новый счет для оплаты.

Прежде чем подтвердить совершение транзакции, перепроверьте адрес и сумму. Помните, что транзакции в блокчейне необратимы. Далее следуйте процедуре вашего кошелька для авторизации и отправки транзакции. Это может включать двухфакторную аутентификацию или другие меры безопасности.

### 

[hashtag](https://docs.cryptocloud.plus/ru/start/how-to-pay-instruction#id-6.-ozhidanie-podtverzhdeniya-ot-blokcheina)

6\. Ожидание подтверждения от блокчейна

После подтверждения данные о транзакции будут отправлены в блокчейн. Как только платежный сервис определит вашу транзакцию, статус на экране изменится. Скорость подтверждения операции зависит от выбранной криптовалюты и загруженности сети Blockchain. Обычно подтверждение операции занимает от 2 минут до 10 минут. Оплата счетов в Bitcoin может занимать до 1 часа и более.

Вы можете использовать проводник блокчейна для мониторинга транзакции, например [Blockchairarrow-up-right](https://blockchair.com/ru)
 или [Tronscanarrow-up-right](https://tronscan.org/)
. Вам понадобится идентификатор транзакции, который обычно предоставляется кошельком после отправки платежа.

### 

[hashtag](https://docs.cryptocloud.plus/ru/start/how-to-pay-instruction#id-7.-zavershenie-zakaza)

7\. Завершение заказа

Как только транзакция будет подтверждена платежным сервисом, вы получите уведомление об успешном платеже. В этот момент статус вашего заказа изменится на «Оплачен».

После подтверждения мы приступим к обработке вашего заказа.

### 

[hashtag](https://docs.cryptocloud.plus/ru/start/how-to-pay-instruction#rekomendacii)

*   Сохраняйте идентификатор транзакции и любую другую информацию о вашем платеже для дальнейшего использования.
    
*   Всегда проверяйте, что вы отправляете платеж на корректный адрес, правильно выбрав валюту и сеть.

# H2H-интеграция (White Label) | CryptoCloud

Host2Host (H2H) — это white-label сценарий API-интеграции, при котором реквизиты для оплаты передаются напрямую на вашу сторону без переадресации пользователя на платежную страницу CryptoCloud. Вы полностью контролируете интерфейс оплаты: отображаете адрес кошелька, сеть и сумму в дизайне вашего сайта или приложения, сохраняя единый брендированный пользовательский опыт. Статусы платежей передаются через API-уведомления и могут автоматически обрабатываться в вашей системе.

**Когда использовать H2H:**

*   требуется white-label прием криптоплатежей без упоминания платежного провайдера
    
*   запрещены редиректы на внешние страницы
    
*   необходим полный контроль над UI/UX оплаты
    
*   используется собственный checkout, приложение или терминал
    

### 

[hashtag](https://docs.cryptocloud.plus/ru/api-reference-v2/h2h#kak-rabotaet-h2h)

Как работает H2H

**1\. Авторизация запросов**

Все запросы к API подписываются вашими API-ключами из настроек проекта CryptoCloud.

**2\. Создание счета**

Вы отправляете запрос на [создание инвойса](https://docs.cryptocloud.plus/ru/api-reference-v2/create-invoice)
 через API, указывая:

*   сумму счета
    
*   валюту счета
    
*   параметр `cryptocurrency` — валюта, в которой пользователь будет оплачивать
    

В H2H обязательно указывать `cryptocurrency`, так как под выбранную валюту формируются реквизиты.

**3\. Получение реквизитов**

В ответ API возвращает:

*   уникальный адрес кошелька
    
*   точную сумму к оплате
    

Эти данные отображаются пользователю в вашем интерфейсе (white-label checkout).

**4\. Оплата пользователем**

Пользователь переводит средства на выданный адрес в указанной сети и сумме.

**5\. Уведомление о платеже (Postback)**

После поступления средств CryptoCloud отправляет POST-уведомление на URL для уведомлений, заданный в [настройках проектаarrow-up-right](https://support.cryptocloud.plus/ru/getting-started/project-settings)
.

Вы проверяете подпись уведомления и обновляете статус заказа/услуги в своей системе.

**6\. Проверка статуса счета (опционально)**

[Статус счетаarrow-up-right](https://support.cryptocloud.plus/ru/payments/invoice-statuses)
 можно запросить вручную через API по INV-номеру. Используется для сверки, повторной проверки при сбое уведомлений или ручной обработки.

### 

[hashtag](https://docs.cryptocloud.plus/ru/api-reference-v2/h2h#rezultat-integracii)

Результат интеграции

После подключения H2H:

*   оплата проходит в полностью white-label интерфейсе
    
*   пользователь не покидает ваш сайт или приложение
    
*   бренд CryptoCloud не отображается в процессе оплаты
    
*   статусы платежей автоматически синхронизируются через API
    

### 

[hashtag](https://docs.cryptocloud.plus/ru/api-reference-v2/h2h#vazhno)

Важно

*   пользователь должен оплачивать строго в указанной сети и валюте
    
*   сумма должна совпадать с выданной API
    
*   каждый счет имеет уникальный адрес
    
*   адрес действителен только для одного счета

    /**
     * Class PayoutAPIError
     *
     * Custom exception for API errors.
     */
    class PayoutAPIError extends Exception
    {
        public int $statusCode;
        public array $errorData;
    
        /**
         * PayoutAPIError constructor.
         *
         * @param int $statusCode
         * @param array $errorData
         */
        public function __construct(int $statusCode, array $errorData = [])
        {
            $this->statusCode = $statusCode;
            $this->errorData = $errorData;
            $message = "API Request failed with status $statusCode: " . json_encode($errorData);
            parent::__construct($message, $statusCode);
        }
    }
    
    /**
     * Class PayoutAPIClient
     *
     * Client for creating crypto payouts via CryptoCloud API.
     */
    class PayoutAPIClient
    {
        private string $apiKey;
        private string $baseUrl;
        private array $headers;
    
        /**
         * PayoutAPIClient constructor.
         *
         * @param string $apiKey
         */
        public function __construct(string $apiKey)
        {
            $this->apiKey = $apiKey;
            $this->baseUrl = 'https://api.cryptocloud.plus';
            $this->headers = [\
                'Authorization: Token ' . $this->apiKey,\
                'Content-Type: application/json'\
            ];
        }
    
        /**
         * Create payout request.
         *
         * @param string $currencyCode Currency code (e.g., BTC, ETH).
         * @param string $toAddress Recipient address.
         * @param float $amount Amount to send.
         * @return array Response from API.
         * @throws PayoutAPIError On API failure.
         */
        public function createPayout(string $currencyCode, string $toAddress, float $amount): array
        {
            $url = $this->baseUrl . '/v2/invoice/api/out/create';
    
            $payload = json_encode([\
                'currency_code' => $currencyCode,\
                'to_address' => $toAddress,\
                'amount' => $amount,\
            ]);
    
            $ch = curl_init($url);
    
            curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
            curl_setopt($ch, CURLOPT_POST, true);
            curl_setopt($ch, CURLOPT_POSTFIELDS, $payload);
            curl_setopt($ch, CURLOPT_HTTPHEADER, $this->headers);
    
            $response = curl_exec($ch);
            $httpCode = curl_getinfo($ch, CURLINFO_HTTP_CODE);
            $decoded = json_decode($response, true);
    
            curl_close($ch);
    
            if ($httpCode === 200) {
                return $decoded;
            } else {
                throw new PayoutAPIError($httpCode, $decoded ?? []);
            }
        }
    }

# Автоматический POSTBACK | CryptoCloud

После успешного подтверждения счета на указанный в настройках проекта URL для уведомлений отправляется POST запрос с информацией о платеже.

В системе предусмотрено 2 варианта формата отправки POSTBACK:

1.  JSON
    
2.  application/x-www-form-urlencoded
    

Для настройки типа получаемого POSTBACK необходимо перейти в настройки вашего проекта, далее раздел «Интеграция и API».

    function base64url_decode($data) {
        $remainder = strlen($data) % 4;
        if ($remainder) {
            $data .= str_repeat('=', 4 - $remainder);
        }
        $data = strtr($data, '-_', '+/');

     * Token is expected to be a raw JWT string (no "Bearer " prefix).
     */
    function validateToken($token, $secret) {
        // Token is optional
        if (!isset($token)) return true;
        if (!is_string($token)) return false;
    
        $t = trim($token);
        if ($t === '') return true;
    
        $parts = explode('.', $t);
        if (count($parts) !== 3) return false;
    
        [$headerB64, $payloadB64, $sigB64] = $parts;
    
        $headerJson = base64url_decode($headerB64);
        $payloadJson = base64url_decode($payloadB64);
        if ($headerJson === false || $payloadJson === false) return false;

        if (!is_array($header) || !is_array($payload)) return false;

        // Verify signature

# Статические кошельки | CryptoCloud

### 

[hashtag](https://docs.cryptocloud.plus/ru/api-reference-v2/static-wallet#chto-takoe-staticheskii-koshelek)

Что такое статический кошелек

Статический кошелек позволяет покупателям осуществлять платежи на один и тот же адрес без необходимости создания нового счета и перехода на страницу оплаты каждый раз.

Это удобный способ, который дает возможность принимать средства в определенной криптовалюте с автоматическим зачислением их на баланс продавца после вычета всех комиссий.

Решение отлично подходит для проектов, в которых предусмотрено пополнение баланса аккаунта пользователя, для приема пожертвований, проектов в сфере iGaming и др.

### 

[hashtag](https://docs.cryptocloud.plus/ru/api-reference-v2/static-wallet#kak-rabotaet-staticheskii-koshelek)

Как работает статический кошелек

1.  Пользователь переходит к пополнению баланса на сайте мерчанта.
    
2.  Пользователь создает свои кошельки под разные криптовалюты и пополняет любой из них. Важно - 1 кошелек = 1 валюта
    
3.  Переводит сумму по адресу кошелька или с помощью QR-кода.
    
4.  После пополнения любая сумма в криптовалюте, поступившая на кошелек, зачисляется как успешное пополнение баланса пользователя.
    
5.  Мерчант получает уведомление об успешном платеже.
    

### 

[hashtag](https://docs.cryptocloud.plus/ru/api-reference-v2/static-wallet#preimushestva-staticheskogo-koshelka)

Преимущества статического кошелька

Удобство для покупателей. Оплата в определенной криптовалюте осуществляется на один постоянный адрес без перехода на платежную страницу.

Автоматизация зачисления. Средства автоматически поступают на баланс мерчанта с отправкой postback-уведомления.
